Kunden-Preismodell definieren

Auf der Seite Kunden-Preismodell definieren können Sie für einen bestimmten Service ein kundenspezifisches Preismodell definieren oder ändern. Wenn der Service das plattformeigene Abrechnungssystem verwendet, können Sie das Preismodell direkt auf dieser Seite bearbeiten. Verwendet der Service ein externes Abrechnungssystem, können Sie ein neues oder geändertes Preismodell von diesem System importieren.

Ein Kunden-Preismodell hat Vorrang gegenüber dem ursprünglichen Service-Preismodell. Einführende Informationen zu Preismodellen finden Sie hier.

Üblicherweise ändern Sie ein Kunden-Preismodell nur in Absprache mit dem jeweiligen Kunden. Zum Ändern des Kunden-Preismodells muss der entsprechende marktfähige Service deaktiviert sein. Ihre Änderungen überschreiben die vorherigen Werte. Die Änderungen wirken sich nicht auf laufende Abonnements aus, sondern nur auf nach erneuter Aktivierung des Service angelegte Abonnements.

So definieren Sie ein Kunden-Preismodell:

  1. Wählen Sie in der Liste Kunde den Kunden, für den Sie ein Preismodell definieren möchten.

    Die Liste enthält alle Kunden, die einen Ihrer Services abonniert haben, sowie alle von Ihnen selbst registrierten Kunden, unabhängig davon, ob diese bereits einen Service abonniert haben.

  2. Wählen Sie in der Liste Service-ID den Service, für den Sie ein Preismodell definieren möchten.
  3. Abhängig von dem Abrechnungssystem, das für den zugrunde liegenden technischen Service definiert ist, fahren Sie wie folgt fort:
    • Wenn das plattformeigene Abrechnungssystem verwendet wird, definieren Sie das Preismodell in den weiteren Bereichen auf der Seite. Detaillierte Informationen dazu finden Sie hier.
    • Wenn ein externes Abrechnungssystem verwendet wird, importieren Sie das Preismodell von diesem System und geben die Lizenzvereinbarung an. Detaillierte Informationen dazu finden Sie hier.
Sobald Sie den Service aktivieren, steht dem Kunden das Preismodell zur Verfügung. Es gilt für alle neuen Abonnements des Kunden.
