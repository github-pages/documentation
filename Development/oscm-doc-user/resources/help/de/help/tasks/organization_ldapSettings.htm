LDAP-Einstellungen verwalten

Benutzernamen und Passwörter einer Organisation können in der Plattform oder in einem bestehenden LDAP-System der Organisation verwaltet werden.

Wird ein LDAP-System verwendet, muss die Organisation ihre Benutzer nicht bei der Plattform registrieren. Der Administrator der Organisation kann Benutzer aus dem LDAP-System importieren, um sie automatisch bei der Plattform zu registrieren. Die Benutzer werden im LDAP-System verwaltet. Die Benutzerinformationen werden kontinuierlich mit der Plattform synchronisiert. Die für die Verbindung zum LDAP-System erforderlichen Einstellungen müssen in einer Konfigurationsdatei hinterlegt sein.

Auf der Seite LDAP-Einstellungen können Sie die LDAP-Einstellungen für Ihre Organisation aktualisieren. Dadurch können Sie die vom Plattform-Betreiber definierten systemweiten LDAP-Einstellungen überschreiben oder zusätzliche Einstellungen hinzufügen.

Wenn Sie die LDAP-Einstellungen für Ihre Organisation ändern müssen, weil sich zum Beispiel die URL des LDAP-Servers geändert hat, können Sie die aktuellen Einstellungen in eine lokale Datei exportieren, die Einstellungen in der Datei ändern und anschließend die aktualisierten Einstellungen importieren. Durch den Import überschreiben Sie die aktuellen LDAP-Einstellungen für Ihre Organisation.

Beim Aufruf der Seite LDAP-Einstellungen werden Ihnen die aktuellen LDAP-Einstellungen angezeigt. Sie sehen, welche Einstellungen den systemweiten LDAP-Einstellungen entsprechen (sie gelten solange, bis Sie spezifische Einstellungen für Ihre Organisation definieren) und welche Schlüssel/Werte-Paare festgelegt sind.

So verwalten Sie die LDAP-Einstellungen:

  1. Legen Sie eine LDAP-Konfigurationsdatei auf Ihrem Rechner an und definieren Sie die LDAP-Verbindungseinstellungen in dieser Datei. Alternativ können Sie mit Exportieren die aktuellen LDAP-Einstellungen in einer Datei speichern.
  2. Prüfen Sie die Einstellungen in der Datei und bearbeiten Sie sie, falls erforderlich.

    Nachfolgend sehen Sie ein Beispiel für eine Konfigurationsdatei: 

    LDAP_BASE_DN=ou\=people,dc\=est,dc\=mycompany,dc\=de
LDAP_CONTEXT_FACTORY=com.sun.jndi.ldap.LdapCtxFactory
LDAP_URL=ldap://myldapserver.lan.est.company.de:389
LDAP_ATTR_UID=uid
LDAP_ATTR_LAST_NAME=sn
LDAP_ATTR_FIRST_NAME=givenName
LDAP_ATTR_EMAIL=scalixEmailAddress
LDAP_ATTR_REFERRAL=ignore
  3. Um die Konfigurationsdatei zu laden, klicken Sie das Feld Dateiname oder Durchsuchen und wählen die Datei.
  4. Laden Sie die Einstellungen mit Importieren. Die aktuellen LDAP-Einstellungen werden mit den in der Konfigurationsdatei definierten Einstellungen überschrieben.
  5. Überprüfen Sie mit Verbindung testen, ob eine Verbindung zu Ihrem LDAP-System hergestellt werden kann.

Die LDAP-Einstellungen sind sofort wirksam.

Hinweis: Wenn Sie ungültige Einstellungen importieren, ist für Sie und alle Benutzer Ihrer Organisation keine Anmeldung am System möglich. Wenden Sie sich in diesem Fall an den Plattform-Betreiber, damit er die LDAP-Einstellungen für Ihre Organisation zurücksetzt.
