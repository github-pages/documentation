Service-Preismodell definieren

Auf der Seite Service-Preismodell definieren können Sie ein Preismodell für einen ausgewählten Service definieren oder ändern.

Sie haben einen Service gewählt, für dessen Kostenberechnung das plattformeigene Abrechnungssystem verwendet wird. Sie müssen nun festlegen, ob und wieviel ein Kunde für die Benutzung des Service bezahlen muss.

So definieren oder ändern Sie ein Service-Preismodell:

  1. Wählen Sie den Abrechnungsmodus für das Preismodell:
    • Wählen Sie Kostenlos, wenn der Service kostenlos zur Verfügung gestellt werden soll.
    • Wählen Sie Abrechnung pro rata, wenn die Nutzungsgebühren für die exakte Nutzungsdauer in Millisekunden berechnet werden sollen.
    • Wählen Sie Abrechnung pro Zeiteinheit, wenn die Nutzungsgebühren für ganze Zeiteinheiten berechnet werden sollen, zum Beispiel Tage oder Wochen. Die Zeiteinheit wird im Feld Abrechnungseinheit festgelegt.

    Wenn Ihr Service kostenlos ist, fahren Sie mit Schritt 8 fort, ansonsten mit Schritt 3.

  2. Geben Sie im Feld Beschreibung eine ausführliche Beschreibung des Preismodells ein. Sie können zum Beispiel Details zu jedem Preismodellelement eingeben.

    Die Beschreibung wird Kunden auf einem Marktplatz angezeigt, wenn sie Details zu einem Service aus dem Servicekatalog anfordern.

    Sie können die Beschreibung später in weiteren Sprachen hinterlegen, indem Sie das Preismodell lokalisieren.

    Note: Verwenden Sie ausschließlich die Funktionen des Editors, um Textformatierungen vorzunehmen. Direkte HTML-Codierungen im Text sowie Verweise auf externe HTML-Stylesheets oder -Inhalte werden nicht ausgewertet und können zu Fehlern im Seitenlayout führen.
  3. Definieren Sie einen kostenlosen Nutzungszeitraum für Ihre Kunden, wenn Sie den Service für einen begrenzten Zeitraum ohne Gebühren anbieten möchten. Nach diesem Zeitraum wird der Service automatisch zu einem kostenpflichtigen Service.

    Markieren Sie Kostenlosen Nutzungszeitraum definieren und geben Sie die Dauer in Tagen an. Die Tage werden als Ganzzahl eingeben.

  4. Geben Sie die Abrechnungskonditionen an. Dazu zählen die folgenden Informationen:
    • Währung, in der mit dem Kunden abgerechnet wird. Der Plattform-Betreiber legt die unterstützten Währungen fest. Die Währung kann für ein Abo-Preismodell nicht geändert werden.
    • Abrechnungseinheit: Zeitspanne, nach der die Nutzungsgebühren abgerechnet werden. Eine monatliche, wöchentliche, tägliche oder stundenweise Abrechnung ist möglich. Die Abrechnungseinheit legt auch die Einheit fest, die bei Abrechnung pro Zeiteinheit verwendet wird. Die Zeiteinheit kann für ein Abo-Preismodell nicht geändert werden.
    • Einmalige Gebühr bei Abschluss eines Abonnements. Diese Gebühr wird zu den Nutzungsgebühren für den ersten Abrechnungszeitraum hinzugerechnet.
    • Preis pro Abo: Gebühr, die für ein Abonnement so oft bezahlt werden muss, wie in der Abrechnungseinheit angegeben.
    • Preis für Benutzer: Gebühr, die pro Zeiteinheit, die Benutzer einem Abonnement zugeordnet sind, bezahlt werden muss. Die Zeiteinheit wird im Feld Abrechnungseinheit angegeben. Diese Gebühr kann gestaffelt sein, um beispielsweise abhängig von der Zeit, die Benutzer mit einem Service arbeiten, Rabatte zu gewähren:
      • Klicken Sie das Eingabefeld zur Preiseingabe. Rechts davon wird ein Pluszeichen angezeigt.
      • Klicken Sie das Pluszeichen und geben Sie einen Preis pro Zeiteinheit und eine Obergrenze an. Die Obergrenze bestimmt die Summe der Zeiteinheiten, in denen Benutzer innerhalb eines Abrechnungszeitraums einem Abonnement zugeordnet sind, bis zu der der jeweilige Preis gültig ist.
      • Fahren Sie mit der Eingabe weiterer Preise und Obergrenzen fort.
      • Im letzten Feld Preis pro Zeiteinheit geben Sie den Preis an, der für alle Zeiteinheiten gelten soll, die über der im letzten Feld Obergrenze angegebenen Anzahl liegen.

      Ausführliche Informationen finden Sie hier.

  5. Geben Sie Preise für die abrechenbaren Ereignisse an, die für den Service definiert sind.

    Ein Ereignis ist beispielsweise das An- oder Abmelden von der zugrunde liegenden Anwendung, das Beenden einer bestimmten Transaktion oder das Anlegen oder Löschen bestimmter Daten. Für welche Ereignisse Preise definiert werden können, hängt von der Implementierung der Anwendung und ihrer Integration ab.

    Die definierten Ereignisse werden in der Tabelle aufgelistet. In der Spalte Beschreibung finden Sie jeweils eine Kurzbeschreibung.

    Diese Gebühr kann gestaffelt sein, um beispielsweise abhängig von der Anzahl der Ereignisse, die während der Nutzung eines abonnierten Service auftreten, Rabatte zu gewähren:
    • Klicken Sie das Eingabefeld zur Preiseingabe. Rechts davon wird ein Pluszeichen angezeigt.
    • Klicken Sie das Pluszeichen und geben Sie einen Preis pro Ereignis und eine Obergrenze an. Die Obergrenze bestimmt die Ereignisanzahl, bis zu der der jeweilige Preis gültig ist.
    • Fahren Sie mit der Eingabe weiterer Preise pro Ereignis und Obergrenzen fort.
    • Im letzten Feld Preis geben Sie den Preis an, der für alle Ereignisse des gewählten Typs gelten soll, die über der im letzten Feld Obergrenze angegebenen Anzahl liegen.
    Ausführliche Informationen finden Sie hier.
  6. Geben Sie Preise für Parameter und Optionen an.

    Ob und welche Parameter und Optionen zur Auswahl stehen, ist abhängig von der Implementierung und Integration der zugrunde liegenden Anwendung.

    Die definierten Parameter und Optionen werden in der Tabelle aufgelistet. In der Spalte Beschreibung finden Sie jeweils eine Kurzbeschreibung.

    Für jeden Parameter und jede Option kann ein Preis festgelegt werden, der jeweils für ein Abonnement als ganzes oder pro Benutzer berechnet werden kann. Numerische Parameter sind Multiplikatoren für den angegebenen Preis. Bei booleschen Parametern mit dem Wert true ist der Multiplikator 1. In allen anderen Fällen ist der Multiplikator 0.

    Bei numerischen Parametern (Parameter mit Werten vom Typ Integer oder Long) können Sie die Preise pro Abonnement staffeln, um beispielsweise Mengenrabatte zu gewähren:
    • Klicken Sie das Eingabefeld zur Preiseingabe. Rechts davon wird ein Pluszeichen angezeigt.
    • Klicken Sie das Pluszeichen und geben Sie einen Preis und eine Obergrenze an. Die Obergrenze bestimmt den Wert, bis zu dem der jeweilige Preis gültig ist.
    • Fahren Sie mit der Eingabe weiterer Preise und Obergrenzen fort.
    • Im letzten Feld Preis geben Sie den Preis an, der für alle Werte gelten soll, die über dem im letzten Feld Obergrenze angegebenen Wert liegen.
    Weitere Informationen finden Sie hier.
  7. Ändern Sie nach Bedarf die Lizenzvereinbarung. Die Lizenzvereinbarung muss von Kunden gelesen und akzeptiert werden, wenn sie den Service abonnieren.

    Standardmäßig wird die Lizenzvereinbarung des Herstellers angezeigt. Sie können sie übernehmen oder eine andere eingeben.

    Sie können die Lizenzvereinbarung später in weiteren Sprachen hinterlegen, indem Sie das Preismodell lokalisieren.

    Wiederverkäufer, die den Service anbieten, können Ihre Lizenzvereinbarung übernehmen oder eine eigene hinterlegen. Wenn Sie als Serviceanbieter die Lizenzvereinbarung ändern, wird die Lizenzvereinbarung der Wiederverkäufer nicht aktualisiert.

  8. Schließen Sie Ihre Eingaben mit Speichern ab.
  9. Um die Beschreibung und die Lizenzvereinbarung des Preismodells in einer anderen Sprache zu hinterlegen, klicken Sie Lokalisieren.
  10. Um Preise für Servicerollen zu definieren, die für die zugrunde liegende Anwendung definiert sind, klicken Sie Für Servicerollen definieren. Der Preis, den Sie für eine Servicerolle festlegen, wird bei der Berechnung der Nutzungsgebühren für einen Abrechnungszeitraum zum Grundpreis für Benutzer addiert.

Sobald Sie den Service aktivieren, steht Kunden das Preismodell zur Verfügung. Es gilt für alle neuen Abonnements.

Vermittler und Wiederverkäufer und deren Kunden sehen Ihre Änderungen sofort.
