Trigger verwalten

Organisationen haben häufig eigene Prozesse für die Registrierung von Benutzern, für das Abonnieren von Services oder für die Definition von Preisen. Normalerweise sehen solche Prozesse Genehmigungsverfahren vor und werden mit Hilfe eines Prozesssteuerungssystems automatisiert.

Für bestimmte von der Plattform oder von Benutzern gestartete Aktionen bietet die Plattform die Möglichkeit, mit einem externen Prozesssteuerungssystem zusammenzuarbeiten: Sie können Trigger konfigurieren, die aufgerufen werden, sobald eine Aktion ausgeführt wird. Die Trigger starten den jeweiligen Prozess im Prozesssteuerungssystem. Ist eine Genehmigung erforderlich, wartet die Aktion mit ihrer Ausführung so lange, bis sie im Prozesssteuerungssystem genehmigt oder abgelehnt wird. Ist keine Genehmigung erforderlich, wird die Aktion sofort ausgeführt.

Voraussetzung für die Steuerung von Aktionen durch externe Prozesse ist ein Notification Service. Dieser Service bildet die Schnittstelle zwischen der Plattform und dem Prozesssteuerungssystem. Informationen zu diesem Service erhalten Sie von Ihrem Plattform-Betreiber.

Auf der Seite Prozesse verwalten sehen Sie die Aktionen, die von einem Prozesssteuerungssystem gesteuert und überwacht werden. Auf dieser Seite können Sie noch nicht ausgeführte Aktionen abbrechen oder abgebrochene Aktionen löschen.

Auf der Seite Trigger verwalten sehen Sie alle Trigger, die für Ihre Organisation konfiguriert sind. Sie können Trigger hinzufügen, bearbeiten oder löschen.

So verwalten Sie die Trigger Ihrer Organisation:

  • Mit Hinzufügen konfigurieren Sie einen neuen Trigger. Auf der Folgeseite geben Sie dann die Daten für den Trigger ein.
  • Durch Klicken eines Eintrags in der Liste können Sie einen Trigger bearbeiten oder löschen. Fahren Sie auf der Folgeseite mit der Bearbeitung des Triggers fort oder löschen Sie den Trigger.
