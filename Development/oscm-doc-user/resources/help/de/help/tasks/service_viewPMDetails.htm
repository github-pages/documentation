Details zum Preismodell

Auf den Seiten Service anzeigen und Service verwalten können Sie folgende Details zu Abrechnungsbedingungen und Preisen sehen, wenn der gewählte Service kostenpflichtig ist. Alle Preise werden ohne Mehrwertsteuer angegeben.

Allgemeine Abrechnungsbedingungen

Die allgemeinen Abrechnungsbedingungen umfassen folgende Informationen:

  • Abrechnungsart:
    • Kostenlos: Die Nutzung des Service ist kostenlos.
    • Abrechnung pro rata: Die Nutzungsgebühren werden auf Grundlage der exakten Nutzungsdauer des Service in Millisekunden abgerechnet.
    • Abrechnung pro Zeiteinheit: Die Nutzungsgebühren werden für angefangene Zeiteinheiten abgerechnet, beispielsweise für Tage oder Wochen. Die Zeiteinheit ist durch die Abrechnungseinheit festgelegt.
  • Kostenloser Nutzungszeitraum (in Tagen): Anzahl der Tage, die der Service kostenlos genutzt werden kann. Nach diesem Zeitraum wird der Service automatisch zu einem kostenpflichtigen Service.
  • Währung, in der mit dem Kunden abgerechnet wird. Der Plattform-Betreiber legt die unterstützten Währungen fest. Die Währung kann für ein Abo-Preismodell nicht geändert werden.
  • Abrechnungseinheit: Zeitspanne, nach der die Nutzungsgebühren abgerechnet werden. Eine monatliche, wöchentliche, tägliche oder stundenweise Abrechnung ist möglich. Die Abrechnungseinheit legt auch die Einheit fest, die bei Abrechnung pro Zeiteinheit verwendet wird. Die Zeiteinheit kann für ein Abo-Preismodell nicht geändert werden.
  • Einmalige Gebühr bei Abschluss eines Abonnements. Diese Gebühr wird zu den Nutzungsgebühren für den ersten Abrechnungszeitraum hinzugerechnet.
  • Preis pro Abo: Gebühr, die für ein Abonnement so oft bezahlt werden muss, wie in der Abrechnungseinheit angegeben.
  • Preis für Benutzer: Gebühr, die pro Zeiteinheit, die Benutzer einem Abonnement zugeordnet sind, bezahlt werden muss. Die Zeiteinheit wird im Feld Abrechnungseinheit angegeben. Diese Gebühr kann gestaffelt sein, um beispielsweise abhängig von der Zeit, die Benutzer mit einem Service arbeiten, Rabatte zu gewähren.

Preise für abrechenbare Ereignisse

Für einzelne Ereignisse können Preise festgelegt werden. Ein Ereignis ist beispielsweise das An- oder Abmelden von der zugrunde liegenden Anwendung, das Beenden einer bestimmten Transaktion oder das Anlegen oder Löschen bestimmter Daten. Für welche Ereignisse Preise definiert werden können, hängt von der Implementierung der Anwendung und ihrer Integration ab.

Diese Gebühr kann gestaffelt sein, um beispielsweise abhängig von der Anzahl der Ereignisse, die während der Nutzung eines abonnierten Service auftreten, Rabatte zu gewähren

Preise für Parameter

Ein Preismodell kann Preise für Parameter und Optionen enthalten. Ob und welche Parameter und Optionen zur Auswahl stehen, ist abhängig von der Implementierung und Integration der zugrunde liegenden Anwendung. Der Serviceanbieter kann für jeden Parameter und jede Option einen Preis festlegen, der jeweils für ein Abonnement als ganzes oder pro Benutzer berechnet werden kann. Numerische Parameter sind Multiplikatoren für den angegebenen Preis. Bei booleschen Parametern mit dem Wert true ist der Multiplikator 1, ansonsten 0. In allen anderen Fällen ist der Multiplikator ebenfalls 0.

Bei numerischen Parametern ist eine Preisstaffelung pro Abonnement möglich: Abhängig von den Werten des Parameters können unterschiedliche Preise definiert werden.

Rollenspezifische Preise

Ein Preismodell kann Preise für Servicerollen festlegen, die für die zugrunde liegende Anwendung definiert sind. In der Kostenberechnung für einen Abrechnungszeitraum wird dieser Preis zu dem Grundpreis pro Benutzer addiert.

Die Servicerollen werden bei der Definition des technischen Service festgelegt. In der Anwendung werden sie dann in die entsprechenden Privilegien umgesetzt.
