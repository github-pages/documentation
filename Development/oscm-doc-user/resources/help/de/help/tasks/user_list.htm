Benutzer verwalten

Auf der Seite Benutzer verwalten können Sie die für Ihre Organisation registrierten Benutzer verwalten. Sie können Benutzerdaten und Benutzerrollen ändern, Passwörter zurücksetzen und Benutzer löschen.

Hinweis: Wenn Ihre Organisation ein externes System zur Benutzerauthentifizierung verwendet, können Passwörter nur im externen System geändert werden. Die Option zum Zurücksetzen des Passworts steht nicht zur Verfügung.

So ändern Sie Benutzerdaten und Benutzerrollen:

  1. Wählen Sie in der Liste im oberen Bereich der Seite den Benutzer, dessen Daten oder Rollen Sie ändern möchten.
  2. Bearbeiten Sie die Felder mit den Benutzerdaten im unteren Bereich der Seite. Wenn Ihre Organisation ein externes System zur Benutzerauthentifizierung verwendet, können nur die Felder geändert werden, die nicht mit Einträgen im externen System übereinstimmen.
  3. Um die zugeordneten Benutzerrollen des Benutzers zu ändern, markieren Sie die gewünschten Rollen oder entfernen die jeweiligen Markierungen. Die zur Verfügung stehenden Benutzerrollen hängen von der Rolle Ihrer Organisation ab.
    • Administrator: Jede Organisation muss mindestens einen Benutzer haben, dem die Administratorrolle zugeordnet ist. Ein Administrator kann das Konto seiner Organisation verwalten sowie die Abonnements, Benutzer und Benutzerrollen. Der erste Administrator wird beim Anlegen der Organisation festgelegt.
    • OE-Administrator: Die Benutzer einer Organisation können in Organisationseinheiten (OE) gruppiert werden. Benutzer mit der Rolle OE-Administrator können Organisationseinheiten verwalten, für die sie als Administrator eingesetzt wurden, sowie Abonnements für diese Einheiten anlegen, ändern und beenden. Die Rolle wird Benutzern, die als Administratoren für Organisationseinheiten eingesetzt werden, automatisch zugeordnet.
    • Abo-Manager: Benutzer mit dieser Rolle können Services abonnieren und ihre eigenen Abonnements verwalten. Im Gegensatz zu Administratoren dürfen Abo-Manager nicht die Abonnements anderer Benutzer bearbeiten. Darüber hinaus dürfen sie keine Zahlungsinformationen ändern.
    • Technologie-Manager: Benutzer mit dieser Rolle können technische Services definieren. Die Rolle kann Benutzern in Organisationen mit der Rolle Hersteller zugewiesen werden.
    • Service-Manager: Benutzer mit dieser Rolle können marktfähige Services und Preismodelle definieren sowie marktfähige Services veröffentlichen. Die Rolle kann Benutzern in Organisationen mit der Rolle Serviceanbieter zugewiesen werden.
    • Wiederverkäufer: Benutzer mit dieser Rolle können marktfähige Services eines Serviceanbieters unter ihren eigenen allgemeinen Geschäftsbedingungen veröffentlichen. Die Rolle kann Benutzern in Organisationen mit der Rolle Wiederverkäufer zugewiesen werden.
    • Vermittler: Benutzer mit dieser Rolle können marktfähige Services eines Serviceanbieters veröffentlichen, es gelten aber die allgemeinen Geschäftsbedingungen des Serviceanbieters. Die Rolle kann Benutzern in Organisationen mit der Rolle Vermittler zugewiesen werden.
    • Marktplatz-Manager: Benutzer mit dieser Rolle können festlegen, welche Organisationen Zugang zu einem Marktplatz haben und Services darauf veröffentlichen dürfen. Sie können darüber hinaus die Eigenschaften eines Marktplatzes ändern und Anpassungen des Marktplatzes an die individuellen Vorgaben Ihrer Organisation vornehmen. Die Rolle kann Benutzern in Organisationen zugewiesen werden, die Marktplatzeigentümer sind. Beim Anlegen des Marktplatzes wird sie automatisch allen Administratoren des Marktplatzeigentümers zugewiesen.
    • Betreiber: Benutzer mit dieser Rolle können Konfigurations- und Wartungsaufgaben übernehmen sowie Organisationen verwalten und Marktplätze erstellen. Der erste Benutzer mit der Rolle Betreiber wird angelegt, wenn die Organisation mit der gleichnamigen Rolle beim Installieren der Plattform angelegt wird.

    Wenn Sie für den Benutzer keine Rolle markieren, wird ihm automatisch die Rolle eines Standardbenutzers zugeordnet. Standardbenutzer können die von ihrer Organisation abonnierten Services nutzen.

    Wenn Sie die Rollen eines Benutzers ändern, steht dem Benutzer die den Rollen entsprechende Funktionalität bei seiner nächsten Anmeldung zur Verfügung.

  4. Schließen Sie Ihre Änderungen mit Speichern ab.
So setzen Sie das Passwort eines Benutzers zurück:
  1. Wählen Sie in der Liste im oberen Bereich der Seite den Benutzer, dessen Passwort Sie zurücksetzen möchten. Um ein Passwort zurückzusetzen, darf der Benutzer nicht angemeldet sein.
  2. Klicken Sie Passwort zurücksetzen.

Das Passwort wird sofort zurückgesetzt. Der Benutzer erhält per E-Mail ein neues Passwort, mit dem er sich beim nächsten Mal anmelden kann.

So löschen Sie einen Benutzer:

  1. Wählen Sie in der Liste im oberen Bereich der Seite den zu löschenden Benutzer. Zum Löschen eines Benutzers müssen folgende Voraussetzungen erfüllt sein:
    • Der Benutzer ist keinem Abonnement zugeordnet.
    • Der Benutzer ist nicht der einzige Benutzer in Ihrer Organisation, dem die Administratorrolle zugeordnet ist.
    • Der Benutzer ist nicht angemeldet.
  2. Klicken Sie Löschen und bestätigen Sie den Löschauftrag.

Der Benutzer wird sofort von der Plattform gelöscht.
