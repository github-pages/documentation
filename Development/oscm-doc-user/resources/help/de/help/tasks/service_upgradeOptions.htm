Up-/Downgrade-Optionen definieren

Auf der Seite Up-/Downgrade-Optionen definieren können Sie Services festlegen, die Sie Ihren Kunden als Optionen zum Umstellen von Abonnements anbieten möchten.

Up-/Downgrade-Optionen erlauben es den Kunden, ein Abonnement auf andere marktfähige Services umzustellen. Die Optionen bieten Ihnen die Möglichkeit, flexibel auf sich ändernde Anforderungen Ihrer Kunden zu reagieren. Ein Abonnement für eine Testlizenz kann zum Beispiel auf eine Standard- oder Enterprise-Lizenz umgestellt werden, oder ein Abonnement für eine unbegrenzte Nutzeranzahl kann auf eine begrenzte Nutzeranzahl umgestellt werden.

Sie können eine beliebige Anzahl an Up-/Downgrade-Optionen für einen marktfähigen Service definieren. Allen Optionen muss derselbe technische Service zugrunde liegen, und alle Optionen müssen auf demselben Marktplatz veröffentlicht werden. Wenn der marktfähige Service kostenpflichtig ist, müssen die Preismodelle der Up-/Downgrade-Optionen dieselbe Währung verwenden.

Sie können Up-/Downgrade-Optionen nur für Services definieren, die deaktiviert sind.

Kunden von Vermittlern oder Wiederverkäufern, denen Sie die Berechtigung zum Wiederverkauf des Service erteilt haben, stehen die Up-/Downgrade-Optionen nicht zur Verfügung.

Sobald ein Kunde ein Abonnement umstellt, gilt das Preismodell des neuen Service für die Kostenabrechnung. Dem Abonnement zugeordnete Benutzer und Servicerollen oder bereits gespeicherte Anwendungsdaten bleiben erhalten. Alle Parameter und Optionen sowie die Zahlungsinformationen werden automatisch dem neuen Service zugeordnet, außer sie werden bei der Umstellung explizit geändert.

Beachten Sie, wie sich Up-/Downgrade-Optionen auf die Berechnung der Nutzungsgebühren auswirken (Details zu den Preismodellelementen finden Sie hier):

  • Die Preise für die Servicenutzung können pro Zeiteinheit abgerechnet werden. Wenn ein Kunde ein Abonnement umstellt, das bereits pro Zeiteinheit abgerechnet wurde, werden die Kosten für die Zeiteinheit, in der die Umstellung erfolgt, doppelt berechnet.
  • Ist für einen Service, den Sie als Up-/Downgrade-Option definiert haben, eine einmalige Gebühr festgelegt, so wird diese bei der Umstellung auf diesen Service in Rechnung gestellt. Dies gilt auch, wenn für den umgestellten Service bereits einen einmalige Gebühr berechnet wurde.
  • Ist für einen Service, den Sie als Up-/Downgrade-Option definiert haben, ein kostenloser Nutzungszeitraum festgelegt, beginnt dieser unmittelbar nach der Umstellung. Dies gilt unabhängig davon, ob für den umgestellten Service bereits ein kostenloser Nutzungszeitraum definiert war.

So definieren Sie Up-/Downgrade-Optionen:

  1. Wählen Sie im Feld Service-ID den Service, für den Sie Up-/Downgrade-Optionen definieren möchten.
  2. Markieren Sie die Services in der Tabelle, die als Up-/Downgrade-Optionen zur Verfügung gestellt werden sollen. Die Tabelle listet alle marktfähigen Services auf, denen derselbe technische Service zugrunde liegt wie dem aktuellen marktfähigen Service.
  3. Schließen Sie Ihre Änderungen mit Speichern ab.
Sobald Sie den Service wieder aktivieren, stehen die Up-/Downgrade-Optionen Ihren Kunden zur Verfügung.
