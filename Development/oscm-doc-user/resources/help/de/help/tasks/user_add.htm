Benutzer registrieren

Auf der Seite Benutzer registrieren können Sie für Ihre Organisation neue Benutzer registrieren und deren Benutzerrolle festlegen. Durch den Import einer Datei mit Benutzerdaten können Sie auch mehrere Benutzer mit der Rolle eines Administrators oder eines Managers auf einmal registrieren. Dies ist nur möglich, wenn Ihre Organisation kein externes System zur Benutzerauthentifizierung verwendet. Wenn Ihre Organisation ein externes Authentifizierungssystem verwendet, können Sie die Benutzer aus dem externen System importieren und somit registrieren.

Hinweis: Sie können Benutzer selbst dann registrieren, wenn Ihre Organisation einen externen OIDC-Provider wie Microsoft Azure Active Directory zur Benutzerverwaltung und -authentifizierung verwendet. Jedoch werden diese Benutzer bei der nächsten Synchronisierung mit dem externen Provider wieder gelöscht.

Die Rolle Ihrer Organisation bestimmt, welche Rollen Sie den neuen Benutzern zuordnen können. Die Benutzerrollen steuern, welche Optionen einem Benutzer an der Benutzeroberfläche zur Verfügung stehen:

  • Standardbenutzer: Benutzer mit dieser Rolle können mit Services arbeiten, die ihre Organisation abonniert hat. Jeder registrierte Benutzer ist automatisch ein Standardbenutzer. Weitere Benutzerrollen müssen explizit vom Administrator zugeordnet werden.
  • Administrator: Jede Organisation muss mindestens einen Benutzer haben, dem die Administratorrolle zugeordnet ist. Ein Administrator kann das Konto seiner Organisation verwalten sowie die Abonnements, Benutzer und Benutzerrollen. Der erste Administrator wird beim Anlegen der Organisation festgelegt.
  • OE-Administrator: Die Benutzer einer Organisation können in Organisationseinheiten (OE) gruppiert werden. Benutzer mit der Rolle OE-Administrator können Organisationseinheiten verwalten, für die sie als Administrator eingesetzt wurden, sowie Abonnements für diese Einheiten anlegen, ändern und beenden. Die Rolle wird Benutzern, die als Administratoren für Organisationseinheiten eingesetzt werden, automatisch zugeordnet.
  • Abo-Manager: Benutzer mit dieser Rolle können Services abonnieren und ihre eigenen Abonnements verwalten. Im Gegensatz zu Administratoren dürfen Abo-Manager nicht die Abonnements anderer Benutzer bearbeiten. Darüber hinaus dürfen sie keine Zahlungsinformationen ändern.
  • Technologie-Manager: Benutzer mit dieser Rolle können technische Services definieren. Die Rolle kann Benutzern in Organisationen mit der Rolle Hersteller zugewiesen werden.
  • Service-Manager: Benutzer mit dieser Rolle können marktfähige Services und Preismodelle definieren sowie marktfähige Services veröffentlichen. Die Rolle kann Benutzern in Organisationen mit der Rolle Serviceanbieter zugewiesen werden.
  • Wiederverkäufer: Benutzer mit dieser Rolle können marktfähige Services eines Serviceanbieters unter ihren eigenen allgemeinen Geschäftsbedingungen veröffentlichen. Die Rolle kann Benutzern in Organisationen mit der Rolle Wiederverkäufer zugewiesen werden.
  • Vermittler: Benutzer mit dieser Rolle können marktfähige Services eines Serviceanbieters veröffentlichen, es gelten aber die allgemeinen Geschäftsbedingungen des Serviceanbieters. Die Rolle kann Benutzern in Organisationen mit der Rolle Vermittler zugewiesen werden.
  • Marktplatz-Manager: Benutzer mit dieser Rolle können festlegen, welche Organisationen Zugang zu einem Marktplatz haben und Services darauf veröffentlichen dürfen. Sie können darüber hinaus die Eigenschaften eines Marktplatzes ändern und Anpassungen des Marktplatzes an die individuellen Vorgaben Ihrer Organisation vornehmen. Die Rolle kann Benutzern in Organisationen zugewiesen werden, die Marktplatzeigentümer sind. Beim Anlegen des Marktplatzes wird sie automatisch allen Administratoren des Marktplatzeigentümers zugewiesen.
  • Betreiber: Benutzer mit dieser Rolle können Konfigurations- und Wartungsaufgaben übernehmen sowie Organisationen verwalten und Marktplätze erstellen. Der erste Benutzer mit der Rolle Betreiber wird angelegt, wenn die Organisation mit der gleichnamigen Rolle beim Installieren der Plattform angelegt wird.

So registrieren Sie einen neuen Benutzer:

  1. Geben Sie die folgenden persönlichen Daten für den neuen Benutzer in die entsprechenden Felder ein.
    • Anrede: Anrede oder Titel, zum Beispiel Herr oder Frau.
    • Vorname des Benutzers.
    • Nachname des Benutzers.
    • E-Mail: E-Mail-Adresse des Benutzers. Sie wird verwendet, um den Benutzer über die erfolgreiche Registrierung zu informieren.

      Das System validiert, ob die Syntax der E-Mail-Adresse gültig ist und ob der Domain-Name den Standards der Internet Assigned Numbers Authority (IANA) entspricht.

      Beispiele: user.name@domain.arpa, user.name@domain.org, user@mycompany.lan.uk

    • Benutzername: Name, mit dem sich der Benutzer anmeldet. Benutzernamen dürfen maximal 100 Zeichen lang sein. Die folgenden Zeichen sind nicht erlaubt:

      ! " # $ % & ' * + , / : ; < = > ? \ ^ `

      Hinweis: Falls Ihre Organisation ein externes System zur Benutzerauthentifizierung verwendet, muss der Benutzername mit einem existierenden Benutzernamen im externen System übereinstimmen.
  2. Wählen Sie die Sprache, in der der Benutzer arbeiten möchte. In dieser Sprache werden dem Benutzer die Benutzeroberfläche, Dokumentation und E-Mails angezeigt. Der Benutzer kann die Sprache später ändern. Der Plattform-Betreiber definiert, welche Sprachen unterstützt werden.
  3. Um einem Benutzer besondere Rechte innerhalb Ihrer Organisation zuzuweisen, ordnen Sie ihm eine oder mehrere Benutzerrollen zu. Markieren Sie dazu nach die entsprechenden Optionen.

    Die zur Auswahl stehenden Benutzerrollen hängen von der Rolle Ihrer Organisation ab. Standardbenutzer können nur auf einem Marktplatz registriert werden.

  4. Speichern Sie Ihre Angaben mit Registrieren.

Der neue Benutzer erhält per E-Mail seine Anmeldedaten und ein Passwort für die erste Anmeldung. Wenn Ihre Organisation ein externes Authentifizierungssystem verwendet, werden Passwörter in diesem System verwaltet. Daher enthält die E-Mail an den neuen Benutzer kein Passwort. Der Benutzer muss für die Anmeldung das im externen System hinterlegte Passwort verwenden.

Der neue Benutzer wird automatisch zur Standard-Organisationseinheit Ihrer Organisation hinzugefügt. Dieser Einheit sind alle Benutzer zugeordnet, die aktuell für Ihre Organisation registriert sind.

So registrieren Sie mehrere neue Administratoren und Manager:

  1. Legen Sie eine Datei mit den erforderlichen Benutzerdaten an. Details zum Format und Inhalt der Datei finden Sie hier.
  2. Importieren Sie die Datei mit den Benutzerdaten:
    1. Klicken Sie das Feld Dateiname oder Durchsuchen und wählen Sie die Datei mit den Benutzerdaten.
    2. Klicken Sie Importieren.

      Die Plattform validiert die Benutzerdaten, bevor der eigentliche Import gestartet wird. Folgendes wird geprüft:

      • Ob alle Zeilen mit Benutzerdaten vollständig ausgefüllt sind (alle Felder müssen angegeben sein).
      • Ob die Schreibweise der Anrede und Benutzerrollen korrekt ist.
      • Wenn mehrere Benutzerrollen angegeben sind, ob diese in doppelte Anführungszeichen eingeschlossen ist.
      Wenn die Plattform bei der Validierung einen Fehler feststellt, wird eine Fehlermeldung angezeigt und der Import nicht gestartet.

      Sie müssen die Informationen in der Datei korrigieren, bevor Sie den Importvorgang erneut starten können.

      Sie können nicht mehrere Importvorgänge gleichzeitig starten. Sie müssen warten, bis der aktuelle Vorgang beendet ist. Bei einer großen Benutzeranzahl kann dies einige Zeit in Anspruch nehmen.

Nach dem Import erhält jeder erfolgreich importierte und registrierte Benutzer per E-Mail seine Anmeldedaten und ein Passwort für die erste Anmeldung. Die neuen Benutzer werden automatisch zur Standard-Organisationseinheit Ihrer Organisation hinzugefügt. Dieser Einheit sind alle Benutzer zugeordnet, die aktuell für Ihre Organisation registriert sind.

Sie selbst erhalten eine E-Mail mit folgenden Informationen:

  • Anzahl der erfolgreich importierten und registrierten Benutzer.
  • Zu jedem nicht importierten Benutzer den Benutzernamen und den Grund für den Importfehler.

    Mögliche Gründe sind:

    • Es existiert bereits ein Benutzer mit dem angegebenen Benutzernamen.
    • Der angegebene Benutzername ist ungültig.
    • Die angegebene Benutzerrolle steht für Ihre Organisation nicht zur Verfügung. Als Serviceanbieter können Sie zum Beispiel nicht die Benutzerrolle BROKER_MANAGER zuweisen.
Hinweis: Um für Ihre Organisation mehrere Benutzer als Standardbenutzer zu registrieren, gehen Sie wie folgt vor:
  1. Legen Sie eine Datei mit den Standardbenutzern an, die Sie registrieren möchten. Geben Sie keine Benutzerrolle für diese Benutzer an.
  2. Melden Sie sich an einem Marktplatz an und importieren Sie dort die Datei mit den Benutzerdaten (Benutzerkonto --> Benutzer).
