Aktionen von Service-Managern

Im folgenden werden Meldungen erläutert, die in das Aktionsprotokoll geschrieben werden, wenn Service-Manager eines Serviceanbieters Aktionen im Administrationsportal ausführen. Einige der Aktionen können auch von Wiederverkäufern oder Vermittlern ausgeführt werden.

Folgende weitere <Name/Wert-Paare> werden im Anschluss an den Header und die für alle Meldungen gültigen Name/Wert-Paare ausgegeben:

  • serviceId=<Service-ID>: ID des marktfähigen Service, wie sie beim Anlegen angegeben wurde.

  • serviceName=<Name>: Servicename für Kunden des marktfähigen Service, wie er beim Anlegen angegeben wurde.

Abhängig von der protokollierten Aktion können zusätzliche Name/Wert-Paare ausgegeben werden.

Aktion: Activate/deactivate service

Beschreibung: Ein Service-Manager, Wiederverkäufer oder Vermittler aktivierte oder deaktivierte einen marktfähigen Service.

Weitere Name/Wert-Paare:

  • marketplaceId=<Marktplatz-ID>: ID des Marktplatzes, auf dem der Service veröffentlicht ist.
  • marketplaceName=<Marktplatzname>: Name des Marktplatzes, auf dem der Service veröffentlicht ist.
  • activation=<Ja oder Nein>: Gibt an, ob der Service aktiviert ist. Folgende Werte können ausgegeben werden: ON (aktiviert) oder OFF (nicht aktiviert).
  • inCatalog=<Ja oder Nein>: Gibt an, ob der Service in den Servicekatalog aufgenommen werden soll. Folgende Werte können ausgegeben werden: ON (im Katalog) oder OFF (nicht im Katalog).

Aktion: Assign brokers

Beschreibung: Bei der Vorbereitung zur Veröffentlichung eines Service berechtigte ein Service-Manager einen Vermittler, die Services seines Serviceanbieters zu verkaufen.

Weiteres Name/Wert-Paar:

BrokerId=<ID>: ID der Vermittlerorganisation, der die Berechtigung zum Wiederverkauf erteilt wird.

Aktion: Assign categories

Beschreibung: Bei der Vorbereitung zur Veröffentlichung eines Service legte ein Service-Manager, Wiederverkäufer oder Vermittler eine oder mehrere Kategorien für einen marktfähigen Service fest.

Weiteres Name/Wert-Paar:

listOfCategories=<Namen>: Die zugeordneten Kategorien. Die Listeneinträge sind durch Komma getrennt.

Aktion: Assign resellers

Beschreibung: Bei der Vorbereitung zur Veröffentlichung eines Service berechtigte ein Service-Manager einen Wiederverkäufer, die Services seines Serviceanbieters zu verkaufen.

Weiteres Name/Wert-Paar:

ResellerId=<ID>: ID der Wiederverkäuferorganisation, der die Berechtigung zum Wiederverkauf erteilt wird.

Aktion: Assign service to marketplace

Beschreibung: Bei der Vorbereitung zur Veröffentlichung eines Service definierte oder änderte ein Service-Manager, Wiederverkäufer oder Vermittler den Marktplatz, auf dem der marktfähige Service veröffentlicht werden soll.

Weitere Name/Wert-Paare:

  • marketplaceId=<ID>: ID des Marktplatzes.
  • marketplaceName=<Name>: Name des Marktplatzes.

Aktion: Copy service

Beschreibung: Ein Service-Manager erstellte eine Kopie eines marktfähigen Service.

Weitere Name/Wert-Paare:

  • copyId=<Service-ID>: ID des kopierten marktfähigen Service, wie sie in der Datenbank gespeichert ist.

  • copyName=<Servicename>: Name des kopierten marktfähigen Service, wie er beim Kopieren angegeben wurde.

Aktion: Deassign brokers

Beschreibung: Bei der Vorbereitung zur Veröffentlichung eines Service entzog ein Service-Manager einem Vermittler die Berechtigung zum Wiederverkauf der Services des Serviceanbieters.

Weiteres Name/Wert-Paar:

BrokerId=<Organisations-ID>: ID der Vermittlerorganisation, der die Berechtigung zum Wiederverkauf der Services entzogen wurde.

Aktion: Deassign resellers

Beschreibung: Bei der Vorbereitung zur Veröffentlichung eines Service entzog ein Service-Manager einem Wiederverkäufer die Berechtigung zum Wiederverkauf der Services des Serviceanbieters.

Weiteres Name/Wert-Paar:

ResellerId=<Organisations-ID>: ID der Wiederverkäuferorganisation, der die Berechtigung zum Wiederverkauf der Services entzogen wurde.

Aktion: Define service

Beschreibung: Ein Service-Manager definierte einen marktfähigen Service. Pro Parameter und Parameteroption, die für den marktfähigen Service definiert wurden, wird je eine Meldung Update service parameter ausgegeben.

Weitere Name/Wert-Paare:

  • technServiceName=<Servicename>: Name des technischen Service, der dem marktfähigen Service zugrunde liegt.
  • shortDescription=<Ja oder Nein>: Gibt an, ob eine Kurzbeschreibung für den marktfähigen Service eingegeben wurde. Folgende Werte können ausgegeben werden: YES (Ja) oder NO (Nein).
  • description=<Ja oder Nein>: Gibt an, ob eine Beschreibung für den marktfähigen Service eingegeben wurde. Folgende Werte können ausgegeben werden: YES (Ja) oder NO (Nein).
  • locale=<Sprache>: Code der Sprache, in der die Beschreibungen gespeichert wurden.
  • autoAssignUser=<Ja oder Nein>: Gibt an, ob der Benutzer, der den Service abonniert, automatisch dem Abonnement zugeordnet wird. Folgende Werte können ausgegeben werden: YES (Ja) oder NO (Nein).

Aktion: Define up/downgrade options

Beschreibung: Ein Service-Manager definierte oder änderte die Services, auf die ein Abonnement umgestellt werden kann.

Weitere Name/Wert-Paare:

  • targetId=<Service-ID>: ID des marktfähigen Service, auf den Abonnements umgestellt werden können.
  • targetName=<Servicename>: Name des marktfähigen Service, auf den Abonnements umgestellt werden können
  • upDownGrade=<Ja oder Nein>: Gibt an, ob der Service als Up-/Downgrade-Option hinzugefügt oder entfernt wurde. Folgende Werte können ausgegeben werden: ON (als Option verfügbar) oder OFF (nicht als Option verfügbar).

Aktion: Delete customer price model

Beschreibung: Ein Service-Manager löschte ein Kunden-Preismodell.

Weitere Name/Wert-Paare:

  • customerKey=<Organisations-ID>: ID der Kundenorganisation.
  • customerName=<Organisationsname>: Name der Kundenorganisation.

Aktion: Delete service

Beschreibung: Ein Service-Manager löschte einen marktfähigen Service.

Keine weiteren Name/Wert-Paare.

Aktion: Edit event price in customer price model

Beschreibung: Ein Service-Manager definierte oder änderte den Preis für ein Ereignis in einem Kunden-Preismodell.

Weitere Name/Wert-Paare:

  • customerKey=<Organisations-ID>: ID der Kundenorganisation.
  • customerName=<Organisationsname>: Name der Kundenorganisation.
  • currency=<Währung>: ISO-Code der Währung, in der dem Kunden die Nutzungsgebühren in Rechnung gestellt werden.
  • eventName=<Ereignisname>: Name des Ereignisses, wie er im zugrunde liegenden technischen Service definiert ist.
  • range=<Obergrenze>: Bei gestaffelten Preisen ist dies die Obergrenze, bis zu der der Preis gilt. Falls keine gestaffelten Preise definiert sind, ist die Obergrenze 1-ANY.
  • action=<action>: Bei gestaffelten Preisen ist dies die Aktion, die durchgeführt wurde. Folgende Werte können ausgegeben werden: INSERT (ein gestaffelter Preis wurde definiert), UPDATE (ein gestaffelter Preis wurde geändert) oder DELETE (ein gestaffelter Preis wurde gelöscht). Falls keine gestaffelten Preise definiert sind, wird der Parameter action nicht im Aktionsprotokoll ausgegeben.
  • price=<Preis>: Der im Preismodell definierte Preis.

Beschreibung: Ein Service-Manager definierte oder änderte die Währung, Zeiteinheit, den Abrechnungsmodus und/oder die kostenlose Nutzungsdauer in einem Kunden-Preismodell.
