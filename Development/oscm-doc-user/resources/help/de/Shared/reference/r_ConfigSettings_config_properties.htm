Konfigurationseinstellungen

Die Datei configsettings.properties im Verzeichnis <install_pack_dir>/databases/bes_db enthält alle für die Plattform-Services erforderlichen Konfigurationseinstellungen.

AUDIT_LOG_ENABLED

Optional. Legt fest, ob Benutzeraktionen protokolliert und in der Datenbank gespeichert werden. Es können Aktionen protokolliert werden, die sich auf Abonnements, marktfähige Services und Preismodelle beziehen. Ist die Konfigurationseinstellung auf true gesetzt, kann der Betreiber die Protokolldaten exportieren und so Informationen zu Benutzeraktionen abrufen.

Erlaubte Werte: true, false

Voreinstellung: false

AUDIT_LOG_MAX_ENTRIES_RETRIEVED

Optional. Legt fest, wieviele Protokolleinträge bein einem Export der Protokolldaten abgerufen werden. Wird diese Anzahl überschritten, wird eine Warnung angezeigt und der Betreiber aufgefordert, seine Filterkriterien zu ändern und danach den Export nochmal zu starten. Diese Einstellung ist erforderlich, um die Anzahl von SQL-Anfragen an die Datenbank zu begrenzen. Zu viele Datenbankanfragen können die Leistung des Systems beeinträchtigen.

Erlaubte Werte: Beliebiger Wert zwischen 1 und 1000

Voreinstellung: 100

AUTH_MODE

Obligatorisch. Legt fest, ob die Plattform zur Benutzerauthentifizierung verwendet wird oder ob die Plattform als SAML Service Provider verwendet wird. Diese Einstellung wird beim ersten Starten des Systems ausgewertet. Sie kann nach dem ersten Starten des Systems nicht mehr geändert werden. Die Einstellung läßt sich auch nicht durch eine Upgrade-Installation ändern.

Erlaubte Werte: INTERNAL für Plattform-Authentifizierung oder SAML_SP, wenn die Plattform als SAML Service Provider verwendet werden soll.

Voreinstellung: INTERNAL

BASE_URL

Optional. Diese URL führt zur Startseite der Plattform, falls die Kommunikation nicht über HTTPS erfolgt. Bleibt das Feld leer, wird der Wert von BASE_URL_HTTPS verwendet.

Syntax: http://<host>:<port>/<context-root>

Die Voreinstellung für <context-root> ist oscm-portal.

BASE_URL_HTTPS

Obligatorisch. Diese URL führt zur Startseite der Plattform und dient zum Erstellen der URL, um auf Services über HTTPS zuzugreifen.

Syntax: https://<host>:<port>/<context-root>

Die Voreinstellung für <context-root> ist oscm-portal.

CUSTOMER_SELF_REGISTRATION_ENABLED

Optional. Legt fest, ob sich Kunden auf einem Marktplatz selbst registrieren können. Ist der Wert auf false gesetzt, muss entweder der Betreiber eine Organisation für den Kunden anlegen oder ein Anbieter (Serviceanbieter, Vermittler, Wiederverkäufer) muss den Kunden registrieren.

Erlaubte Werte: true, false

Voreinstellung: true

DECIMAL_PLACES

Optional. Legt die Anzahl der Nachkommastellen fest, die für die Berechnung der Nutzungsgebühren verwendet werden. Diese Einstellung wird nur ausgewertet, wenn Sie von Version 14.1 der Plattform auf eine höhere Version umstellen.

Erlaubte Werte: 2, 3, 4, 5, 6

Voreinstellung: 2

HIDDEN_UI_ELEMENTS

Optional. Legt fest, welche Elemente der Benutzeroberfläche im Administrationsportal und auf Marktplätzen ausgeblendet werden. Sie können mit dieser Konfigurationseinstellung Elemente im Administrationsportal und gleichzeitig Elemente auf Marktplätzen ausblenden.

Marktplätze

Geben Sie einen der folgenden Werte an, um eine Menüoption im Menü Benutzerkonto auf Marktplätzen auszublenden:

  • marketplace.navigation.Profile: Menü Profil
  • marketplace.navigation.Payment: Menü Bezahlung
  • marketplace.navigation.Subscriptions: Menü Abonnements
  • marketplace.navigation.Users: Menü Benutzer & Einheiten
  • marketplace.navigation.Reports: Menü Berichte
  • marketplace.navigation.Processes: Menü Prozesse
  • marketplace.navigation.Operations: Operationen menu

Um mehrere Optionen im Menü Benutzerkonto auszublenden, trennen Sie die Einträge durch Komma.

Administrationsportal

Wenn Sie eine bestimmte Seite vom Administrationsportal ausblenden möchten, müssen Sie zunächst den Wert herausfinden, der zum Ausblenden der Seite angegeben werden muss. Gehen Sie dazu wie folgt vor:

  1. Öffnen Sie die entsprechende Seite der Administrationsportals.
  2. Zeigen Sie die Hilfe zu dieser Seite an.
  3. Sehen Sie sich den Namen der Hilfeseite an. Der Name ist Teil der URL.
  4. Lassen Sie die Dateierweiterung .htm weg und ersetzen Sie den Unterstrich durch einen Punkt.

Beispiel:

Sie möchten die Seite Mehrwertsteuersätze bearbeiten ausblenden. Der Name der Hilfeseite ist organization_manageVats.htm. Dementsprechend heißt die zugehörige Seite des Administrationsportals organization.manageVats. Die Konfigurationseinstellung muss wie folgt angegeben werden:

HIDDEN_UI_ELEMENTS=organization.manageVats

Um mehrere Seiten vom Administrationsportal auszublenden, trennen Sie die Einträge durch Komma.

Nachfolgend sehen Sie weitere Beispiele zum Ausblenden bestimmter Seiten. Die Liste ist nicht vollständig.

  • organization.edit: Seite Profile bearbeiten
  • shop.editSkin: Seite Layout anpassen
  • techService.edit: Seite Servicedefinition ändern

Geben Sie einen der folgenden Werte an, um ein ganzes Menü vom Administrationsportal auszublenden:

  • navigation.myAccount: Menü Benutzerkonto
  • navigation.customer: Menü Kunde
  • navigation.operator: Menü Betrieb
  • navigation.techService: Menü Technischer Service
  • navigation.service: Menü Marktfähiger Service
  • navigation.priceModel: Menü Preismodell
  • navigation.marketplace: Menü Marktplatz
Hinweis: Die Seite Konfiguration aktualisieren im Menü Betrieb wird dem Betreiber standardmäßig angezeigt, wenn er sich anmeldet. Wenn Sie die Seite oder das ganze Menü ausblenden, wird Ihnen die Seite trotzdem nach der nächsten Anmeldung angezeigt. Sie können notwendige Änderungen vornehmen.

REPORT_ENGINEURL

Obligatorisch, wenn Sie Berichte verwenden möchten. Definiert das URL-Template der Report Engine. Wenn das Template nicht korrekt angegeben wird, kann die Plattform keine Berichte generieren, da der Report Web-Service nicht korrekt aufgerufen werden kann.
Wenn HTTP-Zugriff für Ihre Installation konfiguriert ist, muss folgender Wert angegeben werden: 
http://<host IP address>:<port>/eclipse-birt-runtime/frameset?__report=$
   {reportname}.rptdesign&SessionId=$
   {sessionid}&__locale=$
   {locale}&WSDLURL=$
   {wsdlurl}&SOAPEndPoint=${soapendpoint}
Wenn HTTPS-Zugriff für Ihre Installation konfiguriert ist, muss folgender Wert angegeben werden: 
https://<host IP address>:<port>/eclipse-birt-runtime/frameset?__report=$
   {reportname}.rptdesign&SessionId=$
   {sessionid}&__locale=$
   {locale}&WSDLURL=$
   {wsdlurl}&SOAPEndPoint=${soapendpoint}&wsname=ReportSecure&wsport=ReportSecurePort
Hinweis: Der obige Wert muss wie angegeben verwendet werden. Nehmen Sie keine Änderung vor.

REPORT_SOAP_ENDPOINT

Obligatorisch, wenn Sie Berichte verwenden möchten. Definiert den SOAP-Endpunkt des Report Web-Service. Alle Daten für einen Bericht werden über einen Aufruf des Report Web-Service zusammengestellt. Wenn Sie keinen korrekten Wert angeben, kann die Plattform keine Berichte generieren, da der Report Web-Service nicht korrekt aufgerufen werden kann.

Wenn HTTP-Zugriff für Ihre Installation konfiguriert ist, muss folgender Wert angegeben werden:

http://<host IP address>:<port>/Report/ReportingServiceBean

Wenn HTTPS-Zugriff für Ihre Installation konfiguriert ist, muss folgender Wert angegeben werden:

https://<host IP address>:<port>/ReportSecure/ReportingServiceSecureBean

Hinweis: Der obige Wert muss wie angegeben verwendet werden. Nehmen Sie keine Änderung vor.

Alle Daten für einen Bericht werden über einen Aufruf des Report Web-Service zusammengestellt. SSO_STS_ENCKEY_LEN

Obligatorisch, wenn die Plattform als SAML Service Provider installiert wird. Gibt die Länge des Encryption Key an, wie er von dem IdP-System (Identity Provider) verwendet wird.

Erforderlich für die STS-Kommunikation.

Voreinstellung: 128

Diese Einstellung wird bei der Installation ausgewertet und kann im Administrationsportal nicht geändert werden. Gehen Sie wie folgt vor, wenn Sie die Einstelllung ändern müssen:

  1. Stoppen Sie die Domäne bes-domain.
  2. Bearbeiten Sie die Datei configsettings.properties und ändern Sie den Wert für die Einstellung SSO_STS_ENCKEY_LEN.
  3. Führen Sie die Datei build-glassfish.xml im Verzeichnis <install_pack_dir>/install wie folgt aus: 
    <ANT_HOME>/bin/ant 
   -DpatchProps.path=../domains/bes_domain/installer 
   -f build-glassfish.xml patchAndRedeployApplication

    Ersetzen Sie <ANT_HOME> durch das Verzeichnis, in dem Apache Ant installiert ist.

    Hinweis: Abhängig vom Betriebssystem müssen Sie den Ausdruck -DpatchProps.path=<Pfad> in doppelte oder einfache Anführungszeichen setzen.
    Der geänderte Wert wird aus der Datei configsettings.properties ausgelesen und in die WSIT-Dateien im Archiv oscm.ear geschrieben. Das aktualisierte Archiv wird in bes-domain neu installiert.

    Während der Ausführung des Befehls wird im Verzeichnis <install_pack_dir>/domains/bes_domain ein Unterverzeichnis /tmp angelegt, in dem das aktualisierte Archiv oscm.ear gespeichert wird.

  4. Installieren Sie die Datei oscm-portal.war in der Domäne bes-domain.
  5. Starten Sie die Domäne bes-domain neu.

TIMER_INTERVAL_BILLING_OFFSET

Optional. Offset-Wert in Millisekunden für den Timer, der die Neuberechnung von Nutzungsgebühren (Kundenrechnungsdaten) und Umsatzbeteiligungen startet. Das Intervall für diesen Timer ist 1 Tag und kann nicht geändert werden. Wenn kein Offset-Wert definiert ist, wird der Standwert von 4 Tagen verwendet.

Die Kundenrechnungsdaten werden für einen Zeitraum von einem Monat berechnet (Abrechnungszeitraum). Serviceanbieter und Wiederverkäufer können festlegen, wann ihr Abrechnungszeitraum anfangen soll. Die Umsatzbeteiligungen werden immer für den vorangegangenen Kalendermonat am ersten Tag des darauffolgenden Monats berechnet.

Der Offset-Wert für diesen Timer definiert folgendes:

  • Anzahl an Tagen, nach denen die Neuberechnung der Nutzungsgebühren und Umsatzbeteiligungen gestartet wird.
  • Uhrzeit, wann die Neuberechnung am aktuellen Tag ausgeführt wird.

Beispiel:

Ein Serviceanbieter legt den 10. eines Monats als Startdatum für den Abrechnungszeitraum fest. Der Offset-Wert ist auf 4 Tage und 4 Stunden festgelegt. Die Neuberechnung der Kundenrechnungsdaten für den abgelaufenen Abrechnungszeitraum startet am 14. des Folgemonats um 04:00:00.000 Uhr. Die Umsatzbeteiligungen werden am 5. des Folgemonats um 04:00:00.000 Uhr neu berechnet. Ob das Ende eines Abrechnungszeitraums eines Serviceanbieters erreicht ist, wird täglich um 04:00:00.000 Uhr geprüft.

Erlaubte Werte: Beliebiger Wert zwischen 0 und 2419200000 (28 Tage)

Voreinstellung: 345600000 (4 Tage)

TIMER_INTERVAL_INACTIVE_ON_BEHALF_USERS

Optional. Intervall in Millisekunden, in dem das System auf nicht mehr existierende Benutzer überprüft wird, die im Namen einer anderen Organisation handelten. Der Wert 0 bedeutet, dass der Timer nicht aktiv ist.

Für einen technischen Service kann definiert sein, dass eine Organisation im Namen einer anderen Organisation handeln darf (allowingOnBehalfActing). Die Organisation muss Kunde der anderen Organisation sein. Diese Organisation muss wiederum sowohl die Rolle Hersteller als auch Serviceanbieter haben. Zusätzlich muss die Kundenorganisation der anderen Organisation die Berechtigung erteilt haben, sich in ihrem Namen am System anzumelden. Dies wird durch das Abonnieren eines Service erreicht, für den beim zugrunde liegenden technischen Service das Merkmal allowingOnBehalfActing auf true gesetzt ist.

Wenn eine Organisation im Namen einer anderen Organisation handelt, wird ein künstlicher Benutzername erzeugt.

Es kann erforderlich sein, die Plattform-Datenbank von Zeit zu Zeit zu bereinigen und Benutzer, die nicht mehr existieren, zu löschen. Es kann nicht sichergestellt werden, dass der technische Service selbst diese Benutzer löscht.

Erlaubte Werte: 0 sowie beliebiger Wert zwischen 10000 (10 Sekunden) und 9223372036854775807

Voreinstellung: 0

WS_TIMEOUT

Obligatorisch. Zeitspanne in Millisekunden, während der gewartet wird, dass Aufrufe eines Web-Service in einem externen System ausgeführt werden. Nach Ablauf dieser Zeitspanne gibt JAX-WS eine Timeout-Meldung aus.

Ein Aufruf eines Web-Service in einem externen System ist ein Aufruf durch die Plattform. Ein typisches Beispiel ist der Aufruf der Methode createUsers im ProvisioningService einer Anwendung. Wird der Timeout erreicht, bevor der Aufruf einen Rückgabewert liefert, wird die Operation abgebrochen und eine Exception ausgegeben.

Erlaubte Werte: Beliebiger Wert zwischen 1 und 9223372036854775807

Voreinstellung: 30000 (30 Sekunden)

Hinweis: Achten Sie darauf, dass die Timeout-Einstellungen im GlassFish Application Server nicht mit den Timeout-Einstellungen der Plattform in Konflikt stehen oder diese überschreiben.
