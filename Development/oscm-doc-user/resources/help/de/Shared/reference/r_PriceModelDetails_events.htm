Preise für Ereignisse

In einem Preismodell kann für jedes abrechenbare Ereignis ein eigener Preis definiert werden. Ein Ereignis ist beispielsweise das An- oder Abmelden in der zugrunde liegenden Anwendung, das Beenden einer bestimmten Transaktion oder das Anlegen oder Löschen bestimmter Daten. Für welche Ereignisse Preise definiert werden können, ist abhängig von der Implementierung und Integration der zugrunde liegenden Anwendung.

Für Ereignisse ist eine Preisstaffelung möglich: Abhängig davon, wie oft ein Ereignis während des Abrechnungszeitraums auftritt, können unterschiedliche Preise definiert werden. Weitere Informationen finden Sie hier.

Der festgelegte Abrechnungsmodus (pro rata oder pro Zeiteinheit) hat keine Auswirkung auf die Kosten für Ereignisse.
