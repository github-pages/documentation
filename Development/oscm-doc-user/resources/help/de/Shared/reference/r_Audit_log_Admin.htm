Aktionen von Administratoren

Im folgenden werden Meldungen erläutert, die in das Aktionsprotokoll geschrieben werden, wenn Administratoren von Organisationen mit einer beliebigen Rolle Aktionen auf einem Marktplatz ausführen. Die meisten der Aktionen können auch von OE-Administratoren und Abo-Managern ausgeführt werden.

Weitere <Name/Wert-Paare> werden abhängig von der Aktion im Anschluss an den Header ausgegeben.

Aktion: Assign service role

Beschreibung: Ein Administrator, OE-Administrator, oder Abo-Manager legte für einen Benutzer, der einem Abonnement zugeordnet ist, eine Servicerolle fest oder änderte die Servicerolle.

Weitere Name/Wert-Paare:

  • serviceId=<Service-ID>: ID des marktfähigen Service, wie sie beim Anlegen angegeben wurde.

  • serviceName=<Name>: Servicename für Kunden des marktfähigen Service, wie er beim Anlegen angegeben wurde.

  • subscriptionName=<Abo-Name>: Name des Abonnements, wie er beim Abonnieren angegeben wurde.

  • user=<Benutzername>: Name des Benutzers, dem die Servicerolle zugewiesen wurde.
  • userRole=<Servicerolle>: Name der Servicerolle, wie er im zugrunde liegenden technischen Service definiert ist.

Aktion: Assign user to unit

Beschreibung: Ein Administrator oder OE-Administrator fügte einer Organisationseinheit einen Benutzer hinzu.

Weitere Name/Wert-Paare:

  • user=<Benutzername>: Name des Benutzers, der der Organisationseinheit hinzugefügt wurde. Wurden mehrere Benutzer hinzugefügt, sind die Listeneinträge durch Kommas getrennt.
  • unit=<Organisationseinheit>: Name der Organisationseinheit, zu der der Benutzer hinzugefügt wurde.

Aktion: Assign user to subscription

Beschreibung: Ein Administrator, OE-Administrator, oder Abo-Manager ordnete einem Abonnement einen Benutzer zu.

Weitere Name/Wert-Paare:

  • serviceId=<Service-ID>: ID des marktfähigen Service, wie sie beim Anlegen angegeben wurde.

  • serviceName=<Name>: Servicename für Kunden des marktfähigen Service, wie er beim Anlegen angegeben wurde.

  • subscriptionName=<Abo-Name>: Name des Abonnements, wie er beim Abonnieren angegeben wurde.

  • user=<Benutzername>: Name des Benutzers, der dem Abonnement zugeordnet wurde.

Aktion: Deassign service role

Beschreibung: Ein Administrator, OE-Administrator, oder Abo-Manager entzog einem Benutzer, der einem Abonnement zugeordnet ist, eine Servicerolle.

Weitere Name/Wert-Paare:

  • serviceId=<Service-ID>: ID des marktfähigen Service, wie sie beim Anlegen angegeben wurde.

  • serviceName=<Name>: Servicename für Kunden des marktfähigen Service, wie er beim Anlegen angegeben wurde.

  • subscriptionName=<Abo-Name>: Name des Abonnements, wie er beim Abonnieren angegeben wurde.

  • user=<Benutzername>: Name des Benutzers, dem die Servicerolle entzogen wurde.
  • userRole=<Servicerolle>: Name der Servicerolle, wie er im im zugrunde liegenden technischen Service definiert ist.

Aktion: Deassign user from subscription

Beschreibung: Ein Administrator, OE-Administrator, oder Abo-Manager hob die Zuordnung eines Benutzers zu einem Abonnement auf.

Weitere Name/Wert-Paare:

  • serviceId=<Service-ID>: ID des marktfähigen Service, wie sie beim Anlegen angegeben wurde.

  • serviceName=<Name>: Servicename für Kunden des marktfähigen Service, wie er beim Anlegen angegeben wurde.

  • subscriptionName=<Abo-Name>: Name des Abonnements, wie er beim Abonnieren angegeben wurde.

  • user=<Benutzername>: Name des Benutzers, der dem Abonnement nicht mehr zugeordnet ist.

Aktion: Disable access to services

Beschreibung: Ein Administrator oder OE-Administrator legte die Services fest, auf die die Mitglieder einer Organisationseinheit keinen Zugriff mehr haben. Für jeden Service wird eine separate Meldung im Aktionsprotokoll ausgegeben.

Weitere Name/Wert-Paare:

  • unit=<Organisationseinheit>: Name der Organisationseinheit, deren Mitglieder auf den zugeordneten Service nicht mehr zugreifen können.
  • marketplaceId=<Marktplatz-ID>: ID des Marktplatzes, auf dem der Service veröffentlicht ist.
  • marketplaceName=<Marktplatzname>: Name des Marktplatzes, auf dem der Service veröffentlicht ist.

  • serviceId=<Service-ID>: ID des marktfähigen Service, wie sie beim Anlegen angegeben wurde.

  • serviceName=<Name>: Servicename für Kunden, wie er beim Anlegen für den marktfähigen Service angegeben wurde.

  • sellerID=<Verkäufer-ID>: ID der Organisation des Serviceanbieters, Vermittlers bzw. Wiederverkäufers, der den Service anbietet.

Aktion: Edit customer attribute by customer

Beschreibung: Ein Administrator, OE-Administrator, oder Abo-Manager definierte oder änderte den Wert eines eigenen Attributs für Kunden. Für jeden Wert wird eine separate Meldung im Aktionsprotokoll ausgegeben.

Weitere Name/Wert-Paare:

  • customerKey=<Organisations-ID>: ID der Kundenorganisation.
  • customerName=<Organisationsname>: Name der Kundenorganisation.
  • attributeName=<Attributname>: Name des eigenen Attributs.
  • attributeValue=<Attributwert>: Wert des eigenen Attributs.

Aktion: Edit subscription attribute by customer

Beschreibung: Ein Administrator, OE-Administrator, oder Abo-Manager definierte oder änderte den Wert eines eigenen Attributs für Abonnements. Für jeden Wert wird eine separate Meldung im Aktionsprotokoll ausgegeben.

Weitere Name/Wert-Paare:

  • serviceId=<Service-ID>: ID des marktfähigen Service, wie sie beim Anlegen angegeben wurde.

  • serviceName=<Name>: Servicename für Kunden des marktfähigen Service, wie er beim Anlegen angegeben wurde.

  • subscriptionName=<Abo-Name>: Name des Abonnements, wie er beim Abonnieren angegeben wurde.

  • attributeName=<Attributname>: Name des eigenen Attributs.
  • attributeValue=<Attributwert>: Wert des eigenen Attributs.

Aktion: Edit subscription owner

Beschreibung: Ein Administrator oder OE-Administrator definierte oder änderte den Eigentümer eines Abonnements.

Weitere Name/Wert-Paare:

  • serviceId=<Service-ID>: ID des marktfähigen Service, wie sie beim Anlegen angegeben wurde.

  • serviceName=<Name>: Servicename für Kunden des marktfähigen Service, wie er beim Anlegen angegeben wurde.

  • subscriptionName=<Abo-Name>: Name des Abonnements, wie er beim Abonnieren angegeben wurde.

  • subscriptionOwner=<Benutzername>: Benutzername des Administrators, OE-Administrators, oder Abo-Managers, der als neuer Eigentümer des Abonnements festgelegt wurde.

Aktion: Edit subscription parameter configuration

Beschreibung: Ein Administrator, OE-Administrator, oder Abo-Manager änderte Parameteroptionen eines Abonnements. Für jede geänderte Option wird eine separate Meldung im Aktionsprotokoll ausgegeben

Weitere Name/Wert-Paare:

  • serviceId=<Service-ID>: ID des marktfähigen Service, wie sie beim Anlegen angegeben wurde.

  • serviceName=<Name>: Servicename für Kunden des marktfähigen Service, wie er beim Anlegen angegeben wurde.

  • subscriptionName=<Abo-Name>: Name des Abonnements, wie er beim Abonnieren angegeben wurde.

  • parameterName=<Name>: Name der Option.
  • parameterValue=<Wert>: Neuer Wert der Option. Bei booleschen Parametern ON oder OFF (ON, wenn die Option ausgewählt wurde, und OFF, wenn die Option nicht ausgewählt wurde), bei anderen Parametern der eingegebene Wert.

Aktion: Enable access to services

Beschreibung: Ein Administrator oder OE-Administrator legte die Services fest, auf die die Mitglieder einer Organisationseinheit Zugriff haben. Für jeden Service wird eine separate Meldung im Aktionsprotokoll ausgegeben.

Weitere Name/Wert-Paare:

  • unit=<Organisationseinheit>: Name der Organisationseinheit, deren Mitglieder auf den zugeordneten Service zugreifen können.
  • marketplaceId=<Marktplatz-ID>: ID des Marktplatzes, über den die Mitglieder der Organisationseinheit auf den Service zugreifen können.
  • marketplaceName=<Marktplatzname>: Name des Marktplatzes, auf dem der Service veröffentlicht ist.

  • serviceId=<Service-ID>: ID des marktfähigen Service, wie sie beim Anlegen angegeben wurde.

  • serviceName=<Name>: Servicename für Kunden, wie er beim Anlegen des marktfähigen Service angegeben wurde.

  • sellerID=<Verkäufer-ID>: ID der Organisation des Serviceanbieters, Vermittlers bzw. Wiederverkäufers, der den Service anbietet.

Aktion: Remove user from unit

Beschreibung: Ein Administrator oder OE-Administrator entfernte einen Benutzer aus einer Organisationseinheit.

Weitere Name/Wert-Paare:

  • user=<Benutzername>: Name des Benutzers, der aus der Organisationseinheit enfernt wurde. Wurden mehrere Benutzer entfernt, sind die Listeneinträge durch Kommas getrennt.
  • unit=<Organisationseinheit>: Name der Organisationseinheit, aus der der Benutzer entfernt wurde.

Aktion: Report issue

Beschreibung: Ein Administrator, OE-Administrator, oder Abo-Manager meldete dem zuständigen Serviceanbieter oder Wiederverkäufer ein Anliegen zu einem Abonnement.

Weiteres Name/Wert-Paar:

  • serviceId=<Service-ID>: ID des marktfähigen Service, wie sie beim Anlegen angegeben wurde.

  • serviceName=<Name>: Servicename für Kunden des marktfähigen Service, wie er beim Anlegen angegeben wurde.

  • subscriptionName=<Abo-Name>: Name des Abonnements, wie er beim Abonnieren angegeben wurde.

  • issue=<Betreff>: Betreff des Anliegens, wie er vom Administrator, OE-Administrator, oder Abo-Manager eingegeben wurde.

Aktion: Set subscription billing address

Beschreibung: Ein Administrator, OE-Administrator, oder Abo-Manager definierte oder änderte die Rechnungsanschrift für ein Abonnement.

Weitere Name/Wert-Paare:

  • serviceId=<Service-ID>: ID des marktfähigen Service, wie sie beim Anlegen angegeben wurde.

  • serviceName=<Name>: Servicename für Kunden des marktfähigen Service, wie er beim Anlegen angegeben wurde.

  • subscriptionName=<Abo-Name>: Name des Abonnements, wie er beim Abonnieren angegeben wurde.

  • address=<Name>: Name der Rechnungsanschrift, wie sie Benutzern zur Auswahl angezeigt wird.
  • addressDetails=<Anschrift>: Name der Organisation, E-Mail-Adresse und Postanschrift, an die Rechnungen geschickt werden sollen.

Aktion: Set subscription payment type

Beschreibung: Ein Administrator, OE-Administrator, oder Abo-Manager definierte oder änderte die Zahlungsart für ein Abonnement.

Weitere Name/Wert-Paare:

  • serviceId=<Service-ID>: ID des marktfähigen Service, wie sie beim Anlegen angegeben wurde.

  • serviceName=<Name>: Servicename für Kunden des marktfähigen Service, wie er beim Anlegen angegeben wurde.

  • subscriptionName=<Abo-Name>: Name des Abonnements, wie er beim Abonnieren angegeben wurde.

  • paymentName=<Name>: Name der Zahlungsart, wie sie Benutzern zur Auswahl angezeigt wird.
  • paymentType=<Zahlungsart>: Name der Zahlungsart, wie sie von einem Serviceanbieter zur Verfügung gestellt wird.

Aktion: Subscribe to service

Beschreibung: Ein Administrator, OE-Administrator, oder Abo-Manager abonnierte einen marktfähigen Service, der auf einem Marktplatz angeboten wird.

Weitere Name/Wert-Paare:

  • serviceId=<Service-ID>: ID des marktfähigen Service, wie sie beim Anlegen angegeben wurde.

  • serviceName=<Name>: Servicename für Kunden des marktfähigen Service, wie er beim Anlegen angegeben wurde.

  • subscriptionName=<Abo-Name>: Name des Abonnements, wie er beim Abonnieren angegeben wurde.

Aktion: Unsubscribe from service

Beschreibung: Ein Administrator, OE-Administrator, oder Abo-Manager beendete ein Abonnement.

Weiteren Name/Wert-Paare:

  • serviceId=<Service-ID>: ID des marktfähigen Service, wie sie beim Anlegen angegeben wurde.

  • serviceName=<Name>: Servicename für Kunden des marktfähigen Service, wie er beim Anlegen angegeben wurde.

  • subscriptionName=<Abo-Name>: Name des Abonnements, wie er beim Abonnieren angegeben wurde.

Aktion: Up/downgrade subscription

Beschreibung: Ein Administrator, OE-Administrator, oder Abo-Manager stellte ein Abonnement auf einen anderen Service um.

Weitere Name/Wert-Paare:

  • serviceId=<Service-ID>: ID des marktfähigen Service, wie sie beim Anlegen angegeben wurde.

  • serviceName=<Name>: Servicename für Kunden des marktfähigen Service, wie er beim Anlegen angegeben wurde.

  • subscriptionName=<Abo-Name>: Name des Abonnements, wie er beim Abonnieren angegeben wurde.

  • newServiceId=<Service-ID>: ID des marktfähigen Service, auf den das Abonnement umgestellt wurde.
  • newServiceName=<Servicename>: Servicename für Kunden des Service, auf den das Abonnement umgestellt wurde.
