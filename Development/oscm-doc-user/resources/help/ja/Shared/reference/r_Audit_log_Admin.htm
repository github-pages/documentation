管理者の操作

サービスポータルの任意のロールを持った管理者が行った操作によって出力される監査ログについて説明します。これらの操作のほとんどは、サービス購入管理者によっても行うことができます。

記録される操作によっては、<追加の名前と値の組>が追加される場合があります。

操作: Assign service role

説明: 管理者またはサービス購入管理者が購入済サービスの利用者に割り当てられたロールを設定または変更した。

追加の名前と値の組:

  • serviceId=<サービスID>:サービスの作成時に指定したサービスのID。

  • serviceName=<サービス名>: サービスの作成時に指定した利用者向けサービス名。

  • subscriptionName=<購入済サービスの名前>: 購入時に指定した購入済サービスの名前。

  • user=<利用者ID>: ロールを割り当てられた利用者のID。
  • userRole=<ロール>: 技術サービスにて定義したロールの名前。

操作: Assign user to group

説明: 管理者が利用者を利用者グループに追加した。

追加の名前と値の組:

  • user=<利用者ID>: カンマで区切られた、利用者グループに割り当てられた利用者のIDの一覧。
  • group=<利用者グループ名>: 利用者が追加された利用者グループの名前。

操作: Assign user to subscription

説明: 管理者またはサービス購入管理者が利用者を購入済サービスへ割り当てた。

追加の名前と値の組:

  • serviceId=<サービスID>: サービスの作成時に指定したサービスのID。

  • serviceName=<サービス名>: サービスの作成時に指定した利用者向けサービス名。

  • subscriptionName=<購入済サービスの名前>: 購入時に指定した購入済サービスの名前。

  • user=<利用者ID>: 購入済サービスに割り当てた利用者のID。

操作: Deassign service role

説明: 管理者またはサービス購入管理者が購入済サービスに割り当てられた利用者のロールを削除した。

追加の名前と値の組:

  • serviceId=<サービスID>: サービスの作成時に指定したサービスのID。

  • serviceName=<サービス名>: サービスの作成時に指定した利用者向けサービス名。

  • subscriptionName=<購入済サービスの名前>: 購入時に指定した購入済サービスの名前。

  • user=<利用者ID>: ロールを削除した利用者のID。
  • userRole=<ロール>: 技術サービスにて定義したロールの名前。

操作: Deassign user from subscription

説明: 管理者またはサービス購入管理者が購入済サービスへの利用者割り当てを解除した。

追加の名前と値の組:

  • serviceId=<サービスID>: サービスの作成時に指定したサービスのID。

  • serviceName=<サービス名>: サービスの作成時に指定した利用者向けサービス名。

  • subscriptionName=<購入済サービスの名前>: 購入時に指定した購入済サービスの名前。

  • user=<利用者ID>: 購入済サービスから割り当てを解除された利用者のID。

説明: Disable access to services

説明: 管理者が利用者グループのメンバーがアクセスできなくなるサービスを指定した。サービスごとにログエントリーが書かれます。

追加の名前と値の組:

  • group=<利用者グループ名>: サービスへアクセスできないメンバーが所属する利用者グループ名。
  • marketplaceId=<サービスポータルのID>: 利用者グループのメンバーがアクセスすることができたサービスがあるサービスポータルのID。
  • marketplaceName=<サービスポータルの名前>: 利用者グループのメンバーがアクセスすることができたサービスがあるサービスポータルの名前。
  • serviceID=<サービスID>: サービスの作成時に指定したサービスのID。
  • serviceName=<サービス名>: サービスの作成時に指定した利用者向けサービス名。
  • sellerID=<サービス提供関連部門のID>: サービスを提供するサービス提供部門、サービス仲介部門、またはサービス提供代行部門の組織ID。

操作: Edit customer attribute by customer

説明: 管理者またはサービス購入管理者がサービス利用部門のカスタム属性の値を定義または変更した。各属性について別々のログエントリーが記録されます。

追加の名前と値の組:

  • customerKey=<組織ID>: サービス利用部門の組織のID。
  • customerName=<組織名>: サービス利用部門の組織の名前。
  • attributeName=<属性の名前>: サービス利用部門のカスタム属性の名前。
  • attributeValue=<属性の値>: サービス利用部門のカスタム属性の値。

操作: Edit subscription attribute by customer

説明: 管理者またはサービス購入管理者がカスタム属性の値を定義または変更した。各属性について別々のログエントリーが記録されます。

追加の名前と値の組:

  • serviceId=<サービスID>: サービスの作成時に指定したサービスのID。

  • serviceName=<サービス名>: サービスの作成時に指定した利用者向けサービス名。

  • subscriptionName=<購入済サービスの名前>: 購入時に指定した購入済サービスの名前。

  • attributeName=<属性の名前>: カスタム属性の名前。
  • attributeValue=<属性の値>: カスタム属性の値。

操作: Edit subscription owner

説明: 管理者がサービス購入管理者を設定または変更した。

追加の名前と値の組:

  • serviceId=<サービスID>: サービスの作成時に指定したサービスのID。

  • serviceName=<サービス名>: サービスの作成時に指定した利用者向けサービス名。

  • subscriptionName=<購入済サービスの名前>: 購入時に指定した購入済サービスの名前。

  • subscriptionOwner=<利用者ID>: 購入済サービスの新しい管理者として設定された管理者またはサービス購入管理者のID。

操作: Edit subscription parameter configuration

説明: 管理者またはサービス購入管理者が購入済サービスのパラメータのオプションを変更した。すべてのパラメーターのオプションについて、それぞれログのエントリが書き込まれます。

追加の名前と値の組:

  • serviceId=<サービスID>: サービスの作成時に指定したサービスのID。

  • serviceName=<サービス名>: サービスの作成時に指定した利用者向けサービス名。

  • subscriptionName=<購入済サービスの名前>: 購入時に指定した購入済サービスの名前。

  • parameterName=<オプション名>: パラメーターのオプションの名前。
  • parameterValue=<オプションの値>: オプションの新しい値。真偽値のパラメーターのオプションは、ON または OFF (ON: オプションが選択されている場合、 OFF: オプションが選択されていない場合)。 列挙型の場合、パラメーターのオプションに設定された値。

操作: Enable access to services

説明: 管理者が利用者グループのメンバーがアクセスできるサービスを指定した。サービスごとにログエントリーが書かれます。

追加の名前と値の組:

  • group=<利用者グループ名>: メンバーがそのサービスにアクセスできる利用者グループの名前。
  • marketplaceId=<サービスポータルのID>: 利用者グループのメンバーがアクセスすることができるサービスがあるサービスポータルのID。
  • marketplaceName=<サービスポータルの名前>: 利用者グループのメンバーがアクセスすることができるサービスがあるサービスポータルの名前。
  • serviceID=<サービスID>: サービスの作成時に指定したサービスのID。
  • serviceName=<サービス名>: サービスの作成時に指定した利用者向けサービス名。
  • sellerID=<サービス提供関連部門のID>: サービスを提供するサービス提供部門、サービス仲介部門、またはサービス提供代行部門の組織ID。

操作: Remove user from group

説明: 管理者が利用者を利用者グループから削除した。

追加の名前と値の組:

  • user=<利用者ID>: カンマで区切られた、利用者グループから削除された利用者のIDの一覧。
  • group=<利用者グループ名>: 利用者が削除された利用者グループの名前。

操作: Report issue

説明: 管理者またはサービス購入管理者は、購入済サービスについて、サービス提供部門やサービス提供代行部門に問い合わせをしました。

追加の名前と値の組:

  • serviceId=<サービスID>: サービスの作成時に指定したサービスのID。

  • serviceName=<サービス名>: サービスの作成時に指定した利用者向けサービス名。

  • subscriptionName=<購入済サービスの名前>: 購入時に指定した購入済サービスの名前。

  • issue=<テキスト>: 管理者またはサービス購入管理者が入力したお問い合わせの件名。

操作: Set subscription billing address

説明: 管理者またはサービス購入管理者が、購入済サービスの請求先を設定または変更した。

追加の名前と値の組:

  • serviceId=<サービスID>: サービスの作成時に指定したサービスのID。

  • serviceName=<サービス名>: サービスの作成時に指定した利用者向けサービス名。

  • subscriptionName=<購入済サービスの名前>: 購入時に指定した購入済サービスの名前。

  • address=<表示名>: サービス利用者に表示される請求先の名前。
  • addressDetails=<請求先詳細>: 請求書が送付される組織の名前、メールアドレス、住所。

操作: Set subscription payment type

説明: 管理者またはサービス購入管理者が、購入済サービスの支払種別を設定または変更した。

追加の名前と値の組:

  • serviceId=<サービスID>: サービスの作成時に指定したサービスのID。

  • serviceName=<サービス名>: サービスの作成時に指定した利用者向けサービス名。

  • subscriptionName=<購入済サービスの名前>: 購入時に指定した購入済サービスの名前。

  • paymentName=<表示名>: サービス利用者に表示する支払種別の名前。
  • paymentType=<支払種別>: サービス提供部門または支払代行システムによって示された支払種別の名前。

操作: Subscribe to service

説明: 管理者またはサービス購入管理者が、サービスポータルで公開されているサービスの購入を完了した。

追加の名前と値の組:

  • serviceId=<サービスID>:サービスの作成時に指定したサービスのID。

  • serviceName=<サービス名>: サービスの作成時に指定した利用者向けサービス名。

  • subscriptionName=<購入済サービスの名前>: 購入時に指定した購入済サービスの名前。

操作: Unsubscribe from service

説明: 管理者またはサービス購入管理者が購入済サービスを解約した。

追加の名前と値の組:

  • serviceId=<サービスID>:サービスの作成時に指定したサービスのID。

  • serviceName=<サービス名>: サービスの作成時に指定した利用者向けサービス名。

  • subscriptionName=<購入済サービスの名前>: 購入時に指定した購入済サービスの名前。

操作: Up/downgrade subscription

説明: 管理者またはサービス購入管理者が購入済サービスのアップグレードまたはダウングレードをおこなった。

追加の名前と値の組:

  • serviceId=<サービスID>:サービスの作成時に指定したサービスのID。

  • serviceName=<サービス名>: サービスの作成時に指定した利用者向けサービス名。

  • subscriptionName=<購入済サービスの名前>: 購入時に指定した購入済サービスの名前。

  • newServiceId=<サービスID>: アップグレードまたはダウングレードしたサービスのID。
  • newServiceName=<サービス名>: アップグレードまたはダウングレードしたサービス部門向けのサービス名。
