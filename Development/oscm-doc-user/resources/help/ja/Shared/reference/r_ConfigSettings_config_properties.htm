システム設定

<install_pack_dir>/databases/bes_dbにあるconfigsettings.propertiesファイルには、プラットフォームサービスのシステム設定が含まれます。

AUDIT_LOG_ENABLED

オプション項目。購入済サービス、サービス、および料金設定に関連した利用者の操作を記録し、データベースに格納するかどうかを指定します。trueに設定した場合、システム運用管理者は監査ログのデータをエクスポートし、利用者の操作に関する情報を取得することができます。

指定可能な値： true, false

デフォルト： false

AUDIT_LOG_MAX_ENTRIES_RETRIEVED

オプション項目。一回の監査ログのエクスポートで、いくつのログのエントリを取得するかを指定します。この値を超えた場合、フィルタ条件を変更して再度エクスポートを実施するように操作者に警告を表示します。この設定は、監査ログをエクスポートする際、データベースへのSQL要求の数を低く抑えるために必要です。要求数が多いと、システムパフォーマンスの低下になります。

指定可能な値： 1から1000の間の任意の値

デフォルト： 100

AUTH_MODE

必須項目。 プラットフォームのユーザー認証を使用するか、それともプラットフォームがSAMLサービス提供者として機能するかを指定します。このシステム設定は、システムを初めて起動した時に評価され、システムが一度でも起動された後には設定を変更することはできません。アップグレードインストールでも設定を変更することはできません。

指定可能な値： INTERNAL (プラットフォームのユーザー認証を使用) または SAML_SP (プラットフォームはSAMLサービス提供者として機能)。

デフォルト： INTERNAL

BASE_URL

オプション項目。プラットフォームにてHTTPS通信をする必要がない場合に、プラットフォームのトップページにアクセスするのにベースURLが使用されます。未入力の場合BASE_URL_HTTPSが使われます。

指定形式： http://<ホスト>:<ポート>/<コンテキストルート>

<コンテキストルート> のデフォルト値は oscm-portal です。

インターネットのドメイン名は以下のルールを満たしている必要があります:

  • 文字で始まり、文字または数字で終わる。

  • 文字、数値またはハイフンのみで構成される。特殊文字は使用不可。

  • 最大63文字以内で構成される。

BASE_URL_HTTPS

必須項目。HTTPSプロトコルでプラットフォームのホームページにアクセスするため、および、サービスにアクセスするURLを作成するために使われます。

指定形式： https://<ホスト>:<ポート>/<コンテキストルート>

<コンテキストルート> のデフォルト値は oscm-portal です。

インターネットのドメイン名は以下のルールを満たしている必要があります:

  • 文字で始まり、文字または数字で終わる。

  • 文字、数値またはハイフンのみで構成される。特殊文字は使用不可。

  • 最大63文字以内で構成される。

CUSTOMER_SELF_REGISTRATION_ENABLED

オプション項目。サービスポータルでサービス利用部門の登録が可能かどうかを設定します。falseが設定されている場合、サービス利用部門はすべてシステム運用管理者による登録が必要です。

設定可能な値： true、false

デフォルト： true

DECIMAL_PLACES

オプション項目。使用料の計算で使用する小数点数を指定します。この設定は、プラットフォームを V14.1から上位のバージョンに移行する場合にだけ必要です。

設定可能な値： 2, 3, 4, 5, 6

デフォルト： 2

HIDDEN_UI_ELEMENTS

オプション項目。プラットフォームの管理ポータルおよびサービスポータルで非表示にするユーザーインターフェース項目を指定します。この設定でユーザーインターフェース要素を管理ポータルやサービスポータルから非表示にすることができます。

サービスポータル

サービスポータルの[アカウント]メニューからオプションを非表示にするには、以下の値のいずれかを入力します。:

  • marketplace.navigation.Profile:[プロフィール]メニュー
  • marketplace.navigation.Payment:[支払情報]メニュー
  • marketplace.navigation.Subscriptions:[購入済サービス]メニュー
  • marketplace.navigation.Users:[利用者]メニュー
  • marketplace.navigation.Reports:[レポート]メニュー
  • marketplace.navigation.Processes:[トリガー・プロセス]メニュー
  • marketplace.navigation.Operations:[操作]メニュー

アカウントメニューから複数のオプションを非表示にするには、オプションをカンマで区切ります。

管理ポータル

特定ページを非表示にするには、以下の方法で非表示にしたいページの設定値を割り出す必要があります。

  1. プラットフォームの管理ポータルで各ページを開きます。
  2. このページのオンラインヘルプを表示します。
  3. オンラインヘルプHTMLページの名前を参照します。
  4. ファイルの拡張子「.htm」を省略し、下線をドットに置き換えます。

例：

[付加価値税率の管理]ページを非表示にするとします。このページに紐付くオンラインヘルプHTMLページ名はorganization_manageVats.htmです。そこから、対応する管理ポータルページの設定値はorganization.manageVatsになります。このページを非表示にするためには、以下のようにHIDDEN_UI_ELEMENTSキーを編集する必要があります。

HIDDEN_UI_ELEMENTS=organization.manageVats

管理ポータルから複数ページを非表示にするには、エントリをコンマで区切ります。

以下に、特定ページを非表示にするために使用できる値の例を示します。この一覧はすべてを網羅しているわけではありません。

  • organization.edit：[プロフィールの編集]ページ
  • shop.editSkin：[レイアウトのカスタマイズ]ページ
  • techService.edit：[技術サービス説明の編集]ページ

メニューを管理ポータルからカテゴリごと非表示にするには、以下の値のいずれかを入力します。

  • navigation.myAccount：[アカウント]メニュー
  • navigation.customer：[サービス利用部門]メニュー
  • navigation.operator：[操作]メニュー
  • navigation.techService：[技術サービス]メニュー
  • navigation.service：[サービス]メニュー
  • navigation.priceModel：[料金設定]メニュー
  • navigation.marketplace：[サービスポータル]メニュー
注: 操作メニューのシステム設定の変更ページは、システム運用管理者がログインしたときに転送されるデフォルトページです。このページをメニューから非表示にしたり、メニューそのものを非表示とした場合でも、システム設定の変更ページに転送され、必要であれば設定を変更することができます。

HTTP_PROXY

オプション項目。プロキシサーバが存在する場合にそのプロキシサーバを指定します。

例： proxy.domainまたはプロキシサーバIPアドレス。

HTTP_PROXY_PORT

オプション項目。プロキシサーバが存在する場合にそのプロキシサーバに接続するポート番号を指定します。

設定可能な値： 1～65535の範囲の値を指定します。

デフォルト： 1080

IDP_ASSERTION_EXPIRATION

技術サービスのアクセス種別を ユーザアクセス と定義し、かつ、本プラットフォームをSAML 1.1 アイデンティティプロバイダー(IdP)として使用している場合は必須。アクセス種別の詳細については、運用ガイド（技術サービス開発部門編）を参照してください。

この設定は、SAML IdPによって生成されるアサーションが有効なミリ秒数を示します。これは、アサーションのNotAfterOrOn属性とIssueInstant属性の差異です。

設定可能な値： 1～9223372036854775807の範囲の値を指定します。

デフォルト： 1800000ミリ秒（30分）

IDP_ASSERTION_VALIDITY_TOLERANCE

技術サービスのアクセス種別を ユーザアクセス と定義し、かつ、本プラットフォームをSAML 1.1 IdPとして使用している場合は必須。アクセス種別の詳細については、運用ガイド（技術サービス開発部門編）を参照してください。

この設定は、SAML IdPによって生成されるアサーションが過去に有効であったミリ秒数を示します。これはサーバ間の同期問題を回避するために使用されます。この値はアサーションのIssueInstant属性とNotBefore属性の差異です。

設定可能な値： 1～9223372036854775807の範囲の値を指定します。

デフォルト： 600000ミリ秒（10分）

IDP_PRIVATE_KEY_FILE_PATH

技術サービスのアクセス種別を ユーザアクセス と定義し、かつ、本プラットフォームをSAML 1.1 IdPとして使用している場合は必須。アクセス種別の詳細については、運用ガイド（技術サービス開発部門編）を参照してください。

この設定は、DER（Distinguished Encoding Rules）形式（.derファイル）の秘密キーのURLを示します。このURLはSAMLメッセージを署名するためにIdPによって使用されます。

例： /opt/glassfish-3.1.2.2/glassfish/domains/bes-domain/config/cakey.der

この設定を変更した場合、プラットフォームを再起動する必要があります。

IDP_PUBLIC_CERTIFICATE_FILE_PATH

技術サービスのアクセス種別を ユーザアクセス と定義し、かつ、本プラットフォームをSAML 1.0 IdPとして使用している場合は必須。アクセス種別の詳細については、運用ガイド（技術サービス開発部門編）を参照してください。

この設定は、DER（Distinguished Encoding Rules）形式（.derファイル）の公開キー証明書のURLを示します。アプリケーションが署名を確認できるように、このURLがIdPによってSAMLメッセージの署名に追加されます。

例： /opt/glassfish-3.1.2.2/glassfish/domains/bes-domain/config/cacert.der

この設定を変更した場合、プラットフォームを再起動する必要があります。

LDAP_SEARCH_LIMIT

オプション項目。組織が利用者の管理に外部LDAPシステムを使用する場合に、LDAPクエリが返すことができるエントリの最大数。

設定可能な値： 1～9223372036854775807の範囲の値を指定します。

デフォルト： 100

LOG_CONFIG_FILE

オプション項目。プラットフォームのlog4j設定ファイルのパス。

デフォルト： <GLASSFISH_HOME>/glassfish/domains/bes-domain/config ディレクトリの ./log4j.properties ファイル。

LOG_FILE_PATH

必須項目。プラットフォームのログファイルのパス。

デフォルト： ../logs 、すなわち <GLASSFISH_HOME>/glassfish/domains/bes-domain/logs ディレクトリ。

この設定を変更した場合、プラットフォームを再起動する必要があります。

LOG_LEVEL

オプション項目。プラットフォームのBSSのログレベル。この設定はlog4j.propertiesファイルの設定によって明確に指定されない場合、すべてのログクラスに適用されます。

設定可能な値： ERROR、WARN、INFO、DEBUG

デフォルト： INFO

この設定を変更した場合、プラットフォームを再起動する必要があります。

MAIL_JA_CHARSET

オプション項目。日本語で送信されるメールの特殊文字エンコーディング。

デフォルト： UTF-8

MAX_NUMBER_ALLOWED_USERS

必須項目。インストレーションで登録できる最大利用者数。

設定可能な値： 1～9223372036854775807の範囲の値を指定します。

デフォルト： 10

MAX_NUMBER_LOGIN_ATTEMPTS

オプション項目。プラットフォームにログインできる最大試行回数。利用者がこの回数でログインできなかった場合、そのアカウントがロックされます。

設定可能な値： 1～9223372036854775807の範囲の値を指定します。

デフォルト： 3

MP_ERROR_REDIRECT_HTTP

オプション項目。有効でないサービスポータルIDを指定してサービスポータルにHTTPでアクセスしようとした場合に表示するWebページのURLです。このWebページがデフォルトのエラーメッセージの代わりに表示されます。

指定形式： http://<表示するWebページ>

255文字以下の有効なURLを指定します。

MP_ERROR_REDIRECT_HTTPS

オプション項目。有効でないサービスポータルIDを指定してサービスポータルにHTTPSでアクセスしようとした場合に表示するWebページのURLです。このWebページがデフォルトのエラーメッセージの代わりに表示されます。

指定形式： https://<表示するWebページ>

255文字以下の有効なURLを指定します。

PERMITTED_PERIOD_INACTIVE_ON_BEHALF_USERS

オプション項目。サービス利用部門の代わりにログインした利用者が利用されなかった場合に、システムから削除されるまでのミリ秒の時間。値はミリ秒単位です。

設定可能な値： 1～9223372036854775807の範囲の値を指定します。

デフォルト： 604800000（7日）

PERMITTED_PERIOD_UNCONFIRMED_ORGANIZATIONS

オプション項目。組織の最初の管理アカウントを承認するまでの最大許容時間。この時間が経過すると、アカウントが削除されます。値はミリ秒単位です。

設定可能な値： 1～9223372036854775807の範囲の値を指定します。

デフォルト： 604800000（7日）

PSP_USAGE_ENABLED

必須項目。現在の環境にPSP統合を使用するか指定します。

設定可能な値： true, false

デフォルト： false

この設定を変更した場合、プラットフォームを再起動する必要があります。

REPORT_ENGINEURL

プラットフォームのレポートを使用する場合には必須。レポートエンジンのURLテンプレート。正しいURLテンプレートを指定しないと、Report Webサービスを正しく呼び出すことができないため、プラットフォームはレポートを生成できません。
HTTPアクセスを使うように設定している場合の指定形式： 
http://<ホストのIPアドレス>:<ポート>/birt/frameset?
   __report=${reportname}.rptdesign&SessionId=${sessionid}
   &__locale=${locale}&WSDLURL=${wsdlurl}&SOAPEndPoint=${soapendpoint}
   &wsname=Report&wsport=ReportPort
HTTPSアクセスを使うように設定している場合の指定形式: 
https://<ホストのIPアドレス>:<ポート>/birt/frameset?
   __report=${reportname}.rptdesign&SessionId=${sessionid}
   &__locale=${locale}&WSDLURL=${wsdlurl}&SOAPEndPoint=${soapendpoint}
   &wsname=ReportSecure&wsport=ReportSecurePort
注: 上記の値は示されたとおりに使用する必要があります。この値を変更しないでください。

REPORT_SOAP_ENDPOINT

プラットフォームのレポートを使用する場合には必須。Report WebサービスのSOAPエンドポイント。Report Webサービスへの呼び出しによってすべてのレポートデータが取得されます。正しい値を指定しないと、Report Webサービスを正しく呼び出すことができないため、プラットフォームはレポートを生成できません。

HTTPアクセスを使うように設定している場合の指定形式：

http://<ホストのIPアドレス>:<ポート>/Report/ReportingServiceBean

HTTPSアクセスを使うように設定している場合の指定形式:

https://<ホストのIPアドレス>:<port>/ReportSecure/ReportingServiceSecureBean

注: 上記の値は示されたとおりに使用する必要があります。この値を変更しないでください。

REPORT_WSDLURL

プラットフォームのレポートを使用する場合には必須。Report WebサービスのWSDLファイルのURL。Report Webサービスへの呼び出しによってすべてのレポートデータが取得されます。正しい値を指定しないと、Report Webサービスを正しく呼び出すことができないため、プラットフォームはレポートを生成できません。

HTTPアクセスを使うように設定している場合の指定形式：

http://<ホストのIPアドレス>:<ポート>/Report/ReportingServiceBean?wsdl

HTTPSアクセスを使うように設定している場合の指定形式:

https://<ホストのIPアドレス>:<ポート>/ReportSecure/ReportingServiceSecureBean?wsdl

注: 上記の値は示されたとおりに使用する必要があります。この値を変更しないでください。

SEARCH_INDEX_MASTER_FACTORY_NAME

必須項目。検索インデックスアプリケーションのJMS接続ファクトリ名。jms/bss/masterIndexerQueueFactoryを設定し、値を変更しないでください。

SEARCH_INDEX_MASTER_QUEUE_NAME

必須項目。検索インデックスアプリケーションのJMSキュー名。jms/bss/masterIndexerQueueのまま変更しないでください。

SSO_IDP_AUTHENTICATION_REQUEST_HTTP_METHOD

プラットフォームをSAML_SP認証モードでインストールする場合は必須。 HTTP認証要求で利用する方式。IdPに依存して、GET または POST のいずれかのリクエストを使用できます。

WebブラウザのSSOのために必要です。

デフォルト： POST

この設定を変更した場合、プラットフォームを再起動する必要があります。

SSO_IDP_TRUSTSTORE

プラットフォームをSAML_SP認証モードでインストールする場合は必須。IdPの公開鍵証明書を保持しているトラストストアのファイルへのパスとファイル名。

WebブラウザのSSOのために必要です。

デフォルト： <path>/cacerts.jks

ほかのIdPシステム、例えばOpenAM、Cloudminder、またはActive Directoryはインストールされ、利用される次第、この設定を変更する必要があるかもしれません。この設定を変更した場合、プラットフォームを再起動する必要があります。

SSO_IDP_TRUSTSTORE_PASSWORD

プラットフォームをSAML_SP認証モードでインストールする場合は必須。IdPの公開鍵証明書を保持しているトラストストアのパスワード。

WebブラウザのSSOのために必要です。

GlassFishのデフォルトパスワード： changeit

パスワードは大文字小文字を区別します。またIdPの出力と一致している必要があります。

この設定を変更した場合、プラットフォームを再起動する必要があります。

SSO_IDP_URL

プラットフォームをSAML_SP認証モードでインストールする場合は必須。IdPサービスエンドポイントのためのSAML転送先URL。

WebブラウザのSSOのために必要です。

セキュリティ上および機密性の理由から、HTTPSプロトコルを利用することを推奨します。

指定形式： https://<ホスト>:<ポート>/<転送先サービスエンドポイント>

この設定を変更した場合、プラットフォームを再起動する必要があります。

SSO_ISSUER_ID

プラットフォームをSAML_SP認証モードでインストールする場合は必須。プラットフォームを識別する一意な識別子。 IdPは、プラットフォームからのSAML認証要求を識別するのにこのIDを使います。

WebブラウザのSSOのために必要です。

プラットフォームとIdPの間で情報の交換が一度でもおこなわれた後にこの設定を変更する場合は、プラットフォームとIdPの両方を変更する必要があります。IdPの運用管理者に新しいメタデータ交換ファイルを送付する必要がある場合があります。

デフォルト： CT_MG

この設定を変更した場合、プラットフォームを再起動する必要があります。

SSO_STS_ENCKEY_LEN

プラットフォームをSAML_SP認証モードでインストールする場合は必須。IdPで使われている暗号化キーの長さ。

STS通信のために必要です。

デフォルト： 128

この設定はインストールプロパティであり、管理ポータルからは変更できません。暗号化キーの長さを変更する必要がある場合は、以下の手順を実施します：

  1. bes-domain ドメインを停止します。
  2. ファイル configsettings.properties を編集し、SSO_STS_ENCKEY_LEN の値を必要に応じて変更します。
  3. <install_pack_dir>/install にあるファイル build-glassfish.xml を以下のように実行： 
    <ANT_HOME>/bin/ant -DpatchProps.path=../domains/bes_domain/installer 
   -f build-glassfish.xml patchAndRedeployApplication
    <ANT_HOME>はApache Antのインストールディレクトリです。
    注: オペレーティングシステムによっては、-DpatchProps.path=<path> コマンドをダブルクォーテーションまたはシングルクォーテーションで囲む必要がある場合があります。
    変更した設定のパラメーター値は、ファイル configsettings.properties から読み込まれ、oscm.ear アーカイブに含まれるWSITファイルに書き込まれます。更新されたアーカイブは bes-domain ドメインに再配備されます。

    上記コマンドの実行中にサブディレクトリ /tmp が作成され、更新された oscm.ear アーカイブが保存されます。アーカイブの再配備に失敗した場合、/tmp/oscm.ear アーカイブを手動で再配備することができます。

  4. ファイル oscm-portal.war を bes-domain ドメインに再配備します。
  5. bes-domain ドメインを再起動します。

SSO_STS_METADATA_URL

プラットフォームをSAML_SP認証モードでインストールする場合は必須。STSのMEXアドレス (Issuer Metadata Exchange) のURL。

STS通信のために必要です。

セキュリティ上および機密性の理由から、HTTPSプロトコルを使うことを推奨します。

指定形式： https://<ホスト>:<ポート>/<MEXアドレス>

この設定はインストールプロパティであり、管理ポータルからは変更できません。URLを変更する必要がある場合は、以下の手順を実施します:

  1. bes-domain ドメインを停止します。
  2. ファイル configsettings.properties を編集し、SSO_STS_METADATA_URL の値を必要に応じて変更します。
  3. <install_pack_dir>/oscm-setup にあるファイル build-glassfish.xml を以下のように実行： 
    <ANT_HOME>/bin/ant 
   -DpatchProps.path=../domains/bes_domain/installer 
   -f build-glassfish.xml patchAndRedeployApplication

    <ANT_HOME>はApache Antのインストールディレクトリです。

    変更した設定のパラメーター値は、ファイル configsettings.properties から読み込まれ、oscm.ear アーカイブに含まれるWSITファイルに書き込まれます。更新されたアーカイブは bes-domain ドメインに再配備されます。

    上記コマンドの実行中にサブディレクトリ /tmp が作成され、更新された oscm.ear アーカイブが保存されます。アーカイブの再配備に失敗した場合、/tmp/oscm.ear アーカイブを手動で再配備することができます。

  4. ファイル oscm-portal.war を bes-domain ドメインに再配備します。
  5. bes-domain ドメインを再起動します。

TAGGING_MAX_TAGS

必須項目。タグクラウドを構成するタグの最大数。

タグクラウドは、定義された検索用語（タグ）を含むサービスポータルの領域です。サービスでタグを使う頻度が多いほど、タグの文字が大きく表示されます。サービスポータル運用者によってタグクラウドがサービスポータルで有効になっている場合に、サービス利用部門はタグを使用してサービスを検索することができます。

設定可能な値： 0～2147483647の範囲の値を指定します。

デフォルト： 20

TAGGING_MIN_SCORE

必須項目。タグをタグクラウドで表示させるために必要な最少利用回数。

タグクラウドは、定義された検索用語（タグ）を含むサービスポータルの領域です。サービスでタグを使う頻度が多いほど、タグの文字が大きく表示されます。対応するサービスポータル運用者によってタグクラウドがサービスポータルで有効になっている場合に、サービス利用部門はタグを使用してサービスを検索することができます。

設定可能な値： 1～2147483647の範囲の値を指定します。

デフォルト： 1、タグクラウドで表示するにはタグが最低1回使用される必要があります。

TIME_ZONE_ID

設定可能な値：Javaではすべての時間帯がサポートされます。これはPSTのような略称、America/Los_Angelesのようなフルネーム、GMT-8:00のようなカスタムIDで指定することができます。使用できるIDの一覧については、java.util.TimeZoneを参照してください。

TIMER_INTERVAL_BILLING_OFFSET

オプション項目。購入済サービスの利用料金（サービス利用部門請求データ）または売上配分用請求データを計算する請求データ作成処理のためのタイマーのオフセット。値はミリ秒単位です。このタイマーの間隔は常に１日間で変更できません。オフセットが設定去れていない場合、デフォルトの値である4日間が適用されます。

サービス利用部門請求データは一カ月の期間（請求期間）に対して計算されます。それぞれのサービス提供部門およびサービス提供代行部門は異なる請求期間の開始日を設定できます。売上配分用請求データは毎月1日に過去の月に対して計算されます。

請求データ作成処理のためのオフセットの設定は以下のとおりです。:

  • サービス利用部門請求書を計算する日および売上配分用請求データを計算する日を遅らせる日数。
  • 毎日の請求データ作成処理の開始を遅らせる時間。

例:

サービス提供部門が請求期間の開始日を10日に設定し、オフセットに4日と4時間を設定した場合、そのサービス提供部門に関する請求データ作成処理はそれ以降の月の14日の4時に開始されます。売上配分用請求データは4日の4時に計算されます。毎日の請求期間が終了したかどうかの確認は毎日の4時におこなわれます。

設定可能な値： 0～2419200000(28日間)の範囲の値を指定します。

デフォルト： 345600000(4日間)

オプション項目。すべての組織に対する割引を解除するタイマーのオフセット。請求処理間隔は1日で、変更できません。オフセットはミリ秒単位で、1月1日の00:00を基準とします。

TIMER_INTERVAL_INACTIVE_ON_BEHALF_USERS

オプション項目。別の組織の代理をおこなっている実在しない利用者の確認が実行される時間間隔。値はミリ秒単位です。0は、このタイマーが無効なことを示します。

技術サービス定義には、組織が別の組織名で行動することを示すためのフラグ（allowingOnBehalfActing)を含めることができます。指定された組織は、別の組織のサービス利用部門でなければなりません。これは技術サービス開発部門およびサービス提供部門ロールを両方有している必要があることを意味します。さらに、サービス利用部門は、別の組織が自らログインすることを許可している必要があります。これは、技術サービスがallowingOnBehalfActingフラグをtrueに設定している購入済サービスによって実現できます。

組織が別の組織名で行動する場合、疑似利用者IDが生成されます。

技術サービス自身がこのような利用者を常に削除するようにはできないため、購入済サービスに存在しなくなった利用者を定期的に削除してプラットフォームのデータベースを整理する必要があります。

設定可能な値： 0、及び10000（10秒）～9223372036854775807の範囲の値を指定します。

デフォルト： 0

WS_TIMEOUT

必須項目。外部Webサービス呼び出しのタイムアウト時間をミリ秒で指定します。この時間が経過すると、JAX-WSフレームワークによってタイムアウト例外が発生します。

外部Webサービス呼び出しはプラットフォームから実行される呼び出しです。アプリケーションによって実装されるProvisioningServiceインターフェースのcreateUsersメソッドの呼び出しが典型的な例です。Webサービス呼び出しが戻る前に本タイムアウト時間に達した場合、操作は中止され、例外が発生します。

設定可能な値： 1～9223372036854775807の範囲の値を指定します。

デフォルト： 30000（30秒）

注: リクエストタイムアウトなど、GlassFishアプリケーションサーバで設定されるタイムアウトが、 プラットフォームの構成設定で定義されるタイムアウトと不整合を起こすことがないようにしてください。
