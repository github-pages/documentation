ユーザーインターフェースのリソース

言語の管理ページのエクスポート機能で生成されたMicrosoft Excel形式のファイルを用いてユーザーインターフェースのリソースの翻訳や更新をおこないます。

エクスポートした Microsoft Excel形式のファイルには３つのワークシートがあります。:

  • User interface: 管理ポータルおよびサービスポータルのユーザーインターフェースに表示されるすべてのテキスト、ラベル、およびメッセージ。
  • Email: 本プラットフォームで自動で生成されるすべてのメールの件名および本文。
  • Platform objects: 本プラットフォームが提供するサービスのパラメーターの名前、イベントの名前、レポートのタイトル、および支払種別の名前。 パラメーターとイベントの名前は、サービスを定義したときにサービス提供部門に表示されるだけではなく、サービスを購入する利用者にも表示されます。

各ワークシートには５つの列があります:

  • Key:　ラベルや文字列を識別するために使います。Key を変更しないでください。
  • de system, en system, ja system: インストール時にデフォルトでサポートしている言語（ドイツ語、英語、日本語）の名前とラベル。 これらシステムのデフォルトの文字列は翻訳する際の参照用です。
  • Add your language code here または <言語コード>: この列には、新規にサポートする言語の文字列を入力します。

    Add your language code hereは、言語の管理ページで言語を選択せずにデータをエクスポートした場合に出力されます。 これは通常、新規に言語を追加する場合です。 この列は空になります。

    言語の管理ページでいずれかの言語を選択した場合は、その言語のコードがヘッダーに出力されます。 これは通常、選択した既存の言語の翻訳を更新する場合です。 システムのデフォルト言語(de, en, または jp)をカスタマイズしたいときは、システムのデフォルト言語を選択することもできます。

    サービスポータル運用者がこれら文字列を変更することもできます。この場合は、サービスポータル --> テキストのカスタマイズメニューからおこないます。 変更は、そのサービスポータル運用部門が管理しているサービスポータルに適用されます。

ファイルを翻訳する手順は以下の通りです:

  1. ３つのワークシートすべてについて、５番目の列のヘッダーに提供する言語のISO言語コードを入力します。
  2. 各キーについて、５つ目の列にその翻訳を入力します。

    翻訳が提供されていない場合、英語のラベルや文字列が使われます。 もし英語のラベルも文字列も定義されていない場合、言語無依存のキーが表示されます。

    以下に注意してください:

    • 説明文（.descriptionで終わるキー）のテキストではHTMLマークアップをテキスト中で使えます。 例えば、太字にするには <b> を、改行するには <br> をなどが使えます。
    • Unicode文字セットをすべて使えます。
    • 変更したくない場合は、表のセルからテキストを削除します。 これは、ユーザーインターフェースへの変更を管理するのに有用です。
    • HTMLフィールドには有効なデータが含まれ、ページのレイアウトが崩れていないことを確認してください。 例えば、テキストが長すぎるとユーザーインターフェース上では複数の行に分割されるかもしれません。 翻訳を確認する環境は、別個に用意してください。
  3. ファイルを.xls形式のMicrosoft Excelファイルで保存します。
  4. 言語の管理 ページでファイルをインポートします。

    ファイルをインポートすると、５つ目の列に記入した言語の既存のラベルや文字列は上書きされます。 有効化したら即座にその言語が利用可能になります。 プロフィールの言語の設定に同じ言語を設定した利用者は、 即座にカスタマイズした結果が反映されます。

    注: Excelファイルのすべてのワークシートのすべてのセルはテキストフォーマットであることを確認してください。
