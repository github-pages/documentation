トリガーの編集

トリガーの編集ページでは、所属組織に新しいトリガー定義の追加または、既存のトリガーを変更または削除することができます。

アクションをプロセスで制御するには通知用のサービスが必要です。通知用のサービスはプラットフォームとプロセス制御システムのインターフェースの役割を担います。通知用サービスの実装の詳細については、システム運用管理者にお問い合わせください。

新しいトリガーを追加するには：

  1. 表示内容フィールドで、プロセスの管理ページで利用者が保留プロセスの確認時に表示されるトリガー名を入力します。
  2. 種別一覧からプロセス制御の対象となるアクションを選択します。

    一覧には、トリガーを設定できるすべてのアクションが含まれます。

    いずれの組織に対しても設定可能なトリガーは以下のとおりです:

    • 購入済サービス名の変更などの購入済サービスを変更する処理 (購入済サービスの編集).
    • 請求書作成を完了し、請求期間に対する請求データを計算する処理 (請求書作成の完了).
    • 購入済サービスを追加する処理 (サービスの購入).
    • 購入済サービスに利用者を割り当てまたは除去する処理 (購入済サービスへの利用者の割り当て).

      サービスを購入する際、新たに購入したサービスへの利用者の自動割り当てに対しては、トリガーを設定することはできません。 購入済サービスへの利用者の割り当てのトリガーは購入済みサービスへ利用者を手作業で割り当てた場合にだけ有効です。

    • 購入済サービスをアップグレードまたはダウングレードする処理(購入済サービスのアップ/ダウングレード).
    • 購入済サービスを解約する処理 (購入済サービスの解約).
    • 利用者を登録する処理 (利用者の登録).

      トリガーは、一人の利用者を登録するためにしか設定できません。LDAPシステムや利用者情報ファイルからの複数利用者の一括インポートは、プロセス制御にて実行することはできません。

    サービス提供部門に対してのみ設定可能なトリガーは以下のとおりです:

    • サービス利用部門を登録する処理 (サービス利用部門の登録).
    • サービス利用部門に対する支払種別の設定を変更する処理 (サービス利用部門向け支払種別の管理).
    • サービスを有効化する処理 (サービスの有効化).
    • サービスを無効化する処理 (サービスの無効化).
    • 利用者が当該のサービス提供部門が提供しているサービスを購入する処理 (サービスの購入 (任意の利用者))
    • 利用者が当該サービス提供部門の提供するサービスの購入済サービスを変更する処理 (購入済サービスの変更 (任意の利用者))
    • 利用者が当該サービス提供部門の提供するサービスの購入済サービスを解約する処理 (購入済サービスの解約 (任意の利用者))
  3. 対象種別フィールドで、エントリをWebサービスのままとします。

    現時点では、これがサポートされている唯一の対象種別です。

  4. 対象URLフィールドで、所属組織に導入される通知サービスに関するWSDLファイルへのURLを入力します。 通知サービスの導入の詳細については、システム運用管理部門に連絡してください。
  5. 外部プロセス制御システムでアクションが承認または拒否されるまでプロセスの実行を一時停止するかどうかを指定します。

    一時停止しない場合、アクションは即座に実行されます。プロセス制御システムは一時停止しないトリガーが正常に実行されたかどうか通知されます。

    サービス提供部門向けの以下のトリガーについてはアクションの実行は停止されないことに注意してください： サービスの購入 (任意の利用者)、購入済サービスの変更 (任意の利用者)、そして 購入済サービスの解約 (任意の利用者)。

  6. トリガー定義を保存するには、保存をクリックします。

    [トリガーの管理]ページに戻り、追加をクリックして別のトリガーを設定するか、トリガーの一覧のエントリをクリックして、更新・削除することができます。

トリガー定義を更新するには：
  1. 必要に応じてトリガー定義を変更します。個別フィールドの詳細については、上記を参照してください。

    更新しようとしているトリガーがプロセス制御システムにて対応するプロセスを既に開始している場合、トリガーの名前しか変更することはできません。トリガーの他の項目を変更するには、その前に対応するプロセスを停止する必要があります。

  2. 更新されたトリガー定義を保存するには、保存をクリックします。

[トリガーの管理]ページに戻り、追加をクリックして別のトリガーを設定するか、トリガーの一覧のエントリをクリックして、更新・削除することができます。

トリガー定義を削除するには：

  1. 削除をクリックします。
  2. トリガーが削除されたことを確認します。

[トリガーの管理]ページに戻り、追加をクリックして別のトリガーを設定するか、トリガーの一覧のエントリをクリックして、更新・削除することができます。
