技術サービスのインポート

技術サービスのインポートページでは、XMLファイルから技術サービスの定義をインポートできます。これは、アプリケーションをプラットフォームに統合するプロセスの1つです。

既存の技術サービスを更新するには、更新する技術サービスと同じ名前（ID）を指定した技術サービス定義XMLファイルをインポートします。

以下を確認します。

  • この技術サービスに準拠したサービスが存在しない場合は、その技術サービスのすべてのプロパティを変更できます。
  • 技術サービスに準拠したサービスがある場合には、以下を行うことができません。
    • パラメーターとオプションの変更
    • イベント種別の変更
    • パラメーター、オプション、イベント種別の削除
  • ロールに料金が定義されている場合や、関連購入済サービスにロールのある利用者が割り当てられている場合は、ロールを削除できません。

サービス定義のインポートページでは、既存のサービスの可用性を確認し、詳細を表示することもできます。

技術サービス定義XMLファイルの詳細については、運用ガイド（技術サービス開発部門編）を参照してください。

技術サービスをインポートするには：

  1. ファイルシステムで技術サービス定義を含むファイルを検索するには、入力フィールドまたは参照をクリックします。
  2. 技術サービスをインポートするには、インポートをクリックします。
インポートした技術サービスが、利用できる技術サービス一覧に追加されます。次のステップで、サービスをこの技術サービスに準拠して定義できるように、技術サービスにサービス提供部門を1つ以上指名します。

既存技術サービスの詳細を参照するには、技術サービスの一覧で任意のエントリをクリックします。ポップアップする技術サービスページでは、技術サービスの名前、アクセス種別、サービスのURL、プロビジョニングサービスのURL、サービスの詳細説明、パラメーター定義とイベント定義、ロール定義と操作定義を参照できます。

利用できるパラメーターやイベント、ロール、操作定義などは、利用しているアプリケーションの実装およびその統合によって異なります。
