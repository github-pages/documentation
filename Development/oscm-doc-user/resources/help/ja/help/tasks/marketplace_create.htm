サービスポータルの登録

サービスポータルの登録ページでは、プラットフォームで動作するサービスポータルを作成できます。

サービスポータルの登録時にはサービスポータルの機能の選択および運用管理をおこなう組織の設定がおこなえます。設定した組織には自動的にサービスポータル運用部門のロールが割り当てられ、その組織のすべての管理者にはサービスポータル運用者のロールが設定されます。サービスポータル運用者はサービスポータルの管理やカスタマイズがおこなえます。

サービスポータルを登録するには、サービスポータル運用部門となる組織がすでに登録されている必要があります。

プラットフォームがSAML_SP認証モードでインストールされている場合、 サービスポータルにテナントを関連付けることができます。テナントを関連付けることでサービスポータルにアクセスする利用者の認証に使用するアイデンティティプロバイダー(IdP)を規定することができます。

サービスポータルを登録するには：

  1. サービスポータルに名前を指定します。

    サービスポータル名はサービスポータル運用者がサービスポータルを管理またはカスタマイズする際に選択肢として表示されます。

  2. 該当するチェックボックスを選択して、サービスポータルの機能を設定します。サービスポータル運用者は必要に応じて後で機能設定を更新できます。

    • タグ付け: 選択されている場合、サービスポータルでタグクラウドが使用できます。タグクラウドには、サービスポータルで提供されるサービスを特徴付ける検索用語が含まれます。サービスポータルの利用者は、これらのタグを使用してサービスにフィルタ処理を行うことができます。技術サービス開発部門がタグを指定します。技術サービス開発部門は提供サービスの基盤となるアプリケーションを用意します。

      選択されていない場合、タグクラウドと個々に設定されているタグはサービスポータル上で表示されず、サービスに設定されているタグは反映されません。

    • レビュー: 選択されている場合、サービス利用部門は購入したサービスに対してレーティングとコメントを記入できます。利用者はそのレーティングとコメントを見ることができます。

      選択されていない場合、サービスポータル上でレーティングとコメントは使用できません。

    • ソーシャルブックマーク: 選択されている場合、ソーシャルネットワークやコミュニケーションサービスに直接アクセスするためのブックマークがサービスポータル上のサービスに提供されます。利用者はこのブックマークを使ってソーシャルネットワーク上でサービスを推薦することなどができます。

      選択されていない場合、ソーシャルブックマークの機能は使用できません。

    • 許可されたサービス提供部門のみ: 選択されている場合、このサービスポータルには許可されたサービス提供部門、サービス仲介部門およびサービス提供代行部門がサービスを公開できます。このサービスポータルのサービスポータル運用者がサービス提供許可の設定をおこないます。

      選択されていない場合、すべてのサービス提供部門、サービス仲介部門およびサービス提供代行部門がサービスポータルにサービスを公開できます。サービス提供関連の部門はサービスポータルの管理者による明示的な許可なしでサービスを公開できます。

    • カテゴリ: 選択されている場合、サービスポータル上でカテゴリが表示されます。 サービスポータル運用者は、サービス提供部門、サービス仲介部門およびサービス提供代行部門がサービスポータルに公開するサービスに対して割り当てられるカテゴリを定義します。 カテゴリを用いることで利用者はサービスポータル上でサービスをフィルタリングして閲覧できます。 また、カテゴリはサービスポータルの全文検索機能にも使用されます。

      選択されていない場合、利用者はカテゴリによるフィルタリングなどのサービスの検索はおこなえません。 選択していない場合でも、カテゴリを定義し、サービスに割り当てることはできます。 既存のカテゴリ定義や割り当てには影響しません。

      注: サービスポータル運用部門は、サービスポータルのホームページに表示するサービスをグルーピングするのにカテゴリを使用できます。 この機能を無効にしてもサービスポータルのホームページには何の影響もありません。 この機能を無効にしても、サービスはカテゴリ毎にグルーピングされます。
  3. サービスポータルの運用部門となる組織を選択します。組織は複数のサービスポータルの運用部門になることができます。 この一覧には、プラットフォームで管理されるすべての組織が含まれます。
  4. プラットフォームがSAML_SP認証モードでインストールされている場合、サービスポータルの利用者が所属するテナントを選択します。本項目を設定しない場合、サービスポータルの利用者はインストール時に設定したデフォルトのIdPを使用して認証されます。 この一覧には、プラットフォームで管理されるすべてのテナントが含まれます。
  5. サービスポータルを作成するには、保存をクリックします。

サービスポータル運用部門の管理者には、サービスポータル運用者ロールが割り当てられます。サービスポータルが作成されたことがサービスポータルの運用部門の管理者にメールで通知されます。メールには2つのURLが含まれます。1つはサービスポータル運用者がサービスポータルの管理およびカスタマイズに利用します。もう1つのURLは、サービスポータルとそこで公開されているサービスへの公開アクセスに使用できます。

サービスポータルを作成すると、システム運用管理者は、サービスポータル運用部門の変更、サービスポータルの削除、テナントの変更を必要に応じて行うことができます。
