サービスポータルの変更

サービスポータルの変更ページでは、以下の操作を実行できます。

  • サービスポータル運用部門の設定。システム運用管理者がおこなえます。
  • サービスポータル運用部門向け売上配分設定およびサービス仲介部門およびサービス提供代行部門向けのデフォルトの売上配分設定がおこなえます。売上配分の設定はシステム運用管理者がおこないます。
  • サービスポータルの別名設定。サービスポータルを運用する組織のサービスポータル運用者がおこなえます。
  • サービスポータルの機能および設定の変更。サービスポータルを運用する組織のサービスポータル運用者がおこなえます。

サービスポータルの運用管理部門を変更するには(システム運用管理者のみ):

  1. 管理部門を変更するサービスポータルをサービスポータル一覧から選択します。

    一覧には、プラットフォームに存在するすべてのサービスポータルが含まれます。

  2. 別の組織をサービスポータルの運用管理部門として割り当てます。

    注：新しい運用管理部門をサービスポータルに割り当てると、以前の運用管理部門の組織からサービスポータル運用部門のロールが、またその利用者からサービスポータル運用者のロールが削除されることがあります。 これはそのサービスポータルが運用管理部門であった組織にとって最後の運用管理対象サービスポータルであった場合に起こります。

  3. 変更を保存するには、保存をクリックします。

    新しく割り当てられた組織の管理者には、サービスポータル運用者のロールが割り当てられます。彼らにはメールで通知されます。メールには2つのURLが含まれます。1つはサービスポータル運用者がサービスポータルの管理およびカスタマイズをおこなうためのURLです。 もう1つのURLは、サービスポータルへの公開アクセスに使用できます。

売上配分を設定するには(システム運用管理者のみ):

  1. サービスポータルから、売上配分設定をおこなうサービスポータルを選択します。

    一覧にはプラットフォームに存在するすべてのサービスポータルが含まれます。

  2. 以下の項目にパーセントの値を設定します:
    • サービスポータル運用部門への売上配分率 (%): サービスポータル運用部門がそのサービスポータル上でサービスを提供した組織から受け取る売上配分の売上配分率です。 サービスポータル運用部門向け売上配分率はサービスポータルに公開されているすべてのサービスに適用されます。
    • サービス仲介部門への売上配分率 (%): サービス仲介部門が提供したサービスについて、そのサービスの再提供設定を付与したサービス提供部門からサービス仲介部門が受け取るデフォルトの売上配分の売上配分率です。 このデフォルトの売上配分率はサービス仲介部門の売上配分ページでサービス固有の売上配分率または組織固有の売上配分率が設定されていない場合に適用されます。
    • サービス提供代行部門への売上配分率 (%): サービス提供代行部門が提供したサービスについて、そのサービスの再提供設定を付与したサービス提供部門からサービス提供代行部門が受け取るデフォルトの売上配分の売上配分率です。 このデフォルトの売上配分率はサービス提供代行部門の売上配分ページでサービス固有の売上配分率または組織固有の売上配分率が設定されていない場合に適用されます。

    売上配分率の入力例:

    • サービスポータル運用部門への売上配分率 (%): 10.00
    • サービス仲介部門への売上配分率 (%): 20.00
    • サービス提供代行部門への売上配分率 (%): 30.00

    ケース1: サービス提供部門のみがサービスポータルにサービスを公開し、システム運用管理部門の売上配分率が10%に設定されている場合。 サービスポータル運用部門はサービスを提供したサービス提供部門から収入の10%を受け取り、サービス提供部門は残りの金額からシステム運用管理部門の売上配分率を引いた額を受け取ります(90% - 10% = 80%)。

    ケース2: サービス提供代行部門がサービスポータルでサービスの再提供設定がサービス提供部門から与えられており、システム運用管理部門の売上配分率が10%に設定されている場合。 サービスポータル運用部門は収入の10%を受け取り、サービス提供代行部門は収入の30%を受け取り、サービス提供部門は残りの金額からシステム運用管理部門の売上配分率を引いた額を受け取ります(60% - 10% = 50%)。

    ケース3: サービス仲介部門がサービスポータルでサービスの再提供設定がサービス提供部門から与えられており、、システム運用管理部門の売上配分率が10%に設定されている場合。 サービスポータル運用部門は収入の10%を受け取り、サービス仲介部門は収入の20%を受け取り、サービス提供部門は残りの金額からシステム運用管理部門の売上配分率を引いた額を受け取ります(70% - 10% = 60%)。

    次の請求書作成処理において、売上配分が計算され請求データとして出力されます。

    設定された売上配分率は、システム運用管理者、サービスポータル運用部門、サービス提供部門、サービス提供代行部門およびサービス仲介部門の間で同意された条件に基づきます。システム運用管理部門は正しい値を設定しなければなりません。 売上配分率の合計は100%を上回ることが可能となります。これは、サービス提供部門が一時的にサービスの強く売りこみたい場合にサービス提供代行部門やサービス仲介部門の配分を80%などの高い値に設定するなどの場合を想定しています。 このとき仮に、サービスポータル運用部門の売上配分率が30%でシステム運用管理部門の売上配分率が10%だとすると、合計の売上配分率は120%となり、サービス提供部門の収入はマイナスの値となります。

  3. 売上配分を保存するには、保存をクリックします。

    売上配分は次回の請求書作成処理から適用が開始されます。個々の売上配分が計算され請求データとして出力されます。

サービスポータル名を変更するには(サービスポータル運用管理者のみ):

  1. サービスポータル一覧から、名前を変更するサービスポータルを選択します。

    一覧には、所属組織が運用しているすべてのサービスポータルが含まれます。

  2. サービスポータルの新しい名前を指定します。

    サービスポータルを管理またはカスタマイズする際およびサービス提供関連の部門に対するサービスの公開許可を設定する際にこのサービスポータル名を利用できます。

  3. 新しい名前を保存するには、保存をクリックします。

サービスポータルが新しい名前でただちに表示されます。

サービスポータルの機能および設定を変更するには:

  1. 機能および設定を変更するサービスポータルをサービスポータルの一覧から選択します。

    一覧には、所属組織が運用しているすべてのサービスポータルが含まれます。

  2. 必要に応じてサービスポータルのタグ付け機能を変更します。 選択されている場合、サービスポータルでタグクラウドが使用できます。タグクラウドには、サービスポータルで提供されるサービスを特徴付ける検索用語が含まれます。サービスポータルの利用者は、これらのタグを使用してサービスにフィルタ処理を行うことができます。技術サービス開発部門がタグを指定します。技術サービス開発部門は提供サービスの基盤となるアプリケーションを用意します。

    選択されていない場合、タグクラウドと個々に設定されているタグはサービスポータル上で表示されず、サービスに設定されているタグは反映されません。

  3. 必要に応じてサービスポータルのレビュー機能を変更します。 選択されている場合、サービス利用部門は購入したサービスに対してレーティングとコメントを記入できます。利用者はそのレーティングとコメントを見ることができます。

    選択されていない場合、サービスポータル上でレーティングとコメントは使用できません。 サービス利用部門がすでにレーティングおよびコメントを入力済みの状態でこの機能を無効化した場合、これらのレーティングとコメントのデータは残りますが非表示となります。 この機能を再度有効化すると、レーティングとコメントが再表示されます。

  4. 必要に応じてサービスポータルのソーシャルブックマーク機能を変更します。 選択されている場合、ソーシャルネットワークやコミュニケーションサービスに直接アクセスするためのブックマークがサービスポータル上のサービスに提供されます。利用者はこのブックマークを使ってソーシャルネットワーク上でサービスを推薦することなどができます。

    選択されていない場合、ソーシャルブックマークの機能は使用できません。

  5. 必要に応じてサービスポータルの許可されたサービス提供部門のみ機能を変更します。 選択されている場合、このサービスポータルには許可されたサービス提供部門、サービス仲介部門およびサービス提供代行部門がサービスを公開できます。 このサービスポータルのサービスポータル運用者がサービス提供許可の設定をおこないます。

    選択されていない場合、すべてのサービス提供部門、サービス仲介部門およびサービス提供代行部門がサービスポータルにサービスを公開できます。サービス提供関連の部門はサービスポータルの管理者による明示的な許可なしでサービスを公開できます。

    公開されたサービスポータルから、許可されたサービス提供関連の組織のみが公開できるサービスポータルの設定に変更した場合、その時点でそのサービスポータルにサービスを公開しているサービス提供関連の組織はすべて自動的に許可された組織として設定されます。 さらに、公開されたサービスポータルで設定されたブラックリストの設定も残り、その組織はそのサービスポータルへのサービスの公開が禁止されます。

    その逆の設定についても同様であり、ブラックリストに設定されているサービス提供関連部門を除いて、サービス提供関連の部門に対する許可は公開されたサービスポータルにおいても残り、許可されたサービス提供関連の部門がサービスポータルに公開しているサービスも残ります。

  6. 必要に応じてサービスポータルのカテゴリ機能を変更します。 選択されている場合、サービスポータル上でカテゴリが表示されます。 サービスポータル運用者は、サービス提供部門、サービス仲介部門およびサービス提供代行部門がサービスポータルに公開するサービスに対して割り当てられるカテゴリを定義します。 カテゴリを用いることで利用者はサービスポータル上でサービスをフィルタリングして閲覧できます。 また、カテゴリはサービスポータルの全文検索機能にも使用されます。

    選択されていない場合、利用者はカテゴリによるフィルタリングなどのサービスの検索はおこなえません。 しかし、カテゴリの定義およびサービスへの割り当ては行うことができます。 既存のカテゴリの定義や割り当ては影響を受けません。

    注: サービスポータル運用者は、サービスをグルーピングしてサービスポータルのホームページに表示するのにカテゴリを使うことができます。 カテゴリ機能を無効にしてもサービスポータルのホームページには何の影響もありません。 カテゴリ機能を無効にしても、サービスはカテゴリ毎にグルーピングされます。
  7. 保存をクリックし、更新された機能および設定を保存します。

更新した機能および設定はただちに適用されます。 サービスポータルの訪問者には、サービスポータルに次回ログインしたときに変更が反映されます。
