プロセスの管理

組織のアクションはプロセス制御の対象となり、外部システムの承認を受けるまで実行されません。

管理者は、各アクションを実行する際に承認が必要かどうかを指定できます。承認が必要な場合、プロセス制御システムから承認されるか拒否されるまでアクションは待機中になります。承認が不要な場合、そのアクションは即座に実行されます。

アクションをプロセスで制御するには通知用のサービスが必要です。通知用のサービスはプラットフォームとプロセス制御システムのインターフェースの役割を担います。通知用サービスの実装の詳細については、システム運用管理部門にお問い合わせください。

外部プロセス制御の対象となるアクションが実行された場合、そのステータスがプロセスの管理ページに表示されます。利用者のアクションを契機とするプロセスは参照、中止、削除することができます。ステータスには以下のものがあります。

  • 実行中：アクションは初期化されました。
  • 承認待ち：アクションを実行する前にプロセス制御システムで承認する必要があります。
  • 承認済み：アクションがプロセス制御システムで承認され、実行されました。例えば、サービスの申し込みを要求した場合に、その購入済サービスが作成されました。
  • 却下：アクションがプロセス制御システムで承認されなかったため、実行されません。
  • 通知済み：このアクションには正常に実行され、プロセス制御システムは実行されたことについて通知されています。
  • 外部システムエラー: プロセス制御システムとの通信が失敗し、アクションは実行されません。
  • 失敗:プロセス制御システムにてアクションが承認された後に、アクションの実行中にエラーが発生した。アクションは実行されていません。
  • 中止：アクションは中止されました。

プロセスを管理するには：

  • サービス購入管理者としてログインしている場合、プロセスの管理ページにはプロセス制御の対象すべてと自分のアクションが表示されます。説明欄に、要求されたアクションが表示されます。ステータス欄に現在の処理ステータスが表示されます。
  • 管理者としてログインしている場合、対応するチェックボックスを選択して、すべてのアクションを一覧に表示するかまたは自分のアクションのみを表示するのかを選択します。チェックボックスを選択しない場合、プロセスをトリガーした所属組織のすべての利用者のアクションが表示されます。

アクションの実行を中止するには：

アクションの実行を中止するには、アクションの横にあるチェックボックスを選択して中止をクリックします。応答待ちのアクションのみ中止できます。

ステータスが「中止」に変わります。このアクションにはプロセス制御システムによる承認は必要なくなります。中止されたアクションは実行されず、一覧から削除することができます。プロセス制御システムはアクションが中止されたことについて通知を受け、承認を要求する処理を中止することができます。

アクションを削除するには:

アクションの横にあるチェックボックスを選択して削除をクリックします。中止のアクションのみ完全に削除できます。
