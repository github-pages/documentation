利用者の管理

利用者の管理ページでは、不正アクセスが疑われる場合などに特定の利用者のアカウントをロックできます。 また、利用者が設定回数（デフォルト：3回）以上、間違ったパスワードでログイン失敗を繰り返した場合のロックを解除することもできます。 さらに、利用者のパスワードをリセットすることができます。

利用者の認証に外部システムを使わない組織では、利用者情報ファイルをインポートすることで１回の操作で複数の利用者を登録することができます。

ページのトップに、プラットフォームで登録された利用者数と設定された登録された最大利用者数を見ることができます（デフォルト：10）。一定の間隔（デフォルト：12時間）で、システムは登録された利用者数を確認します。登録できる利用者数を超えたという結果が返却された場合、メールで通知されます。

この作業はすべて組織の管理者の要求に応じて行い、作業に必要な利用者IDはその組織の管理者が提供します。

注: プラットフォームがINTERNAL認証モードでインストールされており、組織が利用者の管理や認証に外部LDAPシステムを使用している場合、LDAPシステムにマッピングされていない項目のみ編集可能です。 このため、パスワードは外部システムにおいてのみ変更できます。 利用者情報ファイルで定義した複数の利用者のインポートについても、この場合はできません。

プラットフォームがSAML_SP認証モードでインストールされている場合、パスワードのリセットボタンは使用できません。 さらに、利用者のロックおよびロック解除はプラットフォーム内でのみ有効であり、アイデンティティプロバイダー(IdP)には適用されません。 このため、プラットフォームでロックされた利用者はIdPにはまだログインすることができます。 パスワードはIdP上でリセットする必要があります。 利用者情報ファイルで利用者を定義する場合、利用者IDはIdPのものと一致している必要があります。

認証モードの詳細については、「運用ガイド システム運用管理部門編」を参照ください。

パスワードのリセット、もしくは利用者のロック、ロック解除するには:

  1. 利用者IDを入力するか、一覧から選択します。
  2. パスワードのリセット、ロック解除またはロックするをそれぞれクリックします。

すべての場合において該当の利用者にはメールで通知されます。

組織に複数の利用者を登録するには:

  1. 組織の管理者に利用者情報を含むファイルを提供するように頼むか、もしくは自分でファイルを作成します。 ファイルの書式や内容の詳細は複数利用者インポートのための利用者情報ファイルを参照ください。
  2. 利用者情報ファイルをインポート:
    1. 利用者をインポートする組織を選択します。
    2. 一般のサービス利用者をその組織にインポートする場合、利用者が割り当てられるサービスポータルを選択します。
    3. ファイル名 フィールドか ファイル選択 をクリックし、利用者情報を含んでいるファイルを選択します。
    4. インポートをクリックします。

      十さしにインポートを開始する前に、システムは利用者情報ファイルの妥当性をチェックします。システムは以下について確認します:

      • 利用者情報を含む行が完全（すべてのフィールドが指定されている）であること。
      • 性別およびロールの値が正しい綴りになっていること。
      • 複数の利用者ロールは、ダブルクォートで囲まれていること。
      利用者情報ファイルの検証でエラーを見つけた場合は、対応するメッセージを表示し、インポートは開始しません。

      インポートをおこなうには、その前にファイルの情報を修正する必要があります。

      インポートを多重でおこなうことはできません。以前のインポートが完了するまで待つ必要があります。登録する利用者数が多い場合、処理が終わるまでしばらく時間がかかります。

インポートが終了すると直ぐに、インポートおよび登録に成功したすべての利用者に対してログイン情報と初めてログインするときに利用する初期パスワードを記載したメールが送信されます。 新規に登録された利用者は自動的に組織のデフォルトのユニットに追加されます。デフォルトユニットには、組織に現在登録されているすべての利用者が含まれています。

インポートを実施した利用者には、以下の情報を含んだメールが送信されます:

  • インポートおよび登録に成功した利用者の数。
  • インポートに失敗した利用者ごとに：利用者IDと失敗理由。

    考えられる原因は:

    • 指定されたIDの利用者が既に存在する。
    • 指定された利用者IDが無効である。
    • 指定された利用者ロールが、その組織では利用できない。例えば、サービス提供部門の場合、利用者ロールBROKER_MANAGERは指定できません。
