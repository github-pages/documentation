LDAP設定

組織の利用者IDとパスワードはプラットフォーム上または組織の既存のLDAPシステム上で作成または維持管理できます。

LDAPシステムを使用する場合、組織はプラットフォーム上で手動で利用者を登録する必要はありません。組織の管理者はLDAPシステムから利用者をインポートすることができ、それによって自動的にプラットフォームに利用者を登録できます。その利用者の管理はLDAPシステム上でおこなわれます。プラットフォームは継続的に利用者情報の同期をおこないます。プラットフォームがLDAPシステムに接続できるよう、設定ファイルには接続パラメーターを定義する必要があります。

LDAP設定ページでは、組織のLDAP情報を管理することができます。それによって、システム運用管理者はプラットフォームのデフォルトの設定パラメーターを上書き、削除、追加できます。

LDAP設定の変更が必要な場合(例:接続URLが変更されるなど)、利用者は現在の設定をファイルとしてエクスポートし、必要な値を変更して再度インポートします。注:この操作は組織の既存のLDAP設定すべてを上書きします。

LDAP設定ページにアクセスすると、現在のLDAP設定の内容は読み取り専用のテーブルとして一覧表示されます。テーブルには、現在の設定値のキー／値の組み合わせと設定値がプラットフォームの標準の値であるかどうか(組織特定の値が設定されていない場合はチェックがつく)が表示されます。

LDAP設定を管理するには:

  1. ファイルシステム上でLDAP設定ファイルを作成し、ファイルにLDAP接続設定を定義します。あるいは、エクスポートをクリックすることで現在のLDAP設定ファイルを取得できます。
  2. ファイルの設定内容を確認し、必要に応じて変更します。

    設定ファイルのサンプル: 

    LDAP_BASE_DN=ou\=people,dc\=est,dc\=mycompany,dc\=de
LDAP_CONTEXT_FACTORY=com.sun.jndi.ldap.LdapCtxFactory
LDAP_URL=ldap://myldapserver.lan.est.company.de:389
LDAP_ATTR_UID=uid
LDAP_ATTR_LAST_NAME=sn
LDAP_ATTR_FIRST_NAME=givenName
LDAP_ATTR_EMAIL=scalixEmailAddress
LDAP_ATTR_REFERRAL=ignore
  3. ファイル名入力部分またはファイル選択をクリックし、設定ファイルを選択します。
  4. インポートをクリックします。 すべての既存のLDAP設定が設定ファイルの内容で上書きされます。
  5. 接続テストをクリックして、LDAPシステムへの接続を確認します。
新しいLDAP設定はすぐに組織に適用されます。
注: 誤ったLDAP接続パラメーターを入力した場合、それ以降プラットフォームにログインすることができなくなります。この場合はシステム運用管理者に連絡し、組織のLDAP設定をリセットするよう依頼してください。
