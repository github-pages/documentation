利用者のインポート

利用者のインポートページでは、外部のLDAPシステムから利用者をインポートして登録できます。また、利用者にロールを設定することができます。ログイン時に利用者を認証するためにLDAPシステムが使用されます。

組織を作成や登録するにあたって、事前にLDAPベースの認証を指定し、LDAP接続設定を定義する必要があります。

利用者に割り当てることができるロールは組織のロールに応じて決まります。利用者はそのロールによっておこなえる操作が決まります:
  • サービス利用者：特に権限を持たないロールの利用者は、所属組織が契約しているサービスを使用できます。登録されているすべての利用者は自動的にこのロールが割り当てられます。本名称は便宜的で、管理ポータルで本利用者のために特別なロールを設定することはありません。
  • 管理者：各組織には、このロールの利用者が少なくとも１人いなければなりません。管理者は組織のアカウント、サービスの購入およびロールの管理がおこなえます。組織の登録時にその組織の最初の管理者が登録れます。
  • サービス購入管理者: このロールは、サービスを購入することができ、自分がサービス購入管理者になっている購入済サービスを管理することがことができます。管理者とは異なり、サービス購入管理者は他の者がサービス購入管理者になっている購入済サービスや、請求や支払に関連した購入済サービスの情報を取り扱うことはできません。
  • 技術サービス管理者：このロールの利用者は技術サービスを登録できます。このロールは技術サービス開発部門の利用者に割り当てることができます。
  • サービス管理者：このロールの利用者はサービスの登録、サービスの料金設定およびサービスポータルへのサービスの公開がおこなえます。サービス提供部門の利用者に割り当てることができます。
  • サービス提供代行者: このロールの利用者はサービス提供部門のサービスを異なる利用規約を適用して公開できます。サービス提供代行部門の利用者に割り当てることができます。
  • サービス仲介者: このロールの利用者はサービス提供部門のサービスを利用規約を変更せずに公開できます。サービス仲介部門の利用者に割り当てることができます。
  • サービスポータル運用者：このロールの利用者は、サービスポータルへアクセスしサービスを公開できる組織の定義、およびサービスポータルの更新やカスタマイズをおこなえます。サービスポータル運用部門の利用者に割り当てることができます。サービスポータルが登録されるとき、サービスポータル運用部門となる組織のすべての管理者に自動的にこのロールが割り当てられます。

外部のLDAPシステムから利用者をインポートするには：

  1. インポートする利用者を特定します。 以下のフィルタ条件を使用できます。
    • ワイルドカード（*）：LDAPシステムに存在する全利用者が一覧表示されます。表示される利用者は100人までです。必要に応じて利用者IDの開始文字とワイルドカードなど、特定のフィルタ条件を使用します。
    • 文字とワイルドカード（*）：指定した文字に一致する利用者IDの全利用者が一覧表示されます。例えば、*o*と入力すると、利用者IDにoという文字を含むすべての利用者が一覧表示されます。
    • 利用者ID：特定の利用者を1人だけインポートする場合は、そのIDを直接入力します。

    検索をクリックすると、フィルタ条件に一致する利用者の一覧が表示されます。

  2. インポートする利用者のチェックボックスを選択します。 利用者のメールアドレスを読み込むLDAP属性が対応する値にマッピングされていない場合、手動で各利用者のメールアドレスを入力する必要があります。
  3. 利用者に組織内の特別ロールを割り当てる場合、管理者、サービス購入管理者、サービス管理者、技術サービス管理者、サービス提供代行者、サービス仲介者、サービスポータル運用者またはシステム運用管理者チェックボックスを必要に応じて選択します。

    利用できる利用者ロールは所属組織のロールによって異なります。チェックボックスを選択しない場合、利用者に自動的に利用者ロールが割り当てられます。

  4. インポートを開始するには、インポートをクリックします。

選択した利用者がインポートされ、登録されます。インポートと登録が完了すると、サービスにアクセスするためのURLを含むメールが利用者に送信されます。これらの利用者は、組織のデフォルトユニットに自動的に追加されます。デフォルトユニットには、組織に現在登録されているすべての利用者が含まれています。

インポートしようとする利用者の利用者IDはすでにプラットフォームで存在した場合、その利用者のメールアドレス（例：userSmith@organization.com）はシステムへログインするための利用者IDとして利用されます。さらにもう一つの利用者を同じ利用者IDでインポートした場合、新規利用者IDは既存のIDの後ろにアットマーク（@）と利用者が所属する組織のID（例：userSmith@abcdefg）で構成されます。利用者には該当利用者IDが記載されたメールが送信されます。

パスワード、名、姓など、外部のLDAPシステム側で管理される利用者データは、外部のLDAPシステムでしか変更できません。データが変更されている場合、プラットフォームが自動的に同期します。注:LDAPシステムの利用者IDが変更されている場合、自動の同期はできません。この場合、利用者情報を再度インポートする必要があります。
