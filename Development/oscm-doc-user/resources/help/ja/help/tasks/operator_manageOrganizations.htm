組織の管理

組織の管理ページでは、以下の操作を行うことができます。

  • 別のロールを組織（技術サービス開発部門、サービス提供部門、サービス提供代行部門またはサービス仲介部門）に追加する。
  • 組織の所在地および連絡先データを更新する。
  • 組織の識別名を設定する。

    組織が、組織とシステム運用管理部門間で証明書を使ったセキュアな通信を行う場合に必要です。

    プラットフォームは、クライアント証明書の識別名を確認します。証明書内の識別名がここで指定した識別名に一致する場合、呼び出し側はプラットフォームへの接続が許可されたものとみなされます。

    Java実行環境では以下の呼び出しによって、証明書から組織の識別名を取得できます。 

    <インストールされたJREのホームディレクトリ>/bin/keytool -printcert -file <ファイル名.crt>

    証明書の統合方法については運用ガイド（システム運用管理者編）に詳しい説明があります。

  • プラットフォームがSAML_SP認証モードでインストールされている場合、組織のテナントを変更もしくは関連付けします。

    組織にテナントを関連付けることにより、組織情報の変更後にテナント向けのサービスポータルLおよび管理ポータルのURLが記載されたメールが送付されます。テナントは事前にテナント管理ページにて作成する必要があります。サービスポータルの一覧はテナントに関連するすべてのサービスポータルを表示します。

    本項目を設定しない場合、デフォルトのテナントに関連するURLが設定されます。

  • サービス提供部門またはサービス提供代行部門がサービス利用部門に、プラットフォームの請求システムを使用しているサービスに対して提供することができる支払種別を定義します。

    デフォルトでは、サービス提供部門およびサービス提供代行部門はサービス利用部門が有料のサービスに申し込んだ際、請求書受領時の支払いを可能とする設定になっています。

    サービス提供部門またはサービス提供代行部門から請求書以外の支払種別を要求された場合、PSPのサービスを利用する必要があります。

    PSPサポートの統合処理については運用ガイド（システム運用管理者編）に詳しい説明があります。

組織を管理するには：

  1. 編集するデータの対象組織を組織ID一覧から選択します。

    この一覧には、プラットフォームで管理されるすべての組織が含まれます。

  2. 必要に応じてデータを編集します:
    1. 組織名: 組織の名前 (例:サービス利用管理部門)
    2. 代表メールアドレス: 組織と通常の連絡を行うためのEメールアドレス。組織には登録後に請求書の送り先として別のメールアドレスを登録することができます。

      指定されたメールアドレスの書式が有効な構文か確認され、ドメイン名がIANA（Internet Assigned Numbers Authority）によって定義、管理されている標準に適合しているか確認されます。

      例: user.name@domain.arpa, user.name@domain.org, user@mycompany.lan.uk

    3. 電話番号: 組織の担当者に連絡ととるための電話番号もしくは携帯電話番号。
    4. Web: 組織のWebサイトのURL。サービス利用部門が購読したいと考えているサービスを提供している組織について詳細な情報を得るためにアクセスするWebサイトです。
    5. 言語: 組織が使用するデフォルトの言語です。
    6. 所在地: 組織の所在地。組織の管理者は組織の登録後に請求書の送り先として別の住所を登録することができます。
    7. 国: 組織が所在する国。
    8. 説明: 組織の説明文。(例:組織のプロフィール情報)
      注: 注:テキストをフォーマットするために提供されたエディタを用いて編集してください。HTMLを手作業で編集し、外部のスタイルシートやコンテンツを参照した場合、画面のレイアウトにエラーが生じる可能性があります。
    9. [サービス提供部門の場合のみ] 標準のシステム運用管理部門の売上配分率 (%): システム運用管理部門とサービス提供部門の間で合意された売上配分率を入力してください。 システム運用管理部門はプラットフォーム上で管理されているサービスポータルで購入されたサービスについて、この売上配分率に基づく金額をサービス提供部門から受けとります。 標準のシステム運用管理部門の売上配分率は、操作メニューのシステム運用管理部門の売上配分設定画面でサービス固有の売上配分率を設定しない限り、サービス提供部門が提供するサービスすべてに適用されます。 これはサービス提供部門が直接サービスを提供したかどうか、つまりサービス仲介部門やサービス提供代行部門をとおしてサービスを提供したかどうかにかかわらず適用されます。 サービス提供部門がサービス利用部門に適用する割引は、売上配分率から差し引かれます。
      注: 売上配分と割引は プラットフォームの請求システムを使用しているサービスに対してだけ計算されます。外部の請求システムを使用しているサービスは計算には含まれません。
    10. [技術サービス開発部門、サービス提供部門、サービス提供代行部門、サービス仲介部門の場合]

      プロフィール画像: サービスポータルで使用する画像。プロフィール画像は利用部門がサービスの詳細を参照した際に表示されます。

      画像のアップロード入力部分またはファイル選択をクリックし、画像のファイルを選択します。利用可能なファイルフォーマットはGIF、JPG、PNGです。プロフィール画像の最大サイズは 154 x 154 ピクセルです。推奨されるサイズは 154 x 60 ピクセルです。

  3. 組織のロールで必要に応じて新しい組織に付与するロールを設定します。 サービス提供代行部門とサービス仲介部門を組み合わせることおよび技術サービス開発部門またはサービス提供部門と組み合わせて設定することはできません。
  4. 識別名フィールドで組織のDNを入力します。 DNは組織の証明書で定義され、以下の要素で構成されます。
    • CN（共通名）
    • OU（組織単位）
    • O（組織）
    • L（場所）
    • ST（州）
    • C（国名）

    DNの個別の要素はコンマで区切ります。

  5. プラットフォームがSAML_SP認証モードでインストールされている場合、テナントフィールドが表示されます。 組織にテナントを関連付けることにより、組織情報の変更後にテナント向けのサービスポータルLおよび管理ポータルのURLが記載されたメールが送付されます。

    テナントを更新することでテナントから組織をはずす、もしくは他のテナントを関連付けることができます。

    注: 他のテナントを関連付ける場合、IdPの管理者は他のテナントに関連づいているIdPシステムで本組織のすべての利用者が適切に構成されていることを確認する必要があります。本組織の利用者がすでに同名で新しいIdPシステム内に存在する場合、テナントの変更はできません。

    サービス提供部門、サービス提供代行部門、サービス仲介部門がテナントに関連する本マーケットプレイスに対するサービスの公開権限を持っている場合、新しいテナントを関連付けると権限が削除されます。

  6. 組織がサービス利用部門に提供することができる支払種別を定義します。[追加]をクリックし、新規支払種別ページに進みます。 支払種別の追加はサービス提供部門およびサービス提供代行部門に対してのみ行うことができます。
    注: サービス提供部門またはサービス提供代行部門に割り当てられた支払種別は削除できなくなります。
  7. 入力内容を保存するには、保存をクリックします。

変更内容については、組織の管理者にメールで通知されます。
