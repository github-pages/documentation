サービスの管理

サービスの管理ページでは、サービスポータルへ登録済のサービスが準拠する技術サービスの要素を除き、サービスポータルへ登録済のサービスの定義のすべての要素を変更できます。サービスを更新すると、利用者向けサービス名、簡単な説明、詳細説明、カスタムタブの名前も他の言語で提供できます。

まだ有効化されていないサービスのみが更新できます。既存のサービス利用部門の購入済サービスには影響はありません。変更内容は新しい購入済サービスに対してのみ有効になります。サービス利用部門が購入できるようにするには、再度そのサービスを有効化する必要があります。

サービス仲介部門またはサービス提供代行部門に対してサービスの再提供設定を付与している場合、たとえそのサービスが再度有効化される前であっても、サービス仲介部門、サービス提供代行部門およびサービス利用部門には変更されたサービスの内容が表示されます。既存の購入済サービスには変更内容は反映されません。おこなったすべての変更はそれ以降に新しく購入された場合にのみ反映されます。

サービス定義の要素と情報を変更するには：

  1. 変更するサービスポータルへ登録済のサービスをサービスID一覧から選択します。

    一覧にはログインしている利用者の組織が登録したすべてのサービスが含まれます。

  2. 以下の要素と情報を変更できます。
    • サービスID：サービスの識別子です。これは所属組織のサービスポータルへ登録済のサービス内で一意である必要があります。

      サービスIDは40文字以内で以下の文字を含めることはできません。

      !" # $ % & ' * + , / :; < = > ?\ ^ `

      このサービスIDは、サービスを更新または削除するとき、あるいはオプションのアップグレード／ダウングレードや料金設定を指定するときなどに選択できます。

    • サービス画像：サービスを特定するイメージファイルです。

      画像のアップロードフィールドまたは参照をクリックし、そのイメージを含むファイルを選択します。サポートされるファイル形式はGIF、JPG、PNGです。サービス画像のサイズは80x80ピクセルでなければなりません。をクリックします

      サービスを公開するとサービスポータルのサービスタカログにサービス画像が表示されます。イメージを利用すると、サービス利用部門がサービスを識別しやすくなります。

    • 利用者向けサービス名：サービスの有意な名前です。サービス利用部門にとって認知しやすい値にする必要があります。

      サービスを公開するとサービスポータルのサービスタカログにサービス利用部門向けのサービス名が表示されます。指定しない場合、代わりにサービス名が設定されていない旨のメッセージが表示されます。

    • 概要：サービスの簡単な説明です。簡単な説明の長さは256文字までに制限されています。

      サービスを公開するとサービスポータルのサービスタカログに簡単な説明が表示されます。通常、この説明には、そのサービスが提供する機能の概要が含まれます。

    • 説明：サービスによって提供される機能の詳細な説明です。

      サービス利用部門がサービスの詳細を要求すると、サービスポータルにサービスの詳細説明が表示されます。

      注: 書式を変更する際は、エディタが提供するコントロールのみを使用してください。手作業でHTMLタグを入力したり外部のスタイルシートやコンテンツを参照すると、ページレイアウトがエラーになる場合があります。
  3. 自動的に利用者を割り当てるを選択または選択解除することで、購入手続きが完了した後にこのサービスを購入する利用者が購入済サービスに自動的に割り当てられるかどうかを指定することができます。
  4. サービスパラメーター表で値欄の値を編集することによって、そのサービスの購入済サービスと使用法パラメーターを変更することができます。値は、技術サービスに対して定義された範囲内でなければなりません。 パラメーターに利用者による設定のチェックボックスを選択することによって、サービスを契約するサービス利用部門にオプションとして提供されるかどうかを指定できます。
  5. 必要に応じて、外部パラメーターの設定ツールへのURLを指定、もしくは変更します。 この項目を指定することで、希望の外部ツールにリンクすることができます。 利用者は、サービスを購入する際にこの外部ツールを使って購入済サービスやパラメーターを構成できるようになります。
    外部ツールを使った場合、サービスを購入する際に表示されるサービスのパラメーターテーブルは指定されたツールのインターフェースで置き換わります。
    注: カスタムタブに指定するWebページやWebアプリケーションに接続するURLは安全であることを確認してください。プラットフォームはURLの正当性(シンタックスや内容)を確認しません。

    安全な通信を行うためウェブページやウェブアプリケーションに送られるリクエストは プラットフォームによって署名されます。これには プラットフォームに特定の設定が必要です。ウェブページやウェブアプリケーションに使用されるアプリケーションサーバーは署名の検証のためトラストストアに プラットフォームの証明書が必要です。必要に応じて、システム運用管理部門にお問い合わせください。

  6. 必要に応じて、カスタムタブのURLを変更します。この項目を指定することでサービスポータルの購入済サービスの詳細画面にタブを追加することができます。本URLはカスタムタブ向けのWebページもしくはWebアプリケーションのコンテンツを表示します。たとえば、IaaSサービスの利用者向けにサービス詳細やサービスの操作に関する情報を表示することが可能です。

    OpenStackやAmazon Web Services (AWS)といったIaaSサービスを提供する購入済サービスの詳細を参照するために、事前に定義したカスタムタブを使用することができます。カスタムタブでは次の情報により配備した仮想マシン(VM)の一覧が表示されます。:

    • 購入済サービスの名前(ヘッダー)
    • 接続のための情報
    • 個々のVMに対しての情報:
      • インスタンス名
      • インスタンスID
      • インスタンスタイプ
      • インスタンスの状態
      • パブリックIP
      • プライベートIP

    次のフォーマットでカスタムタブへアクセスするためのURLを設定することができます:

    http://<app-host-name>:<app-port>/<service controller name>/serverInformation.jsf

    使用するプロトコルにより、次のフォーマットを使用します:

    https://<app-host-name>:<app-port>/<service controller name>/serverInformation.jsf

    例:

    https://MyAPPHost:8881/oscm-app-openstack/serverInformation.jsf

  7. 必要に応じて、利用者に表示されるカスタムタブの名前を変更します。40文字以内の英数字で指定します。特殊文字は使用できません。

  8. 保存 をクリック、変更を保存します。
  9. 利用者向けサービス名、概要、説明、カスタムタブ名を別の言語で提供するには、ローカライズをクリックします。サービスの説明項目のローカライズページでは、サポートされているすべての言語でサービスについて説明する要素を提供できます。

サービスを有効化した後に、サービス利用部門は変更後のサービスを購入できます。一方、サービス仲介部門、サービス提供代行部門およびそれらのサービス利用部門はすぐにその変更された後の内容を見ることができます。
