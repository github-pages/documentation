LDAP設定の管理

組織の利用者IDとパスワードはプラットフォーム上または組織の既存のLDAPシステム上で作成または維持管理できます。

プラットフォーム上でメンテナンスした場合、利用者はプラットフォームのデータベースに保存されます。組織の管理者は新しい利用者を登録することができ、必要に応じてシステム運用管理者にパスワードのリセットを依頼できます。

LDAP設定の管理ページでは、プラットフォーム全体に適用するLDAP管理情報を設定することができます。さらに組織のLDAPシステムの設定の確認と再設定ができます。（例:LDAPシステムに接続できないなど）

プラットフォームのLDAP情報の設定はLDAP情報がプラットフォーム上で複数の組織によって再利用される場合に役立ちます。また、情報の上書き、削除、組織ごとの設定の追加もおこなえます。

プラットフォームのLDAP情報を設定するには:

  1. ファイルシステム上でLDAP設定ファイルを作成し、ファイルにプラットフォームのLDAP接続設定を定義します。

    設定ファイルのサンプル: 

    LDAP_BASE_DN=ou\=people,dc\=est,dc\=mycompany,dc\=de
LDAP_CONTEXT_FACTORY=com.sun.jndi.ldap.LdapCtxFactory
LDAP_URL=ldap://myldapserver.lan.est.company.de:389
LDAP_ATTR_UID=uid
LDAP_ATTR_LAST_NAME=sn
LDAP_ATTR_FIRST_NAME=givenName
LDAP_ATTR_EMAIL=scalixEmailAddress
LDAP_ATTR_REFERRAL=ignore
  2. ファイル名入力部分またはファイル選択をクリックし、LDAP設定を選択します。
  3. インポートをクリックします。 設定ファイルに設定されている内容は読み取り専用のテーブルとして一覧表示されます。
  4. 入力内容を保存するには、保存をクリックします。

組織のLDAP設定を確認または更新するには:

  1. LDAP設定を定義または変更する組織を選択します。

    現在設定されているLDAP設定の値が一覧表示されます。また、設定値がプラットフォームのデフォルト値であるかどうかが表示されます。

  2. エクスポートをクリックして、現在の設定が記載されたファイルをファイルシステム上に保存します。
  3. 設定ファイルを編集します。
  4. LDAP設定を選択します。ファイル名入力部分またはファイル選択をクリックし、設定ファイルを選択してください。
  5. インポートをクリックし、メッセージを確認します。
    注: この操作は組織の既存のLDAP設定すべてを上書きします。
  6. 接続テストをクリックして、LDAPシステムへの接続を確認します。
