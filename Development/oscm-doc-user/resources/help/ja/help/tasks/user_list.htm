利用者の管理

利用者の管理ページでは、所属組織に登録された利用者を管理できます。利用者情報や利用者のロールの変更や、利用者のパスワードのリセット、利用者の削除ができます。購入済サービスを行っている場合、利用者にサービスへのアクセス権を付与したり、サービスへのアクセス権を剥奪することもできます。

注: 所属組織が利用者の管理や認証に外部システムを使用している場合、パスワードは外部のシステムにおいてのみ変更できます。したがって、その場合はパスワードをリセットするためのオプションは表示されません。

利用者情報と利用者のロールを変更するには：

  1. ページ上部にある一覧から、情報と利用者のロールを変更する利用者を選択します。
  2. ページ下部のフィールドで、対応するフィールドを編集して利用者情報を変更します。

    利用者の管理と認証に外部システムを使用している場合、外部システムのエントリにマッピングされていない項目のみ編集できます。

  3. 選択した利用者のロールを変更するには、対応する利用者のロールチェックボックスを選択するか、選択を解除します。選択できる利用者のロールは所属組織のロールによって異なります。
    • 管理者：各組織には、このロールの利用者が少なくとも１人いなければなりません。管理者は組織のアカウント、サービスの購入およびロールの管理がおこなえます。組織の登録時にその組織の最初の管理者が登録れます。
    • ユニット管理者: ユニットを利用して組織の利用者をグループ分けすることができます。ユニット管理者の利用者ロールを持つ利用者は、管理者として設定されているユニットに対して、サービスの購入、変更、解約を行うことができます。また、コスト管理のためにレポートを生成することができます。
    • サービス購入管理者: このロールは、サービスを購入することができ、自分がサービス購入管理者になっている購入済サービスを管理することがことができます。管理者とは異なり、サービス購入管理者は他の者がサービス購入管理者になっている購入済サービスや、請求や支払に関連した購入済サービスの情報を取り扱うことはできません。
    • 技術サービス管理者：このロールの利用者は技術サービスを登録できます。このロールは技術サービス開発部門の利用者に割り当てることができます。
    • サービス管理者：このロールの利用者はサービスの登録、サービスの料金設定およびサービスポータルへのサービスの公開がおこなえます。サービス提供部門の利用者に割り当てることができます。
    • サービス提供代行者: このロールの利用者はサービス提供部門のサービスを異なる利用規約を適用して公開できます。サービス提供代行部門の利用者に割り当てることができます。
    • サービス仲介者: このロールの利用者はサービス提供部門のサービスを利用規約を変更せずに公開できます。サービス仲介部門の利用者に割り当てることができます。
    • サービスポータル運用者：このロールの利用者は、サービスポータルへアクセスしサービスを公開できる組織の定義、およびサービスポータルの更新やカスタマイズをおこなえます。サービスポータル運用部門の利用者に割り当てることができます。サービスポータルが登録されるとき、サービスポータル運用部門となる組織のすべての管理者に自動的にこのロールが割り当てられます。
    • システム運用管理者：このロールの利用者は構成変更、保守作業、組織の管理およびサービスポータルの登録がおこなえます。プラットフォームがインストールされるとシステム運用管理部門と共に最初のシステム運用管理者が作成されます。

    チェックボックスを選択しない場合、自動的に利用者のロールが割り当てられます。特に権限を持たないロールの利用者は、所属組織が契約しているサービスを使用できます。

    利用者のロール割り当てを変更すると、その利用者の次回ログイン時に新しい割り当てが有効になります。

  4. 変更を保存するには、保存をクリックします。
利用者のパスワードをリセットするには:
  1. ページ上部にある一覧から、パスワードをリセットする利用者を選択します。利用者がログアウトしている場合のみ、その利用者のパスワードをリセットできます。
  2. パスワードのリセットをクリックします。

パスワードがただちにリセットされます。次回ログイン時に使用する新しい初期パスワードを通知するメールが該当の利用者に送信されます。

利用者を削除するには：

  1. ページ上部にある一覧から、削除する利用者を選択します。利用者を削除するには、以下の条件を満たす必要があります。
    • 該当の利用者が購入済サービスに割り当てられていないこと
    • 該当の利用者がその組織の唯一の管理者ロールとして割り当てられていないこと
    • 該当の利用者がログイン中ではないこと
  2. 削除をクリックし、削除します。

利用者がただちにプラットフォームから削除されます。
