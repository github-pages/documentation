サービス利用部門の登録

サービス利用部門の登録ページでサービス利用部門の情報を登録しておけば、サービス利用部門は自分の組織の登録操作を省略できます。

サービス利用部門を登録すると、新しい組織と最初の管理者アカウントが作成されます。すべてのログイン情報がこの管理者に自動的にメールで送信されます。 組織のデフォルトユニットも同時に作成されます。管理者は、デフォルトユニットの最初のメンバーになります。 組織に登録される利用者はすべてデフォルトユニットに自動的に追加されます。組織の管理者は追加のユニットを作成することができます。

サービス利用部門を登録するには、サービス利用部門を管理するサービスポータルが存在する必要があります。

サービス利用部門を登録するには：

  1. 新しいサービス利用部門の最初の管理者情報として以下の利用者データを入力します。
    • 性別: 男性または女性が設定できます。
    • 性: 組織の管理者の姓。
    • 名: 組織の管理者の名。
    • メールアドレス: 組織の管理者のメールアドレス。アカウントおよびアクセス情報の作成についての通知がこのメールアドレスに送信されます。

      指定されたメールアドレスの書式が有効な構文か確認され、ドメイン名がIANA（Internet Assigned Numbers Authority）によって定義、管理されている標準に適合しているか確認されます。

      例: user.name@domain.arpa, user.name@domain.org, user@mycompany.lan.uk

    • 希望する利用者ID：組織の管理者の利用者ID。利用者IDは100文字以内で以下の文字を含めることはできません。

      !" # $ % & ' * + , / :; < = > ?\ ^ `

      注: 利用者の認証に利用部門の組織では外部システムを利用する場合、登録する利用者IDは外部システムの既存のIDに一致させる必要があります。
    • 言語：デフォルトで管理者が使用する言語。 管理者に提供されるユーザーインターフェースや参考ドキュメント、メールなどは、ここで指定した言語で表示されます。 組織の管理者は後で言語を変更することが可能です。 利用可能な言語は、システム運用管理部門が定義するサポート言語に依存します。
  2. 新しいサービス利用部門に以下のデータを指定します。サービス利用部門を登録すると、そのデータを編集できるのは新しい組織の管理者のみになります。組織のデータを更新する場合、その組織の管理者として登録されている必要があります。
    • 組織名：会社名などの組織の名称。
    • 代表メールアドレス：組織の窓口となるメールアドレス。請求書送付用の別のメールアドレスを後で指定することができます。

      指定されたメールアドレスの書式が有効な構文か確認され、ドメイン名がIANA（Internet Assigned Numbers Authority）によって定義、管理されている標準に適合しているか確認されます。

      例: user.name@domain.arpa, user.name@domain.org, user@mycompany.lan.uk

    • 電話番号：組織の電話番号。
    • Webサイト：組織のホームページのURL。
    • 所在地：組織の所在地。
    • 国：組織が所在する国名。

      国別付加価値税率が定義されている場合、ここで指定した国名によってサービス利用部門向けの請求データの計算に使用される付加価値税率が決定されます。

    • 説明：組織の説明。会社のプロフィールからの情報などを示します。

      入力したテキストは、利用者のプロフィールで設定した言語で保存されます。テキストをローカライズするには、利用者のプロフィールの言語を変更し、該当する翻訳を入力します。

      注: 書式を変更する際は、エディタが提供するコントロールのみを使用してください。手作業でHTMLタグを入力したり外部のスタイルシートやコンテンツを参照すると、ページレイアウトがエラーになる場合があります。
  3. サービスポータルドロップダウンリストからサービス利用部門が使用するサービスポータルを選択します。 リストにはサービスポータル運用部門によってアクセス権を与えられているすべてのサービスポータルが含まれます。
  4. サービス利用部門のLDAPシステムによる認証を使用するには、LDAPを用いた認証を使用するチェックボックスを選択します。
    注: 利用者の認証がSAML 2.0 アイデンティティプロバイダー(IdP)で行われている場合、このオプションは利用できません。

    LDAP設定ファイルを作成し、このファイルでLDAP接続設定を定義します。設定ファイルを読み込むには、LDAP設定入力フィールド または参照をクリックし、その設定が含まれるファイルを選択します。組織固有の設定ファイルを指定しない場合、プラットフォームのLDAP接続設定が使用されます。

    外部のLDAPシステムを使用する場合、サービス利用部門は手動で利用者を登録する必要はありません。サービス利用部門の登録者はLDAPシステムから利用者をインポートすることができ、利用者の基本情報を自動的に登録することができます。

    設定ファイルのサンプルは以下の通りです: 

    LDAP_BASE_DN=ou\=people,dc\=est,dc\=mycompany,dc\=de
LDAP_CONTEXT_FACTORY=com.sun.jndi.ldap.LdapCtxFactory
LDAP_URL=ldap://myldapserver.lan.est.company.de:389
LDAP_ATTR_UID=uid
LDAP_ATTR_LAST_NAME=sn
LDAP_ATTR_FIRST_NAME=givenName
LDAP_ATTR_EMAIL=scalixEmailAddress
LDAP_ATTR_REFERRAL=ignore

    サンプルで示されているデフォルトの値は、LDAP_CONTEXT_FACTORYおよびLDAP_ATTR_UID用に定義されています。値が設定されていない場合, プラットフォームのデフォルトのLDAP設定で異なる値が定義されない限りは、組織を作成する際にこの値が適用されます。

  5. 入力内容を保存するには、保存をクリックします。

サービス利用部門の登録が完了すると、ログインデータや初回ログインで使用する初期パスワードが新しいサービス利用部門の管理者にメールで送信されます。

組織が外部認証システムを利用する場合：

  • そのシステムでは、パスワードが管理されます。 そのため、パスワードの記載が無いメールが管理者に送信されます。外部認証システムに登録されているパスワードはログインのために利用する必要があります。
  • 管理者の利用者IDはすでにプラットフォームで存在した場合、その管理者のメールアドレス（例：userSmith@organization.com>） はシステムへログインするために利用されます。さらにもう一つのサービス利用部門組織が作成され、管理者の利用者IDは同一である場合、新規利用者IDは既存のIDの後ろにアットマーク（@）とサービス利用部門組織のID（例：userSmith@abcdefg）で構成されます。管理者には該当利用者IDが記載されたメールが送信されます。

通常は次のステップとして、サービス提供部門またはサービス提供代行部門は、サービス利用部門が利用する購入済サービスの支払種別を選択できるように、使用可能な支払種別を選択します。請求先および支払種別は、プラットフォームの請求システムを使用する有料のサービスに対してだけ関係します。ただし、当該サービスに対しても、システム運用管理部門は、サービス利用部門に支払情報の入力を行わせないようにプラットフォームを設定することができます。この設定が行われている場合、支払種別の設定を行うことはできません。
