サービスの登録

サービスの登録ページでは、サービス利用部門に提供するための、サービスポータルへ公開するサービスを登録できます。サービスは直接、またはサービスの再提供設定を与えたサービス仲介部門およびサービス提供代行部門を介してサービス利用部門に提供できます。

サービスの登録とは、サービスのパラメーター、利用条件、制限事項を指定することです。

サービスを登録するには：

  1. 新しいサービスポータルへ登録済のサービスが準拠している技術サービスを選択します。

    技術サービスの説明が表示されます。

  2. 新しいサービスに以下の要素を指定します。
    • サービスID：サービスの識別子です。所属組織のサービスポータルへ登録済のサービス内で一意である必要があります。

      サービスIDは40文字以内で以下の文字を含めることはできません。

      !" # $ % & ' * + , / :; < = > ?\ ^ `

      このサービスIDは、サービスを更新または削除するとき、あるいはオプションのアップグレード／ダウングレードや料金設定を指定するときなどに選択できます。

    • サービス画像：サービスを特定するイメージファイルです。

      画像のアップロードフィールドまたは参照をクリックし、そのイメージを含むファイルを選択します。サポートされるファイル形式はGIF、JPG、PNGです。サービス画像のサイズは80x80ピクセルでなければなりません。

      サービスを公開するとサービスポータルのサービスタカログにサービス画像が表示されます。イメージを利用すると、サービス利用部門がサービスを識別しやすくなります。

    • 利用者向けサービス名：サービスの有意な名前です。サービス利用部門にとって認知しやすい値にする必要があります。このフィールドには、サービスポータルへ登録済のサービスが準拠する技術サービス名が事前に設定されています。これは必要に応じて更新できます。

      サービスを公開するとサービスポータルのサービスタカログにサービス利用部門向けのサービス名が表示されます。指定しない場合、代わりにサービス名が設定されていない旨のメッセージが表示されます。

      サービスを更新することによって、後で他の言語でカスタム組織向けサービス名を指定することもできます。

    • 概要：サービスの簡単な説明です。簡単な説明の長さは256文字までに制限されています。

      サービスを公開するとサービスポータルのサービスタカログに簡単な説明が表示されます。通常、この説明には、そのサービスが提供する機能の概要が含まれます。

      サービスを更新することによって、後で他の言語で簡単な説明を追加できます。

    • 説明：新しいサービスによって提供される機能の詳細な説明です。このフィールドには、サービスポータルへ登録済のサービスが準拠する技術サービスの説明が事前に設定されています。これは必要に応じて更新できます。

      サービス利用部門がサービスの詳細を要求すると、サービスポータルにサービスの詳細説明が表示されます。

      サービスを更新することによって、後で他の言語で説明を指定できます。

      注: 書式を変更する際は、エディタが提供するコントロールのみを使用してください。手作業でHTMLタグを入力したり外部のスタイルシートやコンテンツを参照すると、ページレイアウトがエラーになる場合があります。
  3. 購入手続きが完了したときにサービスを購入した利用者が自動的に購入済サービスに割り当てられるようにする場合、自動的に利用者を割り当てるチェックボックスを選択します。 デフォルトでは、サービスを購入する利用者を含めてすべての利用者は、サービスを利用する前に明示的に購入済サービスに割り当てる必要があります。
  4. 購入済サービスおよび使用に関連するパラメーターの値をサービスパラメーター表で入力します。
    • 購入済サービスのパラメーターには、購入済サービス当たりの利用者数の上限や、購入済サービスの最長期間などがあります。利用できる購入済サービスのパラメーターは、サービスの基盤となる技術サービスによって異なります。 使用法パラメーターには、基盤となるアプリケーションの特定機能の有効化または無効化、作成可能なフォルダ、ファイル、オブジェクトの上限数などが含まれます。基盤となる技術サービスによって使用法パラメーターが定義・提供されます。
    • 表の説明欄で、各パラメーターの簡単な説明を参照できます。この説明は技術サービス開発部門が提供します。
    • パラメーターに利用者による設定のチェックボックスを選択することによって、サービスを契約するサービス利用部門にオプションとして提供されるかどうかを指定できます。
    • 表の値欄で、各パラメーターの値を定義できます。値は、技術サービスに対して定義された範囲内でなければなりません。
  5. 必要に応じて、パラメーター設定ツールのURL を指定します。 この項目を指定することで、希望の外部ツールにリンクすることができます。 利用者は、サービスを購入する際にこの外部ツールを使って購入済サービスやパラメーターを構成できるようになります。
    外部ツールを使用する場合、パラメーターを設定するための表 サービスのパラメーター は、サービス購入処理中は外部ツールのインターフェースに置き換わります。
    注: パラメーター設定ツールのURLは、セキュアである必要があります。 指定されたURLについて、プラットフォームは構文もセマンティックも検証しません。
  6. 必要に応じて、カスタムタブのURLを指定します。この項目を指定することでサービスポータルの購入済サービスの詳細画面にタブを追加することができます。本URLはカスタムタブ向けのWebページもしくはWebアプリケーションのコンテンツを表示します。たとえば、IaaSサービスの利用者向けにサービス詳細やサービスの操作に関する情報を表示することが可能です。
    注: 必ず安全なカスタムタブ向けのWebページやWebアプリケーションに接続するURLを指定してください。プラットフォームはURLの正当性(シンタックスや内容)を確認しません。
  7. 必要に応じて、利用者に表示されるカスタムタブの名前を指定します。40文字以内の英数字で指定します。特殊文字は使用できません。

  8. 保存をクリックしてサービスを登録します。

サービスが登録され、その後の操作がおこなえるようになります。まず、料金設定を設定し、公開するサービスポータルを選択する必要があります。その後、サービスを有効化することでサービス利用部門はそのサービスを購入できるようになります。

任意の操作として、サービスのアップグレード／ダウングレードオプション用の定義やサービス仲介部門およびサービス提供代行部門への再提供設定の付与なとがおこなえます。
