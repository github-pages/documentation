請求システムの管理

請求システムの管理ページで、技術サービス提供部門およびサービス提供部門が、料金設定の提供およびサービスの料金計算に使用できる請求システムの参照、追加、および管理を行うことができます。

プラットフォームには支払種別の管理とサービスのコスト計算のための請求システムが付属しています。 ただし、付属の請求システムを使用せずに一つ以上の外部の請求システムと統合することもできます。これは、Amazon Web Services (AWS)など、独自の価格設定、および請求機構を持つ外部のプロバイダーによるサービスを提供する場合に適しています。また、プラットフォームで提供されていない計測情報に基づいて価格設定、および請求を行いたいサービスにも適しています。

技術サービス提供者は、技術サービスを定義する際に、使用する請求システムとして、プラットフォームのもの、または統合された外部の請求システムを選択することができます。請求システムのうちどれかひとつがデフォルトとして設定されます。明に請求システムを指定しない場合、デフォルトの請求システムが技術サービスに設定され、当該技術サービスに基づくサービスおよび購入済サービスは、デフォルトの請求システムを使用します。

請求システムの管理ページでは使用可能な請求システムをそれぞれ参照することができます。このページを最初に訪れた際はデフォルトの請求システムの詳細が表示されます。このページでは次の操作を行うことができます:

  • 使用可能な請求システムの参照
  • 外部の請求システムの追加
  • デフォルト請求システムの設定
  • 外部の請求システムに対する、簡単な説明と接続プロパティの編集
  • 請求システムの削除

請求システムの追加方法:

  1. 次の条件を満たしていることを確認してください:
    • プラットフォームを外部の請求システムに接続するための請求システムアダプターが実装されている。アダプター開発者は外部の請求システムを使用することになるサービス提供者と密に連携してアダプターを開発する必要があります。詳細は技術サービス開発部門編を参照してください。
    • アプリケーションサーバに配備された請求システムアダプターにより、プラットフォームと外部の請求システムが通信可能な状態にある。
    • JNDIルックアップを使用して請求システムアダプターのリモートインタフェースを呼び出すためのパラメーターを知っている。
  2. 請求システムの管理ページのいちばん下にある追加ボタンをクリックします。 請求システムの詳細を定義するためのフィールドが表示されます。
  3. 請求システムの識別子を入力します。 識別子はプラットフォームで一意である必要があります。識別子は後から変更することはできません。識別子は請求システムを使用する技術サービスの定義において指定されます。

    識別子は40文字以内、かつ次の文字を使用することはできません:

    ! " # $ % & ' * + , / : ; < = > ? \ ^ `

  4. 必要に応じて簡単な説明を入力します。

    簡単な説明は40文字以内で指定します。すべての文字を使用することができます。

  5. JNDIルックアップを使用して請求システムアダプターのリモートインタフェースを呼び出すため、接続プロパティを入力します。 少なくとも請求システムアダプターのJNDI名を指定する必要があります。さらにプロパティを追加する場合は、プロパティ名を左のフィールドに値を右のフィールドに入力します。プロパティの追加は"+"ボタン、削除は"-"ボタンをクリックします。
  6. 請求システムとの接続確認を行うには接続テストボタンをクリックします。 接続が成功したかどうかをあわらすメッセージが表示されます。
  7. 請求システムをデフォルトの請求システムにしたい場合、デフォルトに設定ボタンをクリックします。 請求システムはただちにデフォルトとラベルが追加され、元のデフォルト請求システムからはラベルが削除されます。
  8. 請求システムを定義するには保存をクリックします。

技術サービス提供部門はアプリケーションを技術サービスとしてプラットフォームに登録するとき、新しい請求システムをすぐに指定することができます。

デフォルトの請求システムを変更する方法:

  1. デフォルトにしたい請求システムをクリックし、展開します。
  2. デフォルトに設定をクリックします。
  3. 必要に応じて設定を行い保存をクリックします。

既存の請求システムを更新する方法:

  1. 更新したい請求システムをクリックし、展開します。
  2. 必要に応じて簡単な説明を変更します。
  3. 必要に応じて接続プロパティの値を変更します。プロパティの追加は"+"ボタン、削除は"-"ボタンをクリックします。
  4. 請求システムとの接続確認を行うには接続テストボタンをクリックします。
  5. 請求システムをデフォルトの請求システムにしたい場合、デフォルトに設定ボタンをクリックします。
  6. 編集内容を有効にするには保存をクリックします。

請求システムを削除するには:

  1. 削除したい請求システムをクリックし、展開します。

    請求システムを削除するには次の条件を満たしている必要があります:

    • 請求システムがデフォルトに設定されていない。
    • 請求システムを使用している技術サービス、サービス、およびアクティブな購入済サービスが存在していない。
  2. 削除をクリックします。

    請求システムはただちに削除されます。

注: この条件を満たせばプラットフォームの請求システムを削除することもできます。削除したプラットフォームの請求システムを再び使用するには、請求システムIDにNATIVE_BILLINGと指定して請求システムを追加してください。このとき、簡単な説明は任意で指定し、接続プロパティの指定は必要ありません。
