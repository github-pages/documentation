カスタム属性の追加

カスタム属性の追加ページでは、サービス利用部門や購入済サービスのためのカスタム属性を追加できます。

カスタム属性を追加するには:

  1. 属性IDに属性の識別子または名前を入力します。 属性IDは40文字以内に制限され、以下の文字は使用できません。:

    ! " # $ % & ' * + , / : ; < > \ ^ `

  2. カスタム属性をサポートされた言語に訳した名前で表示したい場合: 属性を表示したい言語を選択し、属性名に名前を入力します。

    255文字まで入力することができます。

  3. 既定値に属性の標準として用いる値を入力します。

    255文字まで入力できます。

    既定値は個々のサービス利用部門や購入済サービスに値が設定されていない場合に使用されます。サービス利用部門の管理者によって購入済サービスに対する属性の値が設定された場合(下記、手順3を参照)、その値はサービスの購読時にデフォルトの値としてプリセットされます。サービス利用部門の管理者によってサービス利用部門に対する属性の値が設定されている場合、サービスデータページで組織のプロフィールを編集するときに、対応する項目のデフォルト値としてあらかじめ設定されます。

  4. 利用者による入力チェックボックスにより、購入済サービスのカスタム属性に対してサービス利用部門の管理者またはサービス購入管理者が提供しているサービスを購入する際にカスタム属性を表示するかどうか、およびサービス利用部門のカスタム属性に対してサービス利用部門の管理者が組織のプロフィールを編集するときにカスタム属性を表示するかどうかを選択します。カスタム属性を表示した場合、管理者またはサービス購入管理者はサービス購入時に属性に対して値を設定できます。
  5. 必須チェックボックスにより、カスタム属性の値を入力必須にするかどうかを選択します。 上記手順3の利用者による入力が選択されている場合のみ、このチェックボックスが選択可能となります。
  6. サービス利用部門がOpenStackやAmazon Web Services (AWS)といったIaaSサービスを使用する場合、コントローラーIDに当該コントローラーのIDを入力することができます。詳細は技術サービス開発部門に問い合わせてください。コントローラーIDを指定することで、サービス利用部門がIaaSサービスに対して認証情報を設定することができます。これにより個々の購入済サービスに対して認証情報を管理することを避けることができます。 この機能を有効にする場合、次の点に注意してください:
    • サービス利用部門のカスタム属性を定義する必要があります。
    • 属性IDがサービスコントローラーにおける対応する設定と一致している必要があります。

      例: OpenStackの場合、API_USER_NAMEおよびAPI_USER_PWDをIDとするカスタム属性を作成します。AWSの場合、IDはACCESS_KEY_ID_PWDおよびSECRET_KEY_PWDです。

    • パスワードを入力するカスタム属性の場合、暗号化チェックボックスを選択してください。これによりレポート、ログ、およびDBにおいてパスワードは暗号化され、ユーザーインターフェイスから隠蔽されます。

    • サービス利用部門の管理者は、サービスポータルで組織のプロフィールを編集するときに、IaaSプロバイダーにおける組織の認証情報をカスタム属性に設定することができます。認証情報を変更した場合、サービス利用部門の購入済サービスすべてに反映されます。
  7. カスタム属性をログ、レポート、およびDBにおいて暗号化する場合、暗号化チェックボックスを選択します。 この場合、カスタム属性の入力値は隠蔽されます。
  8. 保存をクリックして入力内容を保存します。

カスタム属性の管理ページが表示され、新しいカスタム属性が画面に表示されます。

新しい属性の名前と値は、次回サービス利用部門の請求書作成処理がおこなわれる際に請求書に記載されます。属性の値が設定されていない場合、属性名のみが値無しで請求書に記載されます。
