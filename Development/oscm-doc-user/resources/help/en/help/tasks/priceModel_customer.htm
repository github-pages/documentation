Defining a customer price model

On the Define for customer page, you can define or update the price model for a selected service and customer. For a service using the platform's native billing system, you can edit the price model directly on this page. For a service using an external billing system, you can import a new or changed price model from the external system.

The customer price model you define or update takes priority over the original service price model. Click here for basic information on price models.

Usually, you change a customer price model only in agreement with the relevant customer. For updating a customer price model, the relevant marketable service must be deactivated. Your changes overwrite the previous values. The update does not affect existing subscriptions. The changes only take effect for new subscriptions of the customer after the service has been activated again.

To define a customer price model:

  1. From the Customer list, select the customer for whom you want to define the price model.

    The list shows all customers that have subscribed to one of your services and all customers you have registered, irrespective of whether they have subscribed to a service.

  2. From the Service ID list, select the service for which you want to define the price model.
  3. Depending on the billing system that is defined for the underlying technical service, proceed as follows:
    • If the platform's native billing system is used, define the price model in the subsequent sections of the page. Click here for details.
    • If an external billing system is used, import the appropriate price model from this system and specify the license agreement. Click here for details.
