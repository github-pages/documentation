Managing the access to a marketplace

On the Manage access page, you can decide who is allowed to access a marketplace.

When a marketplace is created, it is public and accessible by any anonymous or registered users. You can restrict the access to it to the users of specific registered organizations. In this case, the users need to log in to be able to see, subscribe to, and use the services published on the marketplace. New customers cannot register themselves on the marketplace but need to be registered by a supplier, broker, reseller, or the platform operator.

If you turn a marketplace from a public into a non-public one, organizations with existing subscriptions are automatically granted access to it. You can at any time revoke the access permission for each of these organizations in the same way as for organizations which were granted access explicitly. As a prerequisite, all subscriptions must have been terminated.

A marketplace with restricted access can at any time be made public again and thus accessible by any users and organizations.

To manage the access to a marketplace:

  1. From the Marketplace list, select the marketplace for which you want to manage the access permissions.

    The list contains all marketplaces to which you have access as a marketplace manager.

    The Authorized customers only option shows whether the marketplace is public or accessible by users of specific organizations only. In the latter case, the available organizations are listed below with check marks indicating the authorized ones.

  2. Change the access policy for the marketplace as required by selecting or deselecting the Authorized customers only option.
  3. For a marketplace for authorized customers, select the organizations that are to be granted access. Remove the check mark to revoke the access permission for an organization. As a prerequisite, the organization must not have any active subscriptions.
