Importing a service definition

On the Import service definition page, you can import the definition of a technical service from an XML file. This is one of the steps in the process of integrating an application with the platform.

If you want to update an existing technical service, import a technical service definition XML file which specifies the same name (ID) for the technical service as the one you want to update.

Observe the following:

  • If there is no marketable service and no subscription based on the technical service, you can change all properties of the technical service.
  • If there is a marketable service or a subscription based on the technical service, you cannot:
    • Change parameters and their options
    • Change event types
    • Remove parameters, options, or event types
  • Roles cannot be removed if prices are defined for them or users having one of the roles are assigned to a related subscription.
  • If you try to import an incomplete technical service definition XML file, default values are set as available. If the XML file is still incomplete, the import is aborted with an XML incomplete error. The import of a technical service definition with invalid attribute values is aborted with an Invalid attribute value error.

On the Import service definition page, you can also check the availability of existing services and view their details.

For details on technical service definition XML files, refer to the Technology Provider's Guide.

To import a technical service:

  1. Click the entry field or Browse to search your file system for the file containing the technical service definition.
  2. To import the technical service, click Import.
The imported service is added to the list of available services. In a next step, appoint one or more suppliers for the technical service so that marketable services can be defined based on it.

To view the details of existing services, click an entry in the list of services. On the Service page, you can see the service's name, the access type, access URLs and paths, the service's description, parameter and event definitions, user roles, and operations .

It depends on the implementation of the underlying application and on its integration which parameters, events, roles, and operation definitions are available.
