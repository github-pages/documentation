Managing languages

The platform supports multiple languages in which users can work. After installation, English, German, and Japanese language bundles are available. The operator can add additional languages and customize the texts provided in the English, German, and Japanese language bundles. Click here for the list of languages that can be added.

A language bundle consists of the following resources:
  • User interface resources: All texts that appear on the user interface or in generated emails.
  • Online help and FAQ HTML files: The online help topics are available in the administration portal, the FAQs are available on the marketplaces.
Adding a language or customizing existing texts comprises the following steps:
  1. Translating all resources of a language bundle:

    • The user interface resources are translated or updated in a Microsoft Excel file generated with the Export function on the Manage languages page.

      Click here for details.

    • The online help and FAQ files are translated or updated directly in the HTML files.

      Click here for details.

  2. For a new language: Registering the language with the platform on the Manage languages page.
  3. Importing the translated or updated Microsoft Excel file.
  4. Deploying the translated or updated online help and FAQ files.
  5. For a new language: Activating the language in the platform on the Manage languages page. All activated languages are available to users for selection when they edit their user profile.

On the Manage languages page, you can:

  • View the list of supported languages. The default language is English.
  • Register a language with the platform by adding a language code. This is the prerequisite for providing translations in a different language than the ones available after installation (English, German, Japanese).
  • Export the user interface resources and import your translations into the platform.
  • Update existing translations.
  • Activate or deactivate a language.

To register a language:

  1. In the ISO language code field, enter the two-letter ISO code of the language to be added. For example, enter cs for the Czech language.

    The platform supports left-to-right languages. Right-to-left or top-to-bottom languages as well as variants of languages, such as British and US English, are not supported.

    Click here for the list of languages that can be added.

  2. Click Add. By default, the newly added language is deactivated, i.e. it is not yet available to users for selection.

You can now proceed with:

  1. Exporting the user interface resources and translating them.
  2. Importing the translated texts and names.
  3. Translating online help and FAQ files and providing them as described in Online help and FAQs.

    You can also provide a subset of the resources that can be translated. For example, you can decide to have the online help, FAQ, and email in Hungarian, but leave the user interface in English. When a user then sets Hungarian in his user profile, the user interface will still be in English, but the other texts are shown in Hungarian.

  4. Activating the new language so that it becomes available to users when selecting their working language.

To export, translate, and import the user interface resources:

  1. Click Export and save the resources to a Microsoft Excel file in .xls format on your file system.
  2. Translate the resources as required. Click here for details.
  3. Import the translated Microsoft Excel file:

    1. Click the File name field or Browse, and select the file containing the translations.

    2. Click Import.

    Make sure that your customized file is a Microsoft Excel file containing three worksheets with five columns each (keys, the three system default languages de, en, ja, and the new language).

    If a label or string does not exist in the new language, the English term is used. If the English term is not defined either, the technical key is used.

  4. Click Save for your changes to become effective.

To update the translations of an existing language:

  1. Click the line of the language you want to update.

    You can select any of the available languages including the system default languages (de, en, or jp).

  2. Click Export and save the resources to a Microsoft Excel file in .xls format on your file system.
  3. The column headed by the code of the selected language is filled with already existing translations. Edit the strings as required. Click here for details.
  4. Import the updated Microsoft Excel file:

    1. Click the File name field or Browse, and select the file containing the updated translations.

    2. Click Import.

  5. Click Save for your changes to become effective.

To activate a language:

  1. Select the Active check box beside the language you want to activate.

    You can select multiple languages at a time.

  2. Click Save for your changes to become effective.

All active languages are available to users for selection when they edit their user profile.

To deactivate a language:

  1. Deselect the Active check box beside the language you want to deactivate. You cannot deactivate the default language English.

    A message is displayed telling you that the language will no longer be supported as soon as you confirm your changes by clicking Save.

  2. Click Save for your changes to become effective.

If the language you deactivate is set in the user profile of users working with the system:

  • The users' language is set to the default language English. This means that all texts are presented in the default language.
  • The users must update their profile: They can either stay with the default language or select another active language.
  • The deactivated language is no longer available for selection.
