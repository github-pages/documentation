Viewing price model details

On the View service and Manage service page, you can view the following details of the charging conditions and prices for using the service. All prices are without VAT.

Charging conditions

The charging conditions include the following information:

  • Calculation option: The cost calculation option for the price model:
    • Free of charge: The service is free of charge.
    • Pro rata calculation: The costs are calculated based on the exact usage of the service in milliseconds.
    • Per time unit calculation: The costs are calculated for complete time units, for example, whole days or weeks. The time unit is defined under Charges based on.
  • Free trial period (in days): The number of days during which the service can be used for free. After this period, the service automatically turns into a chargeable service.
  • Currency in which a customer is charged for using the service. The supported currencies are defined by the operator. The currency cannot be changed when defining a subscription price model.
  • Charges based on: The time unit for recurring charges. The charges can be on a monthly, weekly, daily, or hourly basis. The option selected here also defines the time unit for per time unit calculation. The time unit cannot be changed when defining a subscription price model.
  • One-time fee to be paid when subscribing to the service. This fee is added to the usage charges for the first billing period.
  • Recurring charge per subscription: The charge to be paid for a subscription on a recurring basis as specified in the Charges based on field.
  • Recurring charge for users: The charge to be paid per time unit users are assigned to a subscription. The time unit is defined in the Charges based on field. For this type of charge, you can specify stepped prices, for example, to grant discounts depending on the amount of time users work with a subscription

Event prices

Event prices refer to individual, billable events. These include, for example, login and logout by users to the underlying application, the completion of specific transactions, or the creation or deletion of specific data. It depends on the implementation and integration of the underlying application which events are available.

For this type of charge, stepped prices can be defined granting the customer, for example, discounts depending on the number of events that occurred while using the service.

Parameter prices

A price model can define prices for service parameters and options. It depends on the implementation and integration of the underlying application whether and which parameters and options are available. The supplier can define a price for every parameter and option, and specify whether this price is to be charged per subscription or per user assigned to the subscription. Numeric parameters are a multiplier for the price. For boolean parameters, the multiplier is 1 if the value is true. In all other cases, the multiplier is 0.

For numeric parameters (parameters with values of type Integer or Long), stepped prices can be defined granting the customer, for example, volume discounts.

Role-specific prices

A price model can define a price for each service role that is defined. This price is added to the base price per user in the cost calculation for a billing period.

The service roles must have been specified in the technical service definition. Service roles can be mapped to corresponding permissions in the underlying application.
