Defining a service price model

On the Define for service page, you can define or update the price model for a service that uses the platform's native billing system, or import a new or changed price model for a service that applies an external billing system.

The price model specifies whether and how much customers subscribing to the service are charged for using a subscription. Click here for basic information on price models.

For each marketable service, a service price model must have been defined before you can activate the service and thus offer it to customers for subscription. You can define exactly one price model for a service. You can, however, also define customer-specific price models or even subscription-specific ones.

For updating a service price model on the Define for service page, the relevant marketable service must be deactivated. If you update an existing price model, your changes overwrite the previous values. The update does not affect existing subscriptions. The changes only take effect for new subscriptions after the service has been activated again.

For granting brokers or resellers the permission to sell a service, it is a prerequisite to define a service price model. Brokers and resellers cannot change the prices you define. Resellers, however, can define a license agreement of their own.

If you update a price model for a service for which you have granted resale permissions, the brokers and resellers as well as their customers can instantly view any changes you make. A service that is already published is immediately offered with the new prices. Existing subscriptions to services are not affected by the update. Any changes you make are effective for new subscriptions only.

Note: For a marketable service that is based on a technical service with the external access type, there is no need to define a price model. With the external access type, the underlying application is responsible for controlling how customers are charged for using the service.

To define a service price model:

  1. From the Service ID list, select the service for which you want to define the price model.
  2. Depending on the billing system that is defined for the underlying technical service, proceed as follows:
    • If the platform's native billing system is used, define the price model in the subsequent sections of the page. Click here for details.
    • If an external billing system is used, import the appropriate price model from this system and specify the license agreement. Click here for details.

After activating the service, the price model is available to customers. It takes effect for all new subscriptions.

Brokers and resellers as well as their customers can instantly view any changes you have made.
