Managing organizations

On the Manage organization page, you can perform the following tasks:

  • Add another role to the organization (technology provider, supplier, reseller, broker)
  • Update the address and contact data of the organization.
  • Set the Distinguished name for an organization.

    This is required if the organization wants to use certificates for trusted communication between the organization itself and the platform operator organization.

    The platform checks the distinguished name in a client's certificate. If it matches the one specified here, the caller is assumed to be authorized to connect to the platform.

    You can retrieve the distinguished name of an organization from its certificate with the following call in your Java runtime environment: 

    <YourJRE>/bin/keytool -printcert -file <filename.crt>

    The process of integrating certificates is described in detail in the Operator's Guide.

  • Define the payment types that a supplier or reseller organization may offer to its customers for services applying the platform's billing system.

    By default, suppliers and resellers can allow customers to pay on invoice so that they can subscribe to a service that is not free of charge.

To manage an organization:

  1. From the Organization ID list, select the organization whose data you want to edit.

    The list contains all organizations managed on your platform.

  2. Edit the data as required:
    1. Organization. Name of the organization, for example the company name.
    2. Email. Email address for general communication with the organization. The organization can later specify a separate email address to which invoices can be sent.

      The system checks whether the syntax of the given email address is valid, and whether the domain name corresponds to the standards as defined and maintained by the Internet Assigned Numbers Authority (IANA).

      Examples: user.name@domain.arpa, user.name@domain.org, user@mycompany.lan.uk

    3. Phone. Telephone or mobile number of a contact person in the organization.
    4. Website. URL of the organization's website. This website can be accessed by customers when displaying details of the organization that offers a service they want to subscribe to.
    5. Language: Default language used in the organization.
    6. Address of the organization. The organization's administrator can later specify a different address to be printed on invoices.
    7. Country where the organization is located.
    8. Description: Description of the organization, for example information from the company's profile.
      Note: Make sure to use only the controls provided in the editor for formatting your text. Manually entered HTML tags and references to external style sheets or content may result in errors in the page layout.
    9. For a supplier organization: Default operator rev. share (%). This is the revenue share agreed between the operator and the supplier, which the operator is to receive from the supplier for selling services on the marketplaces managed on the operator's platform. The default operator revenue share applies to all services offered by the supplier as long as you do not define a service-specific revenue share using the Manage operator revenue share menu option in the Operation menu. It also applies irrespective of whether the supplier sells his services himself, or uses brokers or resellers for selling them. Discounts granted by a supplier to his customers are deducted from the revenue share.
    10. For a technology provider, supplier, reseller, and/or broker organization:

      Profile image: Image used on a marketplace. The profile image is shown as soon as a customer requests details of a service provided by this organization.

      Click in the Upload image field or click Browse, and select the file containing the desired image. Supported file formats are GIF, JPG, and PNG. The maximum size of the profile image is 154 x 154 pixels. The recommended size is 154 x 60 pixels.

  3. In the Organization role section, select a new role for the organization, if required. The reseller and broker roles cannot be combined with each other or with the technology provider or supplier role.
  4. In the Distinguished name field, enter the DN for the organization. The DN is defined in the organization's certificate and composed of the following elements:
    • CN (common name)
    • OU (organizational unit)
    • O (organization)
    • L (location)
    • ST (state)
    • C (country)

    Separate the individual elements of the DN with commas.

  5. Define the payment types that the organization can offer to its customers. Click Add, and proceed to the New payment type page. Adding payment types is possible for supplier and reseller organizations only.
    Note: You can no longer remove a payment type which has been assigned to a supplier or reseller organization.
  6. To save your entries, click Save.

The organization's administrators are notified by email about any changes you made.
