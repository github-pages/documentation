Managing LDAP settings

User IDs and passwords of an organization can be created and maintained in the platform or in an existing LDAP system of the organization.

When using an LDAP system, an organization does not need to register its users manually with the platform. The organization's administrator can import the users from the LDAP system, thus automatically registering them with the platform. The users are managed in the LDAP system. The platform continuously synchronizes its information on the users. Connection settings have to be defined in a configuration file so that the platform can connect to the LDAP system.

On the LDAP settings page, you can define and manage the LDAP settings for your organization. In doing so, you can overwrite, omit, or add parameters to the default settings defined for the platform by the operator.

In case you need to modify the LDAP settings, for example, when the connection URL has changed, you can export the current settings to a local configuration file, change the values as required, and import the settings again. Be aware that with this action you will overwrite all existing LDAP settings for your organization.

When accessing the LDAP settings page, the current LDAP settings are listed in a read-only table. You can see which values are based on default platform settings (they apply as long as you do not define organization-specific settings), as well as the currently defined key/value pairs.

To manage LDAP settings:

  1. Create an LDAP configuration file on your file system and define the LDAP connection settings in the file. You can also click Export to export the current LDAP settings from the platform to a file.
  2. Check the settings in the file and change them as required.

    Here is a sample configuration file: 

    LDAP_BASE_DN=ou\=people,dc\=est,dc\=mycompany,dc\=de
LDAP_CONTEXT_FACTORY=com.sun.jndi.ldap.LdapCtxFactory
LDAP_URL=ldap://myldapserver.lan.est.company.de:389
LDAP_ATTR_UID=uid
LDAP_ATTR_LAST_NAME=sn
LDAP_ATTR_FIRST_NAME=givenName
LDAP_ATTR_EMAIL=scalixEmailAddress
LDAP_ATTR_REFERRAL=ignore
  3. To load the configuration file, click the File name field or Browse, and select the file containing the settings.
  4. Click Import. All existing LDAP settings are replaced by the ones defined in the configuration file.
  5. Click Test connection to make sure that the connection to your LDAP system can be established.
The LDAP settings are immediately applied for your organization.
Note: When you provide invalid LDAP connection parameters, you and all users of your organization can no longer log in to the platform. In this case, contact the platform operator and request him to reset your organization's LDAP settings.
