Adding a custom attribute

On the Add custom attribute page, you can add a custom attribute for your customers or for your subscriptions, respectively.

To add a custom attribute:

  1. Type an identifier or name for the attribute in the Attribute ID field. Attribute IDs are restricted to 40 characters and must not contain any of the following characters:

    ! " # $ % & ' * + , / : ; < > \ ^ `

  2. If you want to offer the custom attribute with a name translated into one of the supported languages: Select the Language in which the attribute is to be displayed, and enter a name for it in the Attribute name entry field.

    You can enter a maximum of 255 characters.

  3. Enter a default value for the attribute in the Default value field.

    You can enter a maximum of 255 characters.

    The default value is used as long as no specific value for an individual customer or an individual subscription is entered. If the value for an attribute for subscriptions is to be specified by the customer (see step 3 below), the corresponding field is preset with this default value during the subscription process. If the value for an attribute for customers is to be specified by the customer administrator, the corresponding field is preset with this default value when editing the organization's profile on the Service data page.

  4. Select the User option check box if the attribute is to be offered to a customer's administrator or subscription manager when he subscribes to one of your services (in case of a subscription-specific attribute), or to a customer administrator when editing the organization's profile. This allows the administrator or subscription manager to enter a value for the attribute.
  5. Select the Mandatory check box if you want to make the specification of the attribute value mandatory. The check box is only available if you selected User option in step 3 above.
  6. If your customers use an IaaS service such as OpenStack or Amazon Web Services (AWS), you can enter the ID of the corresponding service controller in the Controller ID field. Contact your technology provider for details. By specifying a controller ID, you can allow your customers to set his credentials for an IaaS service without having to maintain these credentials for each subscription separately. Be aware of the following if you want to offer this feature to your customers:
    • You need to define a custom attribute for customers.
    • The attribute ID must match the corresponding setting in the service controller configuration settings.

      For example, for OpenStack services you can create two custom attributes with the following IDs: API_USER_NAME and API_USER_PWD. For AWS, the IDs are ACCESS_KEY_ID_PWD and SECRET_KEY_PWD.

    • For the attributes used for entering passwords, select the Encrypted check box so that passwords are encrypted when stored in the database as well as in reports and log files, and so that they are hidden on the user interface.

    • The customer administrator will be able to set the credentials for his organization in the IaaS provider when editing the organization's profile on his marketplace. Whenever the customer administrator changes the credentials, they are applied to all subscriptions of this customer.
  7. Select the Encrypted check box if you want to encrypt the value of the custom attribute when stored in the database, as well as in log files and reports. In addition, any value the customer administrator enters in a field for an encrypted attribute will be hidden.
  8. To save your entries, click Save.

You are returned to the page for managing custom attributes. The new custom attribute is instantly displayed.

The name of the new attribute and its value will be included in the billing data for customers the next time the data is generated. If no value is specified for an attribute, the attribute name will be inserted into the billing data without a value.
