Defining role-specific prices

On the Define role-specific prices page, you can specify charges for service roles that are specified in the technical service definition. Service roles can be used to grant specific privileges to different users. They can be mapped to corresponding permissions in the underlying application.

For each service role, you can define a price. This price is added to the base price per user in the cost calculation for a billing period.

Note: Role-specific prices are relevant only for price models you define for services that are charged for using the platform's native billing system. Price models from external billing systems define their own elements that determine how the charges for a service are calculated.

To define prices for service roles:

  1. Select the service role for which you want to define prices.
  2. In the Recurring charge for users field, enter a value if the price is to be paid on a recurring basis for each user having the selected role.
  3. Specify a price for every available service option.
  4. To confirm your entries and define prices for another service role, click Apply. To confirm your entries and return to the Define for service, Define for customer, or Define for subscription page, click OK.
  5. To save the price model, click Save on the Define for service, Define for customer, or Define for subscription page.
