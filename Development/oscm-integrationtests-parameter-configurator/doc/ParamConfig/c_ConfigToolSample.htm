Integrating the Parameter Configuration Tool Sample with OSCM

Welcome to the sample for integrating a parameter configuration tool with OSCM. The sample tool is implemented in JavaScript. It can be integrated with OSCM to show you the key aspects involved in the integration process.

In the following sections, you can find an overview of the resources that make up the sample and a step-by-step description of how to integrate the sample with OSCM.

Sample Resources

The resources that make up the sample are provided in the oscm-integration-parameter-configurator.zip file:

  • WebContent directory - the source code files of the sample tool.
  • oscm-parameter-configurator.war file - the Web archive to be deployed for the sample tool.

To use the sample, extract the contents of the oscm-integration-parameter-configurator.zip file to a separate directory on your file system, and proceed as described in the next section.

Using the Sample Tool

As a prerequisite for using the sample tool, you must have access to a fully functional OSCM installation. To use the sample tool with OSCM, proceed as follows:

  1. Deploy the oscm-parameter-configurator.war archive to any domain on your Web/application server.
  2. Log in to the OSCM administration portal as a supplier.
  3. Select Marketable Service > Define a service, and create a marketable service. The marketable service may be based on any technical service that defines parameters.

    Select the User option check box for each parameter to offer the parameters to customers when they subscribe to the service.

    In the URL of a parameter configuration tool field, enter the URL pointing to the sample tool. Enter the URL in the following format:

    http://<server>:<port>/oscm-parameter-configurator/index.html

    <server> is the Web/application server where the sample tool has been deployed. <port> is the port to address the domain on the Web/application server.

  4. Define a price model for the service using the Define for service menu option in the Price model menu.
  5. Select Marketable Service > Define publishing options, and select the marketplace to which you want to publish the service.
  6. Select Marketable Service > Activate or deactivate services, and make the service available for subscription on the marketplace.
  7. Log in to the marketplace as a customer, select the service, and click Get it now to start the subscription process.
  8. On the Configuration page, define the subscription details. In the Parameters and options section, click Configure to access the sample tool.

    A modal window is displayed with the sample tool you have integrated.

    If the sample tool is not available at the specified URL, a warning is displayed and the standard OSCM Service parameter table is displayed instead.

  9. Modify the parameters as required with the sample tool. For each parameter, you see its price per subscription. The sample tool does not support prices per user, nor stepped prices.
  10. Click Configure in the modal window to finish.

    The window is closed.

  11. Finish the subscription process and confirm your subscription.

You can now continue with:

  • Using the service with the configured parameters.
  • Updating the configured parameters for the subscription, if required.
